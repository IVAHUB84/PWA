# ADR-022: Глобальный запрет выделения текста и анти-callout/анти-drag по изображениям (нативное ощущение UI)

- **Дата:** 2026-05-30
- **Статус:** принято
- **Связанные BA-требования:** `ba-req/ba-requirements-025.md` (закрывает ФТ-1…ФТ-8, НФТ-1…НФТ-6, КП-1…КП-7; ОВ-1 решён архитектором ниже, ОВ-2 — принятая бизнесом потеря, ОВ-3/ОВ-4 покрыты решением)
- **Связанные ADR:** прямых конфликтов нет. ADR-020/ADR-021 также трогают `docs/style.css`, но в других селекторах и по другим механизмам (press feedback, `text-size-adjust`/media-query). Решение оперирует свойствами `user-select`, `-webkit-touch-callout`, `user-drag`, которые ни в одном принятом ADR не зафиксированы как контракт.

## Контекст

Studio PWA — устанавливаемое статическое приложение (vanilla JS/HTML/CSS, `docs/`, GitHub Pages, без рантайм-npm-зависимостей). При long-press на мобильном интерфейс ведёт себя «по-вебовому»: выделяется текст кнопок/карточек/чипов/меток, на iOS Safari по изображениям всплывает нативное callout-меню сохранения, на десктопе картинку можно «утащить» (drag). Эффект усилен press feedback (req-023, v1.19.x): пользователь удерживает контрол, ожидая визуальной реакции нажатия, а получает системное выделение.

Текущее состояние кода (подтверждено):
- Глобального `user-select: none` нет. Точечно: `.tab` (`docs/style.css:142`), `.pin-key` (`docs/style.css:819`), трек карусели (`docs/style.css:852-853`).
- На слайдах карусели уже есть `img { pointer-events: none; -webkit-user-drag: none; user-drag: none; }` (`docs/style.css:856`) — функциональный свайп карусели (req-015/017) зависит от этого.
- `-webkit-touch-callout` нигде не задан.
- Приложение размещено непосредственно в `<body>` (`docs/prototype.html:18`), отдельной `#app`-обёртки нет; контент вложен в `.phone-wrap`/`.phone`.
- Поля ввода защищены от iOS-автозума правилом `input, textarea, select { font-size: max(16px, 1em) }` (`docs/style.css:8`). OTP-поля — `input.otp-box` × 4 (`docs/prototype.html:406-409`).
- Функционала копирования (`navigator.clipboard`/`execCommand`/кнопок «Скопировать») в кодовой базе нет.
- `style.css` входит в `APP_SHELL` (`docs/sw.js:7`); текущий `CACHE_VERSION = 'v72'` (`docs/sw.js:1`).

**Цель решения:** убрать «вебовость» — запретить выделение текста во всём UI, кроме полей ввода, и отключить нативное контекстное меню/перетаскивание по изображениям, не сломав ввод в формах и функциональные жесты (свайп карусели, скролл, тапы, press feedback). Чисто CSS, без новых зависимостей.

## Рассматриваемые варианты

### Корневой селектор для глобального правила (ОВ-1)

#### Вариант A: `user-select: none` на `body`
- Плюсы: один корень, правило наследуется на весь UI; исключения для полей задаются отдельным селектором с более высокой специфичностью.
- Минусы: затрагивает и desktop-«обвязку» вокруг рамки телефона (фон, scenario-bar) — но это не несёт вреда, там нет выделяемого контента, требующего сохранения.
- Последствия: минимальная, единая точка правила; согласуется с допущением Д-1 BA.

#### Вариант B: `user-select: none` на `.phone` (фактический контейнер экрана)
- Плюсы: ограничивает эффект «телом телефона», не трогая desktop-обвязку.
- Минусы: на десктопе вне рамки выделение остаётся возможным — расходится с ОВ-4 (единое поведение); элементы вне `.phone` (баннеры обновления SW `#sw-update-banner`, push-оверлеи) пришлось бы покрывать отдельно. Больше точек правила, выше риск пропустить узел.
- Последствия: фрагментированное правило, риск регресса при добавлении новых корневых узлов.

**Выбран Вариант A (`body`).** Единый корень, согласуется с ОВ-4 (единое поведение десктоп/мобайл) и Д-1, минимум точек правила, новые корневые узлы покрываются автоматически.

### Способ реализации запрета

#### Вариант 1: чистый CSS (`user-select`, `-webkit-touch-callout`, `user-drag`)
- Плюсы: декларативно, без рантайм-кода и зависимостей; не влияет на pointer/touch-события (свайп/скролл/тап/press feedback не затрагиваются — это разные подсистемы); тривиально откатывается `git revert`.
- Минусы: `user-select`/callout/drag — единственный реализуемый механизм; от «защиты контента» не спасает (и не должен — цель косметическая).
- Последствия: соответствует НФТ-1/НФТ-5, минимальный объём.

#### Вариант 2: CSS + JS-перехват (`contextmenu`/`selectstart`/`dragstart`)
- Плюсы: жёстче подавляет callout на отдельных движках.
- Минусы: рантайм-код, риск перехватить полезные события форм, противоречит «JS-перехват как основное средство — вне рамок» (§2 BA); избыточно для косметической задачи.
- Последствия: лишняя сложность и риск регресса ввода.

**Выбран Вариант 1 (чистый CSS).** JS-перехват вынесен как незапланированный запасной вариант; в плановый объём не входит.

### Исключение для полей ввода

Глобальный `user-select: none` ломает ввод/выделение/вставку в `input`/`textarea` (критично: PIN/OTP, телефон, формы записи, формы админа push). Решение: явный селектор-исключение `input, textarea, [contenteditable]` с `user-select: text` (+ префиксы) и `-webkit-touch-callout: default`. `[contenteditable]` сейчас не используется — добавляется на будущее без риска (Д-3). Специфичность тегового+атрибутного селектора достаточна, чтобы перебить правило на `body` (наследуемое `user-select` переопределяется собственным объявлением элемента).

## Решение

1. **Корень:** глобальное правило на `body` — `user-select: none` (с `-webkit-/-moz-/-ms-` префиксами) и `-webkit-touch-callout: none`.
2. **Исключение:** `input, textarea, [contenteditable]` — `user-select: text` (+ префиксы) и `-webkit-touch-callout: default`. Имеет приоритет над п.1.
3. **Изображения:** глобально для `img` — `-webkit-user-drag: none; user-drag: none;` и `-webkit-touch-callout: none` (запрет drag и callout по картинкам во всём UI, не только в карусели).
4. **Карусель:** существующее правило `.svc-carousel-slide img { pointer-events: none; -webkit-user-drag: none; user-drag: none; }` (`docs/style.css:856`) сохраняется без изменений (функциональный свайп зависит от `pointer-events: none` на `img`). Точечные `user-select: none` на `.tab`/`.pin-key`/треке карусели становятся избыточными, но остаются как есть — их удаление не входит в минимальный объём и не даёт наблюдаемого эффекта (ФТ-6).
5. **Реализация — только CSS** в `docs/style.css`; правок `docs/prototype.html` не требуется (CSS-правило на `img` покрывает все изображения; атрибут `draggable="false"` на отдельные `<img>` — не нужен).
6. **`CACHE_VERSION`** в `docs/sw.js` поднимается (`style.css` в `APP_SHELL`).

**ОВ-2 (потеря ручного копирования телефона/адреса/промокода):** бизнес принял потерю осознанно. Компенсирующий функционал (кнопки «Скопировать», точечные исключения `user-select: text`) в этой задаче НЕ вводится.

## Последствия

- **Положительные:** интерфейс перестаёт «прыгать» в выделение при long-press; на iOS нет callout-меню по картинкам; на десктопе картинку нельзя утащить; ощущение нативности; press feedback (req-023) воспринимается чисто. Декларативное CSS-решение, нулевой рантайм-оверхед, тривиальный откат.
- **Отрицательные / компромиссы:** теряется единственный текущий способ вручную скопировать телефон/адрес студии/промокод (выделением) — принято бизнесом (ОВ-2).
- **Технический долг:** избыточные точечные `user-select: none` (`.tab`, `.pin-key`, трек карусели) после обобщения дублируют глобальное правило — оставлены намеренно, чистка возможна в отдельном косметическом релизе. Если позже потребуется копирование контактов — отдельная задача (кнопка «Скопировать» или точечный `user-select: text`), вне рамок req-025.

## Влияние на код

- `docs/style.css` — добавление глобального блока правил (`body`, исключение `input/textarea/[contenteditable]`, глобальный `img`). Существующие селекторы не удаляются.
- `docs/prototype.html` — не затрагивается.
- `docs/sw.js` — инкремент `CACHE_VERSION` (v72 → v73), т.к. `style.css` в `APP_SHELL`.
- Новых зависимостей нет. Границы модулей и направление зависимостей не меняются. Ломающих изменений публичного контракта нет.
