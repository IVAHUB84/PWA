# ADR-027: Drill-down bottom-sheet записей и клиентов в админке (единый компонент)

- **Дата:** 2026-05-30
- **Статус:** принято
- **Связанные BA-требования:** `ba-req/ba-requirements-030.md` (req-admin-drilldown-029)
- **Связанные ADR:** ADR-024 (контакты студии: кликабельный/копируемый телефон, исключение из глобального `user-select:none` по образцу `.studio-contacts`)

## Контекст

Экраны админ-панели «Сводка» (`renderAdminDashboard`, строки `.booking-row`) и «Клиенты»
(`renderAdminClients` / `_renderClientRows`, строки `.client-row`) — «только для чтения».
Чтобы позвонить клиенту, администратор уходит в веб-кабинет YCLIENTS. Бизнес-цель — связаться
с клиентом за один-два тапа, не выходя из PWA (BA-030, раздел 1).

Действующие силы и ограничения:

- **Переиспользование UI (НФТ-2, Огр-2).** В проекте уже сложился паттерн bottom-sheet:
  динамически создаваемый `div` с `position:fixed;inset:0`, затемнённый фон,
  выезжающая снизу панель, закрытие тапом по фону через
  `overlay.addEventListener('click', e => { if (e.target === overlay) overlay.remove(); })`.
  Канонический образец — `docs/modules/history.js` (`_openHistPeriod`, строки 30–59):
  `overlay.dataset.histOverlay = '1'`, панель
  `background:var(--bg);border-radius:24px 24px 0 0;...;max-width:393px;margin:0 auto`,
  «ручка» `36×4px`. Идентичный паттерн — в `docs/modules/search.js` (строки 35–48).
  Демо-классы `.sheet` / `.sheet-overlay` из `scenarios.js` (десктопная media-query)
  переиспользовать **запрещено** (Огр-2).
- **Нормализация телефона.** В `docs/modules/studio.js` есть хелпер
  `_telHref(phone)` → `'tel:' + String(phone||'').replace(/[^+\d]/g,'')`, но он **приватный
  и не экспортируется** (строки 75–77). Его нужно либо экспортировать, либо вынести в общий util.
- **Источники данных.** Телефон записи берётся из `rec.client.phone` ответа `/records`
  (ОВ-1 РЕШЁН, доп. запросы `/clients` запрещены — Огр-3). Телефон клиента гарантирован:
  `renderAdminClients` маппит `phone: c.phone || ''` (Доп-2).
- **Глобальный `user-select:none`** (`docs/style.css:11–17`, введён в v1.20.0) скрывает
  возможность выделить/скопировать телефон. Нужно точечное исключение по образцу
  `.studio-contacts { user-select:text }` (`style.css:1102`, ADR-024 / BA-027) — ФТ-6, Огр-4.
- **Безопасность.** Имя/услуга/мастер/телефон — пользовательские данные из API; обязателен
  `esc()` (ФТ-10). Хелпер `esc` уже импортируется в `admin.js`.
- **PWA-целостность.** `admin.js`, `style.css`, `prototype.html` входят в `APP_SHELL`
  (`docs/sw.js:5–`, текущий `CACHE_VERSION = 'v79'`). При их изменении версию кэша поднять (НФТ-4).

## Рассматриваемые варианты

### Вариант A: Две независимые функции (`openBookingSheet` / `openClientSheet`), каждая строит свой overlay

- **Плюсы:** прямолинейно, каждая функция знает свою модель данных; нет промежуточной абстракции.
- **Минусы:** дублируется ~90% разметки overlay (фон, панель, «ручка», обработчик закрытия,
  блок телефона с `tel:`). Любая правка геометрии/закрытия/доступности телефона — в двух местах.
  Противоречит принципу «минимизируй дубли» из задания и духу Огр-2.
- **Последствия:** технический долг на каждое будущее изменение шита.

### Вариант B: Единый низкоуровневый хелпер `_openSheet(bodyHtml)` + две тонкие обёртки, формирующие тело

- **Плюсы:** один источник истины для каркаса overlay (фон, панель, «ручка», `data`-атрибут,
  закрытие тапом по фону, кнопка «Закрыть»). Обёртки `_openBookingSheet(i)` / `_openClientSheet(i)`
  только собирают экранированное тело (поля + опциональный блок «Позвонить»). Дубли сведены
  к минимуму, ровно по образцу `history.js`. Логика звонка/экранирования/`user-select` —
  в одном месте.
- **Минусы:** появляется небольшая внутренняя абстракция (одна функция-каркас).
- **Последствия:** последующие drill-down’ы (например, deep-link YCLIENTS из будущего релиза,
  ФТ-7 вне объёма) добавляются как новое тело без переписывания каркаса.

### Вариант C: Полноценный переиспользуемый модуль `sheet.js` с API (open/close, slots, заголовок)

- **Плюсы:** максимально чисто, переиспользуемо всем приложением.
- **Минусы:** избыточно для двух экранов; означает рефакторинг `history.js`/`search.js` под общий
  модуль, что выходит за объём релиза (принцип минимальных изменений, НФТ-1). Риск регрессий в
  не относящихся к задаче экранах.
- **Последствия:** раздувание объёма релиза без бизнес-ценности «здесь и сейчас».

## Решение

Выбран **Вариант B**: единый локальный хелпер-каркас `_openSheet(bodyHtml)` в `docs/modules/admin.js`
и две тонкие обёртки `_openBookingSheet` / `_openClientSheet`, переиспользующие точный overlay-паттерн
из `history.js` (`position:fixed;inset:0`, панель `border-radius:24px 24px 0 0`, «ручка», закрытие
тапом по фону через сравнение `e.target === overlay`). Это минимизирует дубли (отвергнут A), не плодит
сквозной модуль и не трогает чужие экраны (отвергнут C).

**Нормализация телефона:** хелпер `_telHref` из `studio.js` сделать **экспортируемым** и импортировать
в `admin.js` — единый источник нормализации `tel:` (без дублирования регэкспа, без копипасты).
Альтернатива «продублировать строку в admin.js» отвергнута как нарушение DRY при уже существующем хелпере.

**Кликабельность строк:** навешивается через делегирование/`data`-индекс и `window`-функции
(стиль, уже принятый в проекте: `onclick="..."` с `data-*`, см. `_renderClientRows`/`renderAdminFeed`).
Каждая строка получает `data`-индекс записи/клиента и `onclick`, открывающий соответствующий шит.
Это не затрагивает `filterAdminClients`: фильтр пере-рендерит строки через `_renderClientRows`,
а кликабельность задаётся прямо в шаблоне строки — она автоматически сохраняется после фильтрации (ФТ-9).
Источник данных для шита записи — модульный массив отрисованных записей (по образцу `_adminAllClients`),
чтобы по индексу достать `rec` без повторного запроса (Огр-3).

**Доступность телефона:** новый CSS-класс (напр. `.sheet-phone`) с `user-select:text` /
`-webkit-user-select:text` / `-webkit-touch-callout:default` — точечное исключение из глобального
`body{user-select:none}`, по образцу `.studio-contacts` (ADR-024).

## Последствия

- **Положительные:** единый каркас шита; нормализация телефона в одном месте (`_telHref` экспортирован);
  drill-down без новых сетевых запросов; согласованность с уже принятым overlay-паттерном и с линией
  контактов (ADR-024); поиск клиентов не затронут.
- **Отрицательные / компромиссы:** строки `.booking-row` хранятся «по индексу» в модульном состоянии —
  при повторном рендере дашборда состояние перезаписывается (допустимо: шит открывается из текущего списка).
  Делаем `_telHref` публичным API модуля `studio.js` — расширение его контракта (обратимо-совместимое).
- **Технический долг:** общий модуль `sheet.js` (Вариант C) откладываем до момента, когда шит понадобится
  третьему экрану или появится deep-link YCLIENTS (ФТ-7, вне объёма v1.25.0) — тогда оправдан рефакторинг
  каркаса в отдельный модуль.

## Влияние на код

- **`docs/modules/admin.js`** — затрагивается:
  - `renderAdminDashboard` — сохранять отрисованные `recs` в модульном состоянии; добавить
    `data`-индекс и `onclick` на `.booking-row`.
  - `_renderClientRows` — добавить `data`-индекс и `onclick` на `.client-row`
    (с учётом сохранения работы `filterAdminClients`).
  - Новые: `_openSheet`, `_openBookingSheet`, `_openClientSheet`; экспорт обёрток в `window`
    (`Object.assign(window, …)`), как принято в модуле.
  - Импорт `_telHref` из `./studio.js`.
- **`docs/modules/studio.js`** — `_telHref` помечается `export` (расширение контракта модуля).
- **`docs/style.css`** — новый класс доступности телефона в шите (исключение из `user-select:none`);
  при необходимости — лёгкие стили строки/курсора для кликабельности `.booking-row`/`.client-row`.
- **`docs/sw.js`** — `CACHE_VERSION` поднять (`v79` → `v80`), т.к. меняются файлы из `APP_SHELL`.
- **Не ломается:** сетевой слой (новых запросов нет, Огр-3), `filterAdminClients`,
  нижняя навигация и прокрутка (overlay — отдельный `position:fixed`-слой, удаляется при закрытии).
