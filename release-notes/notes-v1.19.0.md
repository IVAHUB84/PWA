# Release Notes v1.19.0

- **Дата:** 2026-05-29
- **Тип релиза:** minor

## Кратко

Studio PWA избавилась от двух мобильных дефектов Safari: текст больше не увеличивается самопроизвольно при повороте телефона, а при повороте в landscape приложение остаётся в привычном вертикальном мобильном виде — desktop-рамка «телефон в рамке» не появляется.

## Что нового

### Возможности
- Нет (релиз — фикс мобильных дефектов).

### Улучшения
- **Фиксация масштаба текста (баг 1):** в `docs/style.css` на корневом элементе (`html`) добавлены `-webkit-text-size-adjust: 100%` и `text-size-adjust: 100%`. Мобильный Safari больше не применяет авто-инфляцию размера шрифтов при широком вьюпорте и/или повороте устройства.
- **Ориентационно-нейтральный мобильный режим (баг 2):** условие единственного мобильного media-query изменено с `@media (max-width: 500px)` на `@media (max-width: 500px), (max-height: 500px) and (pointer: coarse)`. Телефон в landscape (ширина 700–900 px, высота < 500 px, `pointer: coarse`) теперь остаётся в полноэкранном мобильном виде; настоящий десктоп (`pointer: fine`) сохраняет превью «телефон в рамке». Тело media-query не изменялось.

### Исправления
- Откат к desktop-макету при повороте телефона в landscape — устранён.
- Самопроизвольное увеличение шрифтов iOS Safari — устранено.

## Изменения публичного контракта

Нет. Внешний API (YCLIENTS), CLI, схемы данных и манифест PWA не меняются. Меняется только адаптивная вёрстка (CSS) и версия кэша SW (`v67 → v68`). Миграционный путь не требуется — обновление SW штатным механизмом кэширования подхватит новый `style.css` автоматически.

## Известные ограничения
- Порог `max-height: 500px` — эвристика; крупные планшеты в landscape с высотой > 500 px и `pointer: coarse` попадут в desktop-превью. Это осознанный компромисс: планшеты вне scope задачи (BA §2, ADR-021).
- КП-1…КП-8 требуют финальной ручной верификации на реальном устройстве iOS Safari после деплоя — оба бага специфичны для мобильного Safari (см. раздел «Проверка»).
