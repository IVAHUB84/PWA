# Release Notes v1.8.0

- **Дата:** 2026-05-29
- **Тип релиза:** minor

## Кратко

Добавлен отдельный экран «Ближайшие визиты» (`s-upcoming`), на котором клиент видит все свои предстоящие будущие визиты с кнопками «Перенести» и «Отменить» по каждому. Экран открывается с главной по кнопке «Все →» (появляется при 2 и более предстоящих визитах) или тапом по блоку «Ближайший визит» — без необходимости заходить в Профиль → «История».

## Что нового

### Возможности

- **Экран «Ближайшие визиты» (`s-upcoming`).** Показывает все предстоящие (будущие, не отменённые) визиты клиента, отсортированные по дате по возрастанию. Прошлые визиты в списке отсутствуют.
- **Affordance «Все →» на главной.** При наличии 2 и более предстоящих визитов рядом с заголовком «Ближайший визит» появляется кнопка «Все →», ведущая на экран `s-upcoming`. При 0–1 предстоящих кнопка не отображается.
- **Тап по блоку «Ближайший визит» → `s-upcoming`.** Целевой экран тапа по hero-card изменён с `s-history` на `s-upcoming`: теперь и явная ссылка, и тап по карточке ведут в одно место — список предстоящих визитов.
- **Действия по каждому визиту.** На экране `s-upcoming` по каждому визиту доступны «Перенести» (`rescheduleRecord`) и «Отменить» (`cancelRecord`) — то же поведение, что на главном экране. «Повторить»/«Оценить» (для прошлых) отсутствуют.
- **Серверное слияние при входе.** При переходе на экран выполняется `_fetchAndMergeServerRecords` — список отражает актуальное состояние с сервера; повторный вход (через `back()` или навигацию) перестраивает список.

### Улучшения

- **Общий хелпер рендера предстоящих визитов.** Разметка строки предстоящего визита вынесена из `_renderHistoryFromCache` в переиспользуемые функции `_upcomingItemHtml` и `_upcomingListHtml` в `history.js`. Экраны `s-history` и `s-upcoming` используют единственную реализацию — исключено дублирование разметки.

### Исправления

- Нет.

## Изменения публичного контракта

Внешних контрактов (YCLIENTS API, схемы данных в `localStorage`, CLI) не меняется. Состав запросов к YCLIENTS не расширяется: `s-upcoming` использует уже существующий `_fetchAndMergeServerRecords`.

Изменения в публичном поведении приложения:
- Добавлен экран `s-upcoming` (новый экран в `SCREENS`).
- На главной при 2+ предстоящих появляется affordance «Все →».
- Тап по блоку «Ближайший визит» теперь ведёт на `s-upcoming` (ранее — `s-history`).

Миграция данных не требуется.

## Известные ограничения

- Экран `s-upcoming` недостижим обычной навигацией при 0–1 предстоящих визитах (нет «Все →»); при прямом открытии отображается «Нет предстоящих записей». Отдельный дизайн пустого состояния не разрабатывался (решение BA №4).
- После отмены или переноса визита список `s-upcoming` не перерисовывается мгновенно: действия уводят пользователя на другой экран; актуализация происходит при повторном входе на `s-upcoming` (onEnter + серверное слияние).
