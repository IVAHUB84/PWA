# Release Notes v1.10.0

- **Дата:** 2026-05-29
- **Тип релиза:** minor

## Кратко

Каталог услуг переведён на крупные карточки в один столбец с обложкой 16/9 (в духе раздела «Лента»), строка поиска по услугам стала заметно выделена. В каталоге теперь отображается реальный диапазон стоимости услуги «от {min} до {max} ₽», вычисленный из цен мастеров; после выбора конкретного мастера на экране подтверждения и при сохранении записи используется его персональная цена.

## Что нового

### Возможности
- Диапазон цен в каталоге услуг: для услуг, у которых разные мастера работают по разным ценам, отображается «от {min} до {max} ₽» (вычислено из цен мастеров через фоновый сбор `book_services?staff_id=`); единичная сумма — при равных ценах; «По запросу» — при отсутствии цены.
- Конкретная цена выбранного мастера на экране подтверждения «Стоимость» и в сохранённом визите — вместо диапазона.
- В режиме предвыбранного мастера в каталоге отображается его персональная цена, а не общий диапазон.
- Новая утилита `_fmtPriceRange(min, max)` — форматирует диапазон/единичную сумму/«По запросу»; формат единообразен в каталоге и поиске на главной.

### Улучшения
- Каталог услуг `s-services` — один столбец (было: две колонки); карточки крупнее, обложка сверху с пропорцией 16/9.
- Строка поиска на экране услуг (`#serviceSearchBar`) получила контрастную рамку и тень, явно различима на фоне экрана. Общий `.search-bar` и строка поиска на главной (`#homeSearchBar`) — без изменений.
- Фоновый сбор цен мастеров не блокирует первичную отрисовку каталога: сначала показывается фолбэк-цена из `/services/`, затем обновляется до диапазона.
- Кэш цен (`servicePriceRange`, `staffServicePrice`) — в памяти на сессию, без persist в localStorage.

### Исправления
- Цена на экране подтверждения и в записи теперь соответствует конкретному мастеру, а не отображает диапазон по услуге.
- Graceful degradation при недоступности данных мастера (`book_services`): каталог отрисовывается с фолбэк-ценой, ноль ошибок в консоли.

## Изменения публичного контракта

- **Внешний API YCLIENTS / CLI / схема БД** — не меняются. Используются уже применявшиеся эндпоинты `/book_services/{company}?staff_id=` и `/book_staff/{company}`.
- **Поле `price` визита** — значение изменено на конкретную цену выбранного мастера вместо каталожного `priceStr`; схема не меняется, миграция не требуется.
- **`utils.js`** — расширяется аддитивным экспортом `_fmtPriceRange`; `_fmtPrice` не изменена.
- **`state.js`** — аддитивные поля `servicePriceRange` и `staffServicePrice`.
- **CSS** — переопределены `.svc-catalog`, `.svc-cover`, `.svc-cover-img`, `.svc-cover-placeholder`, `.svc-name`, `.svc-meta`; добавлен локальный селектор `#serviceSearchBar`. Раздел «Лента», `.search-bar`, `#homeSearchBar` — без изменений.

## Известные ограничения

- Тесты Т-1…Т-12 требуют ручной проверки в браузере (DevTools → Console, Network, Application → Service Workers) — автотестов в проекте нет.
- `docs/version.json` (`build`) обновляется мейнтейнером при деплое (этап `/deploy`).
- Мок-данные `SERVICES_DATA` в `state.js` хранят готовые строки цены «от X ₽» как fallback до прихода данных из API — переписывание на диапазоны явно вне объёма этого релиза.
