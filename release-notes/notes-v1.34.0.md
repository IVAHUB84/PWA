# Release Notes v1.34.0

- **Дата:** 2026-05-31
- **Тип релиза:** minor

## Кратко

Релиз делает Studio PWA визуально завершённым и удобным на десктопе, планшете и при работе мышью. Фейковый iOS-статус-бар скрыт вне iOS, добавлены hover-состояния для мыши, снята блокировка зума, контентный текст стал выделяемым; мобильный сценарий и ранее принятые решения (a11y v1.33.0, in-app-фидбэк v1.32.0) не затронуты.

## Что нового

### Возможности

- **Оформление устройства на больших экранах.** На ширинах > 500px приложение отображается как осознанно оформленное «устройство»: брендовый градиентный фон (`var(--desktop-bg)`), рамка/тень «устройства» (`var(--phone-shadow)`), центрирование — вместо мелкого «телефона» на сером фоне. Оба токена зарегистрированы в светлой и тёмной теме.
- **Условный iOS-статус-бар.** Полоса `.status-bar` (59px «9:41» и dynamic island) схлопывается до нуля везде, кроме iOS standalone. Отображается только при наличии класса `ios-standalone` на корне — класс ставится при старте на основе детектора платформы.
- **Детектор iOS-standalone.** Экспортирована чистая функция `isIosStandalone({ standalone, iosPlatform }) → boolean` (тестируема без DOM) и хелпер `detectIosStandalone()`, использующий `display-mode: standalone` + `navigator.platform`/`maxTouchPoints`; UA — только как фолбэк. При неоднозначности — скрыть хром (консервативный дефолт).
- **Hover-состояния для мыши.** Новый блок `@media (hover:hover) and (pointer:fine)`: карточки услуг/мастеров, строки списков, слоты, вкладки, кнопки получают визуальную реакцию при наведении мышью. На тач-устройствах hover не «залипает».
- **Снятие блокировки зума.** Из `<meta name="viewport">` удалены `maximum-scale=1.0` и `user-scalable=no`; pinch-zoom и Ctrl+=/Ctrl+− работают на всех платформах (WCAG-совместимость). iOS auto-zoom на инпутах по-прежнему предотвращён `font-size: max(16px,1em)`.
- **Выделяемость контентного текста.** Глобальный `user-select:none` изменён на `user-select:text`; `none` сохранён точечно только на «кнопочных»/служебных элементах (`.tab`, `.tab-bar`, `.pin-key`, `button`, `[role="button"]`, `.fab`, `.drag-handle`, навигационных элементах). Цены, описания, адрес, телефон, имена мастеров теперь копируются.
- **Десктоп-install affordance.** Install-баннер и оверлей используют нейтральную формулировку «Установить приложение» — подходит для Chrome/Edge на десктопе через `beforeinstallprompt`. iOS-шаги Safari («Поделиться» → «На экран Домой») показываются только при `isIOS()`.

### Улучшения

- Токены `--desktop-bg` и `--phone-shadow` добавлены в обе темы (`:root` и `[data-theme="dark"]`).
- `initInstall()` ставит класс `ios-standalone` на `document.documentElement` один раз при старте — CSS управляет видимостью статус-бара без дополнительного JS.

### Исправления

- Нет исправлений багов вне области кросс-платформенного оформления.

## Изменения публичного контракта

**Новые экспорты `install.js`** (только добавление):
- `isIosStandalone({ standalone, iosPlatform }) → boolean` — чистая функция детектора
- `detectIosStandalone() → boolean` — хелпер сбора платформенных флагов

Существующие экспорты `isStandalone`, `isIOS`, `shouldOffer`, `triggerInstall`, `dismissInstall`, `initInstall` и поведение install-флоу не изменены.

**Viewport:** снят зум-лок — поведенческое изменение для пользователя, не API-контракт.

**CSS-класс `ios-standalone`** на `<html>` — внутренний, не публичный API.

Миграция не требуется: localStorage, API YCLIENTS, схемы данных не затрагиваются.

## Известные ограничения

- На больших экранах контент остаётся в «устройстве» шириной 393px — горизонтальное пространство не используется для многоколоночной раскладки. Полный адаптив — технический долг, при подтверждённой потребности выносится отдельным требованием.
- Эвристика iOS-определения (`navigator.platform`/`maxTouchPoints`) не на 100% точна для редких конфигураций (iPadOS Desktop mode) — обрабатывается консервативным дефолтом «скрыть хром».
- PWA работает только по HTTPS (GitHub Pages) или localhost — ограничение платформы.

**Критерии приёмки:**

- КП-1 (десктоп-оформление): требует ручной проверки в браузере — подтверждено по коду: `--desktop-bg`/`--phone-shadow` токены применены к `body`/`.phone`; правила вне мобильного медиа.
- КП-2 (условный iOS-хром): требует ручной проверки на iOS-standalone и десктопе — подтверждено по коду: `.status-bar { height:0 }` по умолчанию; `.ios-standalone .status-bar { height:59px }` только при классе; класс ставится через `detectIosStandalone()`.
- КП-3 (isIosStandalone — чистая функция): ПОДТВЕРЖДЕНО unit-тестами (4 кейса таблицы standalone × iosPlatform → ожидаемый bool, все прошли) + чтением кода: функция не обращается к DOM/navigator.
- КП-4 (hover на мыши): требует ручной проверки мышью на десктопе — подтверждено по коду: блок `@media (hover:hover) and (pointer:fine)` с нужными селекторами присутствует в style.css; на тач hover не сработает (медиазапрос).
- КП-5 (зум): требует ручной проверки — подтверждено по коду: viewport без `maximum-scale` и `user-scalable=no`.
- КП-6 (выделение текста): требует ручной проверки выделения/копирования — подтверждено по коду: `body { user-select:text }`, перечень `none`-классов соответствует политике.
- КП-7 (десктоп-install): требует ручной проверки в Chrome/Edge — подтверждено по коду: install-баннер `aria-label="Установить приложение"`, формулировка нейтральная; iOS-overlay (`s-install`) по-прежнему в DOM для iOS-кейса.
- КП-8 (мобильный не сломан / обе темы / ориентация / консоль): требует ручной проверки — подтверждено по коду: все новые CSS-правила вне `@media (max-width:500px)`; hover-блок под `(pointer:fine)`; мобильный медиа не затронут.
- КП-9 (регрессии записи / toast / a11y): требует ручной сквозной проверки — подтверждено по коду: `ui.js`/`uiQueue.js` не изменены (ADR-034); a11y-правила `:focus-visible`/`prefers-reduced-motion` не затронуты (ADR-035); существующие тесты зелёные.
- КП-10 (CACHE_VERSION): ПОДТВЕРЖДЕНО — `CACHE_VERSION = 'v91'` (было v90); APP_SHELL без новых записей.
