# Release Notes v1.2.0

- **Дата:** 2026-05-28
- **Тип релиза:** minor

## Кратко

Экран выбора мастера (`s-masters`) теперь показывает только тех мастеров, которые оказывают выбранную услугу, — данные берутся из YCLIENTS в реальном времени. Статическая фильтрация по полю `cats` полностью удалена.

## Что нового

### Возможности

- Фильтрация мастеров по выбранной услуге: при переходе на `s-masters` после выбора услуги список содержит только мастеров этой услуги (запрос `GET /book_staff/{company}?service_ids[]={serviceId}` к YCLIENTS).
- Индикатор загрузки «Загрузка мастеров…» на время выполнения запроса.
- Пустое состояние: если YCLIENTS вернул пустой список, показывается сообщение «На эту услугу сейчас нет доступных мастеров», кнопка «Любой мастер» скрывается.
- Fail-open при ошибке API: если запрос завершился ошибкой или таймаутом — показываются все мастера студии, кнопка «Любой мастер» остаётся видимой.

### Улучшения

- Кэш результата фильтрации в памяти на время сессии: повторное открытие экрана для той же услуги не делает повторный сетевой запрос.
- Защита от гонки рендера: при быстрой смене услуги/экрана устаревший async-ответ не перезаписывает актуальный список (проверка `state.serviceId` перед записью в DOM).
- Новая экспортируемая функция `fetchStaffByService(serviceId, _ycGet?)` в `api.js` изолирует запрос и упрощает тестирование (принимает инжектируемый mock).
- Новая экспортируемая функция `_intersectWithMasters(staffResult, mastersData)` в `masters.js` реализует пересечение ответа YCLIENTS с `MASTERS_DATA` и различает три исхода: непустой список / пустое состояние / fail-open.

### Исправления

- Устранён рассинхрон между статической таблицей категорий (`cats`) и реальными данными YCLIENTS: мастера, у которых `cats === []` из-за несовпадения имени, теперь корректно фильтруются по YCLIENTS.

## Изменения публичного контракта

Публичный контракт не изменён:

- Навигация (`selectMaster`, `selectAnyMaster`, `openMasterCard`, `bookServiceFromMaster`, режим `masterPreSelected`, «Все мастера студии») — без изменений.
- Внешний вид карточек мастера (`_masterCardHtml`) и логика бронирования — без изменений.
- Меняется только **состав** списка на `s-masters` в режиме перехода от услуги (намеренно, по требованию).

Миграция данных не требуется.

## Service Worker

- `CACHE_VERSION`: **v34 → v35** (принудительная инвалидация кэша клиентов после изменения `api.js` и `masters.js`).
- Состав `APP_SHELL` не изменился, новые файлы не добавлены.

## Известные ограничения

- Поле `cats` в `MASTERS_DATA` и его наполнение по совпадению имени в `app.js` сохранены как технический долг (ADR-002). Полная очистка — отдельная задача рефакторинга.
- Кэш фильтрации сбрасывается при перезагрузке страницы (приемлемо: данные свежие на каждую сессию).
- PWA работает только по HTTPS (GitHub Pages) или localhost; Service Worker недоступен по `file://`.

## Проверка

### Автоматические тесты

Прогнаны `docs/modules/masters-filter.test.js` (`node --experimental-vm-modules`): **31 тест, 0 упало**.

Охват тестами:
- `_intersectWithMasters`: непустой ответ, пустой ответ, ошибка (null/fail-open), отсутствие совпадений, нормализация числовых/строковых id, отбрасывание ID отсутствующих в `MASTERS_DATA`.
- `fetchStaffByService`: три исхода (массив / пустой массив / null), data не массив, data отсутствует, исключение сети.
- Логика кэша: успешный результат кэшируется после первого запроса; ошибка (`null`) не кэшируется; пустой успешный массив кэшируется.

ESLint (`npm run lint`): **0 ошибок, 0 предупреждений**.
