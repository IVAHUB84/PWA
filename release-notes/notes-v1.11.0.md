# Release Notes v1.11.0

- **Дата:** 2026-05-29
- **Тип релиза:** minor

## Кратко

Между каталогом услуг и выбором мастера появился новый промежуточный экран детальной карточки услуги: клиент видит название, описание, цену-диапазон с длительностью и фотографии работ, и только после этого осознанно переходит к выбору мастера. В каталоге обложка карточки берётся из реального фото услуги YCLIENTS (при наличии) вместо категорийного плейсхолдера. В режиме предвыбранного мастера поведение не изменилось.

## Что нового

### Возможности
- Экран детальной карточки услуги `s-service`: название, описание (из поля `comment` YCLIENTS), строка цена-диапазон + длительность, галерея фото работ, кнопка «Выбрать мастера».
- В каталоге обложка карточки услуги — первое фото из `image_group` YCLIENTS; при отсутствии фото — прежний категорийный плейсхолдер (градиент + эмодзи).
- Галерея: одно фото — статичное изображение без листалки; несколько фото — горизонтальная карусель с нативным CSS scroll-snap и точками-индикаторами; нет фото — категорийный плейсхолдер.

### Улучшения
- Маппинг услуги в `app.js` расширен полями `comment` и `photos` (нормализованный массив URL) — данные берутся из уже загруженного `/services/{company}` без дополнительных запросов.
- Защищённый хелпер `serviceImageUrls(image_group)` нормализует неоднородную структуру `image_group` YCLIENTS (массив фото у услуг / объект размеров у мастеров) в плоский массив URL-строк; любые отсутствия / null / неожиданные форматы → `[]`, ноль ошибок в консоли.
- `resolveServiceImage(service)` расширен приоритетом: фото услуги (`photos[0]`) → `SERVICE_PHOTO_MAP` → категорийный плейсхолдер; сигнатура и формат возврата сохранены.

### Исправления
- В обычном потоке тап по карточке каталога больше не ведёт немедленно к выбору мастера — открывается детальная карточка (осознанное поведение согласно BA-014).

## Изменения публичного контракта

- **Внешний API YCLIENTS / CLI / схема БД** — не меняются. Новых эндпоинтов не вводится: `comment` и `image_group` берутся из уже загружаемого `/services/{company}`.
- **Внутренняя модель услуги (`SERVICES_DATA`)** — аддитивно расширена полями `comment: string` и `photos: string[]`; остальные поля без изменений.
- **`serviceImages.js`** — добавлен экспортируемый `serviceImageUrls`; поведение `resolveServiceImage` расширено (обратная совместимость сохранена).
- **`constants.js`** — `SCREENS` расширен значением `'s-service'`.
- **Поведенческое изменение:** в обычном потоке тап по услуге ведёт на `s-service` вместо прямого перехода к `s-masters`. В режиме предвыбранного мастера поведение прежнее (`s-slots`). Изменение намеренное, согласовано с BA-014 и ADR-013.
- **`CACHE_VERSION`** поднята `v56` → `v57` (изменены файлы `APP_SHELL`: `prototype.html`, `style.css`, `app.js`, `services.js`, `serviceImages.js`).
- Миграция данных не требуется.

## Известные ограничения

- **Карусель не активируется на реальных данных YCLIENTS**, если сервис фактически хранит только одно фото на услугу (`POST /images/service` перезаписывает). Карусель реализована на массиве произвольной длины и включится автоматически, когда в `image_group.images` появятся несколько элементов — это отдельная контентная задача по загрузке фото в YCLIENTS (вне объёма данного релиза).
- Зум / полноэкранный просмотр фото (лайтбокс) — не входит в релиз (явно вне объёма по BA-014 §8 №4).
- `docs/version.json` (`build`) обновляется мейнтейнером при деплое (этап `/deploy`).
- Тесты Т-1…Т-13, требующие браузера, — см. раздел «Проверка» ниже.
