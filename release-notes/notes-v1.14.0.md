# Release Notes v1.14.0

- **Дата:** 2026-05-29
- **Тип релиза:** minor

## Кратко

Три доработки раздела услуг: исправлена механика и геометрия transform-карусели фото на детальной карточке (выравнивание, залипание свайпа, рывок при зацикливании); карточки каталога и слайды карусели получили «мягкие приподнятые» рамки с корректным отображением в тёмной теме; кнопка «‹ Назад» на экране услуг теперь показывается только там, где есть куда возвращаться.

## Что изменено

### Исправления

- **Карусель фото (Задача 1, BA-017).** Устранены три дефекта реализации transform-карусели на детальной карточке `s-service`:
  - Слайды смещались/обрезались, peek был неравномерным: инициализация геометрии (`setPos`, `updateDots`) перенесена в `registerOnEnter('s-service', …)` — измерения происходят **после** того, как экран `#s-service` стал активным/видимым.
  - Свайп залипал: `goTo()` при `transitioning` теперь вызывает `settle()` немедленно перед следующим жестом — подряд идущие свайпы не теряются.
  - Рывок при зацикливании: `settle()` стал идемпотентным (`if (settled) return`), `transitionGuard` используется только как страховка; `setPos(…, false)` с форсированным reflow (`getBoundingClientRect`) перед возвратом `transition`.
  - Добавлен пересчёт позиции при `resize`/повороте экрана; `resize`-обработчик снимается через `_carouselCleanup()` при каждом повторном открытии карточки (нет накопления listener-ов).

### Возможности / Улучшения

- **Рамки карточек (Задача 2, BA-018).** Раздел услуг получил единообразный стиль «мягкая приподнятая»:
  - `.svc-catalog-card`: `border-radius` 16px → 18px; тень в блоке «CARD POLISH» `0 1px 4px rgba(0,0,0,0.05)` → `0 4px 16px rgba(0,0,0,.08)`. `border: 1px solid var(--border)` сохранён.
  - `.svc-carousel-slide`: `border-radius` 16px → 18px; добавлена `box-shadow: inset 0 0 0 1px rgba(0,0,0,.06)` (обводка поверх фото, `flex-basis` не изменён — peek не нарушен).
  - `[data-theme="dark"] .svc-carousel-slide`: `box-shadow: inset 0 0 0 1px rgba(255,255,255,.08)` — обводка читается на тёмном фоне.
  - Прочие карточки (`master-card`, `hist-visit`, `cs-card`, `confirm-card`, `hero-card`) не изменены.

- **Кнопка «Назад» на s-services (Задача 3, BA-019).** Кнопка «‹ Назад» показывается только когда есть куда вернуться (`navHistory.length >= 2`):
  - При входе через таб «Услуги» или FAB «Запись» — скрыта (`visibility: hidden`, высота шапки сохраняется, заголовок не прыгает).
  - При входе через «Записать себя» с главного — видна и работает.
  - Реализовано через дополнение существующего `registerOnEnter('s-services', …)` в `app.js`; кнопка адресована по `id="servicesBackBtn"`, общий класс `back-btn` и логика `go`/`back`/`navHistory` не тронуты.

### Служебное

- `CACHE_VERSION` поднята `v62 → v63` (изменены файлы `APP_SHELL`: `services.js`, `style.css`, `app.js`, `prototype.html`).
- `docs/version.json`: build 29 → 30, version v1.14.0.

## Изменения публичного контракта

Внешний API / CLI / схемы данных не меняются. Источник галереи фото (ADR-014 G1), структура `navHistory`, сигнатуры `go`/`back`/`registerOnEnter` — без изменений. Наблюдаемые изменения UI: карусель работает корректно; рамки карточек и слайдов; кнопка «Назад» на `s-services` появляется условно. Миграционный путь не требуется.

## Известные ограничения

- Единообразие кнопки «Назад» на прочих корневых табах (лента, профиль, история) — отдельным требованием (BA-019 ОВ-2), не в этом релизе.
- Унификация карусели в переиспользуемый компонент — не выносится (ADR-014).
- Введение единой `--radius`/`--card-shadow` — не делалось (вне объёма BA-018).

## Проверка

**Статическая (код):**

- Задача 1 (карусель): `_initCarousel` вызывается через `registerOnEnter('s-service', …)` — геометрия измеряется после видимости экрана. `settle()` идемпотентен. `goTo()` при `transitioning` немедленно вызывает `settle()` — серия свайпов не теряется. `_carouselCleanup()` снимает `resize`-listener при каждом открытии. `esc()` используется на всех URL/alt включая клоны. Ветки 1 фото / 0 фото не изменены. — **пройдено статически**.
- Задача 2 (рамки): значения CSS точно соответствуют ADR-016 L1 (`border-radius: 18px`, `box-shadow: 0 4px 16px rgba(0,0,0,.08)` в CARD POLISH, `inset 0 0 0 1px …` на слайде, тёмная тема через `[data-theme="dark"]`). `flex-basis` слайда не изменён. Прочие карточки в CARD POLISH не тронуты. — **пройдено статически**.
- Задача 3 (кнопка «Назад»): `id="servicesBackBtn"` + `style="visibility:hidden"` в `prototype.html`; `navHistory` импортирован в `app.js`; условие `navHistory.length >= 2` в `registerOnEnter('s-services')`. `navigation.js` не изменён (подтверждено `git diff`). — **пройдено статически**.
- `CACHE_VERSION` v62 → v63: подтверждено в `sw.js`. — **пройдено**.
- Линт (`npm run lint`): чисто, без ошибок. — **пройдено**.

- Вид рамок в светлой и тёмной теме, видимость obvodki слайда на тёмном фоне (BA-018 КП-1, КП-2, КП-3).

- Вид других карточек (BA-018 КП-7).
- Отсутствие «прыжка» заголовка при появлении/исчезновении кнопки (BA-019 КП-2).
- Кнопка «Назад» при входе через таб / через «Записать себя» (BA-019 КП-1, КП-4).
- Кнопка «Назад» на других экранах — не изменилась (BA-019 КП-3).
