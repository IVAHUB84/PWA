# Release Notes v1.12.0

- **Дата:** 2026-05-29
- **Тип релиза:** minor

## Кратко

Детальная карточка услуги получила полноценную галерею: теперь отображаются все фото из YCLIENTS (до 8) вместо одного, карусель визуально считывается как листаемая (скругления 16px, соседние слайды выглядывают по краям), зациклена в обе стороны без скачков. Фото автоматически обновляются при смене контента в YCLIENTS — без переустановки приложения.

## Что нового

### Возможности
- **Полная галерея фото услуги.** При старте приложения каталог дополнительно запрашивает `GET /company/{company}/services/` и сливает полную галерею (`image_group.images` массивом) точечно в поле `photos` каждой услуги по `id`. Если management-эндпойнт недоступен, каталог деградирует на одиночные фото без ошибок.
- **Зацикленная transform-карусель с клонами.** Лента слайдов рендерится как `[clone(last), s0…sN-1, clone(first)]`; переход с последнего фото ведёт на первое и обратно, бесшовно; логический индекс ведётся в JS, сброс позиции на краю — без анимации строго в `transitionend`.
- **Stale-while-revalidate для фото YCLIENTS.** В Service Worker добавлена точечная SWR-ветка для `assets.yclients.com` (пути `main_service_image/`, `gallery_service_image/`): кэш показывается мгновенно, свежее фото дотягивается в фоне — обновление видно при следующем открытии карточки; оффлайн-просмотр ранее виденных карточек сохранён.

### Улучшения
- **Скругления 16px** на слайдах карусели, одиночном фото и категорийном плейсхолдере — согласованы с радиусами карточек проекта.
- **Peek-аффорданса (~14px).** Слайд имеет ширину `calc(100% - 44px)`, контейнер — `overflow:hidden`; соседние слайды выглядывают по краям, сигнализируя о листаемости. Работает и при ровно двух фото.
- **Touch-свайп** на `touchstart`/`touchmove`/`touchend` (все passive): порог 30px по горизонтали, вертикальный свайп не перехватывается.
- `CACHE_VERSION` поднята `v57 → v58` — старые кэши инвалидируются при активации нового SW.

### Исправления
- Убран устаревший слушатель `scroll` для синхронизации точек-индикаторов (ADR-013). Точки теперь синхронизированы по логическому индексу JS, в том числе при зацикленном переходе через край.
- CSS-ветка карусели переведена с `scroll-snap` + `overflow-x:auto` на `transform:translateX` + `overflow:hidden` — исключён горизонтальный скролл контейнера.

## Изменения публичного контракта

- **Внешний API PWA / CLI / схема БД:** не меняются.
- **Потребляемый YCLIENTS API:** добавлен один эндпойнт чтения — `GET /company/{company}/services/` (тот же токен/база, только для слияния `photos`). Прочие эндпойнты и поток записи без изменений.
- **Внутренняя модель услуги (`SERVICES_DATA`):** поле `photos` теперь может содержать до 8 URL вместо одного (обратно совместимо — ветки 0/1/N≥2 уже существовали).
- **Service Worker:** стратегия кэширования для `assets.yclients.com` (пути `main_service_image/`, `gallery_service_image/`) изменена с cache-first на SWR; `CACHE_VERSION` `v57 → v58`.
- **Миграционный путь:** не требуется (данные эфемерные, грузятся из API при старте).

## Известные ограничения

- Бесшовность сброса позиции на краях карусели и плавность свайпа требуют обязательной ручной проверки на iOS/Android-устройствах.
- Зум / полноэкранный лайтбокс фото — вне объёма (явно out of scope по BA-015 §2).
- Автопрокрутка карусели по таймеру — вне объёма (BA-015 §8 №6).
- Счётчик «N / M» — не добавлен (необязателен по BA-015 §8 №2).
