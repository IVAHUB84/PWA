# Release Notes v1.5.0

- **Дата:** 2026-05-28
- **Тип релиза:** minor

## Кратко

В релизе реализован единый механизм предложения установки PWA на экран «Домой»: фирменный оверлей при открытии на мобильном устройстве, программная установка на Android/Chrome через `beforeinstallprompt`, ручная инструкция на iOS, правило «30 дней после отказа / никогда после установки» и унификация всех точек управления видимостью.

## Что нового

### Возможности

- Новый модуль `docs/modules/install.js` — единый источник состояния и поведения установки: перехват `beforeinstallprompt`, детект standalone и платформы, правило повторного показа, показ/скрытие фирменного оверлея и `#installBanner`.
- Фирменный оверлей `#installOverlay` в стиле приложения: заголовок «Установите приложение», мотивационный подзаголовок, кнопки «Установить» и «Позже».
- Программный install prompt на Android/Chrome: нажатие «Установить» вызывает `deferredPrompt.prompt()` через перехваченное событие.
- iOS-ветка: нажатие «Установить» на iOS ведёт на экран-инструкцию `s-install` с ручными шагами «Поделиться → На экран «Домой» → Добавить».
- Правило повторного показа: 30 дней после отказа; после установки — никогда. Единый ключ `yc_install_state` в `localStorage`.

### Улучшения

- `profile.js` / `renderHomeHero` делегирует видимость `#installBanner` модулю `install.js` вместо прямого управления по `yc_install_seen`.
- Кнопки экрана `s-install` («Понятно, добавлю» / «Позже») переведены на `dismissInstall()` — фиксируют отказ в едином ключе состояния.
- `#installBanner` на главной теперь тоже запускает `triggerInstall()` и управляется единым модулем.
- Реактивный показ оверлея: если `beforeinstallprompt` приходит после первого рендера, модуль повторно проверяет условия и может показать оверлей.

### Исправления

- Устранены два конкурирующих предложения установки (оверлей и баннер одновременно): модуль гарантирует единственный активный канал.
- Устранено показывание предложения пользователям, уже запустившим PWA в standalone-режиме.

## Изменения публичного контракта

**Новые window-функции (аддитивно):** `initInstall`, `attachInstallListeners`, `maybeShowInstallOverlay`, `triggerInstall`, `dismissInstall`, `refreshInstallBanner`, `isStandalone`, `isIOS`, `shouldOffer`, `readState`, `writeState`, `showOverlay`, `hideOverlay`.

**localStorage:** вводится ключ `yc_install_state` (JSON: `{ dismissedAt?, installed? }`). Ключ `yc_install_seen` прекращает использоваться как источник видимости (запись/чтение из кода удалены); значение у существующих пользователей сохраняется в storage, но игнорируется. Миграция не требуется — отсутствие `yc_install_state` трактуется как «никогда не предлагали».

**Разметка:** добавлен `#installOverlay` в `docs/prototype.html`. `#installBanner` и `s-install` сохранены с прежними `id`.

**Service Worker:** `CACHE_VERSION` поднята до `v38`; `./modules/install.js` добавлен в `APP_SHELL`. Все пользователи получат новую версию кэша.

**Манифест, иконки, Worker, YCLIENTS-запросы:** без изменений.

## Известные ограничения

- На Android/Chrome `beforeinstallprompt` генерируется браузером по собственным критериям вовлечённости и может не прийти — в этом случае оверлей не показывается (деградация без поломки UI).
- UA-детект iOS приблизителен; ошибка детекта приводит максимум к показу/непоказу инструкции, не к поломке функционала.
- Состояние установки хранится локально на устройстве — на новом устройстве предложение появится снова.
- Физическая зачистка ключа `yc_install_seen` из `localStorage` существующих пользователей отложена до следующего касания.
