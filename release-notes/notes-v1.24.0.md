# Release Notes v1.24.0

- **Дата:** 2026-05-30
- **Тип релиза:** minor

## Кратко

Экран новой push-рассылки стал честным: декоративные элементы выбора аудитории и расписания удалены, остался единственный реально работающий сценарий — «Отправить всем сейчас». Администратор больше не вводится в заблуждение относительно адресата и времени отправки.

## Что нового

### Возможности

- Экран `s-admin-push-new` теперь содержит только фактически работающие элементы: шаблон, текст уведомления, предпросмотр и кнопку «Отправить всем сейчас».
- После отправки администратор видит тост с фактическим числом получателей из `res.sent`.

### Улучшения

- Кнопка отправки `#pushNewSendBtn` всегда показывает «Отправить всем сейчас» — текст фиксирован и не меняется программно.
- «Очистить всё» в истории рассылок очищает всю историю (включая ранее сохранённые `scheduled`-записи), а не оставляет «невидимый мусор».
- Блок «Запланированные» в истории рассылок больше не появляется даже при наличии старых `scheduled`-кампаний в `localStorage` — толерантное чтение без принудительной миграции.
- Описание экрана `s-admin-push-new` в `reference.html` обновлено и соответствует фактическому UI.
- Service Worker обновлён до `CACHE_VERSION = 'v79'` для принудительного подхвата обновлённого `prototype.html`.

### Исправления

- Удалена ссылка на `updatePushAudience` из `app.js` — при входе на экран рассылки в консоли браузера больше не возникает ошибки «функция не найдена».
- Удалён мёртвый JS-код: функции `selectNewAudience`, `pushSchedToggle`, `updatePushAudience`, `_setPushCounts`, переменная `_pushAudienceData`, ветка `isScheduled` в `sendNewPush`, секция «Запланированные» в `renderAdminPush`.

## Изменения публичного контракта

- **Worker API (`cf-worker`):** не изменяется. Контракт `handleSend` (broadcast по `sub:` с фильтром `promo`) остаётся прежним.
- **Формат подписок / VAPID / `prefs`:** не изменяется.
- **`localStorage` `yc_push_campaigns`:** формат совместим. Новые записи не содержат `scheduled:true`. Старые записи со `scheduled:true` игнорируются при рендере; принудительная миграция не выполняется.
- **UI экрана `s-admin-push-new`:** удалены элементы `#naud1`, `#naud2`, `#naud3`, `#segNow`, `#segLater`, `#schedTimeRow` и секции «Аудитория», «Когда отправить». Миграционный путь не требуется — удалённые элементы были декоративными и не влияли на результат отправки.

## Известные ограничения

- Реальная сегментация аудитории (передача списка `clientId` в Worker) и серверное планирование рассылок не реализованы. Это осознанно отложенный «Путь A» (зафиксирован ADR-006 и ADR-026) — адресуется отдельным релизом при появлении бизнес-потребности.
- Поведение при незаполненных настройках Worker (URL/Secret): отправка завершается тостом об ошибке — поведение сохранено без изменений (ОВ-5 BA-029, некритичный).
