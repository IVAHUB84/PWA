# Release Notes v1.13.2

- **Дата:** 2026-05-29
- **Тип релиза:** patch (доработки инбокса уведомлений)

## Кратко

Продолжение доработок по замечаниям к инбоксу: перерисована иконка-веник очистки, на каждой плашке уведомления добавлено меню «три точки» с действиями «Удалить» и «Сделать непрочитанным».

## Что изменено

### UI

- **Новая иконка-веник** в шапке `s-inbox` (`prototype.html`) — переработан SVG (ручка + трапециевидная метла с бороздками-щетинками) вместо прежнего нечитаемого контура.
- **Меню действий на плашке** — кнопка «три точки» (⋯) в правом верхнем углу каждого уведомления (`inbox.js`, `style.css`). По тапу открывается всплывающее меню:
  - **Сделать непрочитанным** — помечает уведомление непрочитанным; на плашке появляется красная точка-индикатор, бейдж на колокольчике обновляется.
  - **Удалить** — удаляет уведомление из истории.
  - Меню закрывается по клику вне его.

### Логика

- В хранилище инбокса (`inboxStore.js`) добавлены операции `deleteNotification(id)` и `setRead(id, read)` — обе работают только в контексте окна (UI). Схема IndexedDB `studio-inbox` (имя/версия/стор/keyPath/формат id) **не менялась**; инлайн-копия записи в `sw.js` не затронута.
- Порядок открытия инбокса изменён: сначала пометка всех прочитанными, затем рендер — непрочитанные точки-индикаторы больше не мелькают при открытии и появляются только после явного «Сделать непрочитанным».

### Служебное

- `CACHE_VERSION` поднята `v61 → v62` (изменены `prototype.html`, `style.css`, `inbox.js`, `inboxStore.js`).
- `docs/version.json`: build 28 → 29, version v1.13.2.

## Поведение «сделать непрочитанным»

Действие помечает уведомление непрочитанным и показывает индикатор + бейдж сразу. При следующем открытии инбокса все уведомления снова помечаются прочитанными (поведение «открытие инбокса = всё прочитано», КП-5 базового релиза не менялось).

## Изменения публичного контракта

Нет. Доставка push, CF Worker, схема подписок (ADR-006), формат хранилища IndexedDB — без изменений.
