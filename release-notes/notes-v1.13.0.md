# Release Notes v1.13.0

- **Дата:** 2026-05-29
- **Тип релиза:** minor

## Кратко

Релиз добавляет инбокс push-уведомлений: колокольчик в шапке главного экрана открывает список всех полученных уведомлений — включая пришедшие при закрытом приложении. Бейдж-счётчик непрочитанных обнуляется при просмотре инбокса; история очищается одной кнопкой с подтверждением.

## Что нового

### Возможности

- **Колокольчик на главном экране `s-home`** — кнопка в правом верхнем углу шапки с aria-меткой «Уведомления». Строка поиска `#homeSearchBar` не затронута.
- **Инбокс (`s-inbox`)** — отдельный экран со списком всех полученных push-уведомлений, отсортированных «новые сверху»; каждый элемент показывает заголовок, текст и дату/время.
- **Сохранение истории через IndexedDB** — запись происходит в SW-обработчике `push`, поэтому уведомления, полученные при закрытом приложении, также присутствуют в инбоксе после открытия.
- **Бейдж непрочитанных** — кружок-счётчик справа внизу колокольчика; при более 9 непрочитанных отображается «9+»; скрыт при нуле.
- **Пометка прочитанными** — при открытии инбокса все непрочитанные помечаются прочитанными, бейдж обнуляется.
- **Очистка истории** — кнопка «Очистить историю» с подтверждением; после очистки — пустое состояние «Уведомлений пока нет».
- **Оффлайн-просмотр** — история читается из локального IndexedDB без обращения к сети.
- **Мгновенное обновление бейджа** — при открытом приложении SW отправляет `postMessage` при получении push, клиент обновляет бейдж без перезагрузки.

### Улучшения

- `CACHE_VERSION` поднята `v59 → v60`; новые модули `inboxStore.js` и `inbox.js` добавлены в `APP_SHELL` — PWA остаётся устанавливаемой, оффлайн не регрессирует.
- `docs/version.json`: build 26 → 27, version v1.13.0.

### Исправления

Регрессий не выявлено. Существующий поток push (`showNotification`, `notificationclick`, тост `#pushNotif`, подписка/отписка) не изменён.

## Изменения публичного контракта

- **Внешний API / CF Worker / YCLIENTS / схема подписок ADR-006:** без изменений.
- **Новое клиентское хранилище:** IndexedDB `studio-inbox` (БД v1, store `notifications`, keyPath `id`). Локальное хранилище на устройстве клиента; миграция не требуется — при первом запуске пустое.
- **Навигация:** добавлен экран `s-inbox` в `SCREENS` (аддитивно); сигнатуры `go`/`back`/`registerOnEnter` не менялись.
- **SW → клиент:** добавлено `postMessage { type: 'PUSH_RECEIVED', ts }` при получении push. Клиенты без слушателя игнорируют.

## Известные ограничения

- Тап по элементу инбокса информационный — deep-link/действие не реализованы (вне объёма этого релиза).
- Удаление отдельного уведомления — вне рамок; только полная очистка истории.
- Синхронизация значка приложения через Badging API (`navigator.setAppBadge`) — не реализована; бейдж только визуальный внутри PWA.
- История ведётся локально на устройстве; синхронизация между устройствами — вне объёма.
- Лимит хранилища — 50 уведомлений; при превышении вытесняются самые старые.
