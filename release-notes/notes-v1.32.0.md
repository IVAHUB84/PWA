# Release Notes v1.32.0

- **Дата:** 2026-05-31
- **Тип релиза:** minor

## Кратко

Все нативные `alert/confirm` в клиентских сценариях (бронирование, отмена, перенос, отзывы) заменены на согласованные in-app примитивы: toast-уведомления и bottom-sheet диалоги. При сетевой ошибке пользователь видит диалог с кнопками «Повторить» и «Позвонить в студию» — без потери контекста. Добавлен ненавязчивый баннер офлайна/устаревших данных.

## Что нового

### Возможности
- **Toast-уведомления** трёх типов (`success` / `error` / `info`): появляются поверх контента, не блокируют интерфейс, авто-скрываются (success/info — 3 с, error — 5 с); одиночный с вытеснением; `aria-live="polite"`.
- **Bottom-sheet диалог ошибки** (`errorSheet`): кнопки «Повторить» (повторяет действие без потери контекста), «Позвонить в студию» (`tel:`, только если номер известен), «Закрыть»; закрытие кликом по фону.
- **Bottom-sheet подтверждения** (`confirmSheet`): замена клиентских `confirm()`, закрытие по фону и кнопке отмены, резолвится в `Promise<boolean>`.
- **Офлайн-баннер**: `setOfflineBanner('offline')` при `!navigator.onLine` и по событию `offline`; скрывается при `online`.
- **Индикатор устаревших данных**: `setOfflineBanner('stale')` при успешной гидрации из снимка и последующем неуспехе фонового revalidate каталога.

### Улучшения
- Переиспользуемые примитивы UI в `docs/modules/ui.js` — готовы к расширению в v1.33 (Esc-закрытие, a11y) без изменения публичных сигнатур.
- Чистая логика тайминга toast вынесена в `docs/modules/uiQueue.js` (без DOM, таймер инжектируется) — покрыта unit-тестами.
- Весь вставляемый текст (включая `meta.message` от YCLIENTS API) экранируется через `esc()`.

### Исправления
- Устранено «тупиковое» состояние при сетевой ошибке брони/отмены/переноса: пользователь может повторить действие или позвонить в студию, не теряя выбранных данных.
- Заменены нативные `alert` в `booking.js` (5 мест) и `review.js` (3 места).

## Изменения публичного контракта

Внешний API YCLIENTS, CLI, схема данных — без изменений. Меняется только воспринимаемый UX: нативные браузерные модалки заменены in-app примитивами. Миграция не требуется. Откат — `git revert` коммита релиза (CACHE_VERSION откатится вместе с `sw.js`).

Новый публичный контракт примитивов (стабилен для v1.33/v1.34):
- `toast(message, type='info', opts={})` — возвращает `void`.
- `confirmSheet({title, message?, confirmText, cancelText='Отмена', danger=false})` → `Promise<boolean>`.
- `errorSheet({title, message, onRetry, phone?})` → `void`.
- `setOfflineBanner(mode)` — `mode ∈ {'offline','stale',null}`.

## Известные ограничения

- Esc-закрытие sheet и фокус-ловушка не реализованы — отложены в v1.33.0 (a11y).
- `admin.js` и `github.js` (`prompt()`) остаются на нативных модалках — это административный слой, вне объёма клиентского UX.
- При одновременном быстром возникновении нескольких событий новый toast вытесняет предыдущий — приемлемо для клиентских сценариев (подтверждения единичны).
