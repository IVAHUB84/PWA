# Release Notes v1.7.0

- **Дата:** 2026-05-28
- **Тип релиза:** minor

## Кратко

В потоке онлайн-записи путь до брони стал короче на один экран: тап по мастеру в списке ведёт сразу на «Выберите время», минуя промежуточную карточку-профиль. Профиль мастера с отзывами сохранён для просмотра вне записи — из «Все мастера студии», главного экрана и поиска.

## Что нового

### Возможности
- В потоке записи (услуга уже выбрана) тап по мастеру в списке `s-masters` открывает сразу экран «Выберите время» (`s-slots`) — промежуточный экран-профиль `s-master` в этом потоке не показывается.
- Поведение «конкретный мастер» теперь согласовано с «Любой мастер»: оба ведут прямо на выбор слота.

### Улучшения
- Контекстный диспетчер `tapMaster(id)` — единая точка принятия решения о действии при тапе. В потоке записи вызывает `selectMaster(id)` (прямой переход на `s-slots`), вне записи — `openMasterCard(id)` (профиль с отзывами).
- Признак режима `_masterTapMode` выставляется синхронно при каждой отрисовке `renderMasters`, без залипания между открытиями в разных режимах.

### Исправления
- Нет регрессий: контракты `selectMaster`/`openMasterCard`/`selectAnyMaster` не изменены; `profile.js` и `search.js` не затронуты; поток записи к YCLIENTS (`book_staff`, `book_dates`, `book_times`, `book_record`) не изменился.

## Изменения публичного контракта

Внешних API-контрактов (YCLIENTS, CLI, схемы БД) не меняется. Изменяется только UX-поведение: в потоке записи тап по мастеру больше не открывает `s-master`. Карточка-профиль `s-master` и все связанные функции (`openMasterCard`, `bookFromMaster`, `bookServiceFromMaster`) сохранены и доступны вне потока записи.

Миграция данных не требуется.

## Известные ограничения

- Функция `tapMaster` принимает решение в рантайме по признаку `_masterTapMode`, выставленному при последнем вызове `renderMasters`. При крайне быстром переходе «Все мастера» → «от услуги» (до завершения загрузки и рендера списка) может сработать режим 'browse'. Команда архитектора/ревьюера зафиксировала этот edge-кейс как теоретический; `_masterTapMode` начинается с безопасного дефолта 'browse' (тап ведёт в профиль, не на прямой переход).
- Точки входа `profile.js` и `search.js` продолжают звать `openMasterCard` напрямую (корректно — там услуга не выбрана). Унификация на `tapMaster` отложена как технический долг.
