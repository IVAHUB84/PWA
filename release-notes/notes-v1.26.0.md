# Release Notes v1.26.0

- **Дата:** 2026-05-30
- **Тип релиза:** minor

## Кратко

Релиз устраняет две проблемы режима студии: случайный выход из режима при навигации по вкладкам и фейковую метрику «4.8 ★» в сводке, которая снижала доверие к реальным данным. Выход перенесён в шапку с подтверждением, рейтинг заменён честным «Средним чеком сегодня».

## Что нового

### Возможности
- Кнопка выхода из режима студии (иконка двери) добавлена в левую часть шапки на всех четырёх экранах (`s-admin`, `s-admin-feed`, `s-admin-clients`, `s-admin-push`). Нажатие показывает `confirm`, при подтверждении выполняется `go('s-login')` без сброса клиентской сессии.

### Улучшения
- Карточка «Средний чек сегодня» на экране «Сводка»: значение `rev / recs.length` из уже загруженных данных, при нуле записей — «—», без деления на ноль.
- Tab-bar на всех четырёх админских экранах содержит ровно 4 вкладки (Сводка / Лента / Клиенты / Рассылка). Вкладка «Выйти» удалена.

### Исправления
- Удалены пустые элементы `stat-diff` из карточек статистики сводки.
- Удалён осиротевший CSS-класс `.stat-diff` (применялся только к удалённым элементам).
- Поднята `CACHE_VERSION` до `v81` (изменены `prototype.html` и `modules/admin.js`, входящие в `APP_SHELL`).

## Изменения публичного контракта

Клиентский (не-админский) контракт PWA не изменился.

Внутренний UI-контракт режима студии:
- Tab-bar становится 4-вкладочным на всех 4 экранах.
- Добавлен глобальный обработчик `window._adminLogout()`.
- Действие выхода переезжает из tab-bar в шапку. Поведение выхода (`go('s-login')` без сброса сессии) сохраняет прежний эффект пятой вкладки.

Миграция не требуется — отдельной админ-сессии в хранилище нет.

## Известные ограничения

- Браузерный `confirm()` для подтверждения выхода не кастомизируется под дизайн-систему PWA (осознанный компромисс — `confirm` является принятым в проекте паттерном подтверждения).
- «Средний чек» — производная метрика из уже загруженных данных, а не «средняя оценка» (реального агрегата отзывов в PWA нет).
- Tab-bar остаётся продублированным в разметке 4 раза (вынос в шаблон/функцию — отдельная задача).

## Проверка

**Lint:** `npm run lint` — 0 ошибок, 0 предупреждений.

**Критерии приёмки (проверено по коду и диффу, браузерный прогон недоступен в этой среде):**

- КП-1 (4 вкладки, нет «Выйти»): подтверждён. Все 4 tab-bar содержат ровно 4 `.tab` (строки 703–708, 721–726, 778–783, 806–811); `go('s-login','tab')` в tab-bar отсутствует.
- КП-2 (переключение и подсветка вкладок): подтверждён по коду. `onclick` и `class="tab active"` на соответствующей вкладке каждого экрана сохранены; логика `go()` не затронута.
- КП-3 (кнопка выхода в шапке, не перекрывает правые кнопки): подтверждён. Кнопка расположена в левом flex-блоке; `_adminResetToken`, «+ Новый», «+ Новая» — в правой части (строки 691, 715–716, 773, 789–796).
- КП-4 (confirm при нажатии выхода): подтверждён. `_adminLogout()` в `admin.js:539–541` содержит `if (confirm('Выйти из режима студии?')) go('s-login');`.
- КП-5 (_adminLogout без clearSession/logout): подтверждён. В теле функции вызывается только `go('s-login')`; `clearSession()` и `logout()` отсутствуют.
- КП-6 (нет «4.8 ★» / «Средняя оценка» в HTML, карточка «Средний чек сегодня», сетка 4 карточки): подтверждён. HTML строки 694–697: 4 карточки, `id="adminStatAvg"` со значением «—», подпись «Средний чек сегодня», «4.8» и «Средняя оценка» в разметке сводки отсутствуют.
- КП-7 (расчёт среднего чека, «—» при нуле, без деления на ноль): подтверждён. `admin.js:143`: `recs.length > 0 ? Math.round(rev / recs.length).toLocaleString('ru-RU') + ' ₽' : '—'`; деление выполняется только при `recs.length > 0`.
- КП-8 (нет пустых stat-diff): подтверждён. `grep stat-diff docs/prototype.html` — пусто; `grep stat-diff docs/style.css` — пусто.
- КП-9 (CACHE_VERSION, нет ошибок, PWA installable): CACHE_VERSION `v81` подтверждён (`sw.js:1`). Браузерный прогон (ошибки в консоли, Service Worker active, Lighthouse installable) — **не проверено в этой среде** (требует запуска локального сервера).
