# Release notes — v1.23.0

**Дата:** 2026-05-30
**Тип:** minor

## Кратко

Добавлен контактный блок студии: адрес, телефон, часы работы и кнопка «Маршрут» теперь показываются на экране подтверждения записи (`s-confirm`) и в профиле (`s-profile`). Данные берутся из YCLIENTS `/company`; при пустом или недоступном ответе отображаются плейсхолдеры.

## Что нового

### Возможности

- Контактный блок на `s-confirm`: под карточкой записи — название студии, адрес, кликабельный телефон (`tel:`), часы работы, кнопка «Маршрут» (Я.Карты) и ссылка 2ГИС.
- Секция «О студии» в `s-profile`: те же контакты, оформлены в стиле существующих секций профиля (`settings-group`).
- Гибридный источник данных: поля компании из YCLIENTS `/company/{id}` имеют приоритет; недостающие значения замещаются плейсхолдерами из нового объекта `STUDIO` в `constants.js`.
- `localStorage`-кэш (`yc_company`): при оффлайн-старте PWA показывает последний живой ответ API, а не только плейсхолдеры.

### Улучшения

- Телефон и адрес в контактном блоке теперь можно выделить и скопировать: добавлено локальное исключение `user-select:text` для класса `.studio-contacts` поверх глобального `body { user-select:none }`.
- Запрос `GET /company` встроен в общий `Promise.all` на старте (`initApp()`), не замедляет критический путь — при ошибке даёт `{}` и блок отображает плейсхолдеры.

### Исправления

- Нет регрессий в существующих кнопках «На главную» и «Добавить услугу к записи» на `s-confirm`.

## Изменения публичного контракта

- Новый ключ `localStorage`: `yc_company` (публичные контактные поля, без секретов). Обратимо-совместимо: отсутствие ключа корректно обрабатывается.
- Новый файл в `APP_SHELL`: `docs/modules/studio.js`. Миграция кэша через подъём `CACHE_VERSION` v76 → v77.
- Внешний API YCLIENTS, CLI, схема БД — не меняются.

## Известные ограничения

- Телефон в плейсхолдере (`+7 495 123-45-67`) — ненастоящий номер; помечен в коде комментарием `// ПЛЕЙСХОЛДЕР — заменить на реальный телефон`. Заменяется при заполнении поля `phone` в админке YCLIENTS — без правки кода.
- У компании `1940850` все контактные поля в YCLIENTS сейчас пусты, поэтому отображаются плейсхолдеры. Данные «оживут» автоматически при заполнении полей в админке.
- `PUT /company/{id}` partner-токеном недоступен (`success:false`) — запись настроек из приложения вне scope.

## Проверка

**Статическая проверка критериев приёмки по коду:**

- КП-1 (контактный блок на `s-confirm`): контейнер `#confirmContacts` в HTML, заполняется в `updateConfirmScreen()` через `renderStudioContacts({variant:'confirm'})`. Включает название, адрес, телефон, часы, кнопку «Маршрут». Кнопки «На главную» и «Добавить услугу» не затронуты. — **Проверено статически.**
- КП-2 (секция «О студии» в `s-profile`): контейнер `#profileStudioContacts` в HTML, заполняется в `renderProfileScreen()` через `renderStudioContacts({variant:'profile'})`; обёртка `settings-group`. — **Проверено статически.**
- КП-3 (tel:): телефон рендерится как `<a href="tel:...">`, `href` нормализован до `+` и цифр функцией `_telHref`. — **Проверено статически.**
- КП-4 (ссылка карты + rel=noopener): кнопка «Маршрут» — `href` Я.Карты, `target="_blank" rel="noopener"`; 2ГИС аналогично. — **Проверено статически.**
- КП-5 (user-select): `.studio-contacts` в `style.css` объявляет `user-select:text` + вендорные префиксы + `-webkit-touch-callout:default`; класс назначается обёртке в `studio.js`. — **Проверено статически.**
- КП-6 (приоритет API): `getCompanyContacts()` использует цепочку свежий API → localStorage → STUDIO; непустое = `trim()` не пустое, координаты — оба ≠ 0. — **Проверено статически.**
- КП-7 (деградация при недоступном API): запрос `.catch(() => ({}))`, `setCompanyData({})` не записывает данные, `getCompanyContacts()` возвращает плейсхолдеры. — **Проверено статически.**
- КП-8 (плейсхолдер-комментарий): `STUDIO.phone` в `constants.js` сопровождается `// ПЛЕЙСХОЛДЕР — заменить на реальный телефон`. — **Проверено статически.**
- КП-9 (экранирование + noopener): все динамические строки через `esc()`; внешние ссылки — `rel="noopener"`; новых секретов нет. — **Проверено статически.**
- КП-10 (CACHE_VERSION, lint, APP_SHELL): `CACHE_VERSION` = `v77` (поднята разработчиком); `studio.js` присутствует в `APP_SHELL`; `npm run lint` чист. — **Проверено статически.**

**Требует проверки в браузере (нет в этой среде):** визуальный вид блоков на `s-confirm` и `s-profile`, реакция на тап-по-телефону, открытие карты, выделение текста на мобильном, консоль DevTools без ошибок, PWA installable по Lighthouse.
