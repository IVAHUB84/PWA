# Release Notes v1.9.0

- **Дата:** 2026-05-29
- **Тип релиза:** minor

## Кратко

Тап по конкретной плашке превью «Ленты студии» на главном экране теперь открывает именно эту публикацию в отдельном экране `s-post`, а не общий список. Ссылка «Все →» сохраняет прежнее поведение. При отсутствии публикации по `id` — мягкий фолбэк на `s-feed` без ошибок.

## Что нового

### Возможности
- Новый экран `s-post` («Публикация»): показывает одну выбранную публикацию — фото/иконка категории, полный текст, тег категории, дата, кнопка «Записаться →».
- Адресация публикации по устойчивому `p.id` (а не по позиции в списке): при изменении состава/порядка постов тап открывает именно выбранный пост.
- Диспетчер `_openFeedPost(id)`: ищет публикацию среди `!p.draft`, пишет `state._openPostId` и открывает `s-post`; при отсутствии — `go('s-feed')` (мягкий фолбэк из `s-home`).
- Страховочный onEnter-фолбэк в `renderPostScreen`: если пост исчез между тапом и входом на экран — `go('s-feed', 'replace')` замещает `s-post` в истории, не создавая петли «Назад».

### Улучшения
- `_feedCardHtml(p)` — общий хелпер рендера карточки публикации в `feed.js`; `_paintFeed` и `renderPostScreen` используют его. Разметка карточки теперь в одном месте, дублирования нет.
- Режим `'replace'` в `go(id, mode)` (`navigation.js`): заменяет верхний элемент `navHistory` вместо `push`; анимация идентична обычному push. Существующие режимы push и `'tab'` не затронуты.

### Исправления
- Плашки превью ленты на главной ранее вели на `go('s-feed')` вместо конкретного поста — устранено.

## Изменения публичного контракта

- **Реестр экранов `SCREENS`** расширен значением `'s-post'` (аддитивно; существующие экраны не затронуты).
- **Контракт навигации `go(id, mode)`** расширен аддитивным режимом `mode === 'replace'` (внутренний контракт навигации, не публичный API). Сигнатура не меняется; существующие вызовы `go(id)` и `go(id, 'tab')` работают как прежде. Миграционный путь не требуется.
- **Поведенческое изменение:** целевое действие тапа по плашке превью ленты изменено с `go('s-feed')` на `_openFeedPost(id)` — намеренно, согласовано с BA и ADR-010.
- Внешний API YCLIENTS, CLI, схема БД, формат `yc_feed_posts` — не изменены.

## Известные ограничения

- `docs/version.json` (`build`, `date`) обновляется мейнтейнером при деплое — не входит в зону тестировщика (CLAUDE.md, ADR-010).
- Браузерные сценарии Т-1…Т-9 (кроме Т-10) требуют ручной проверки в DevTools — автотестов в проекте нет.
- Тест Т-7 (страховочный onEnter-фолбэк, возврат «Назад» → `s-home` без петли) и Т-9 (активная SW `v54`, Lighthouse installable, офлайн) требуют проверки в браузере мейнтейнером перед деплоем.
