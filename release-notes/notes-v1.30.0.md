# Release Notes v1.30.0

- **Дата:** 2026-05-31
- **Тип релиза:** minor

## Кратко

Экран «Выберите время» (`s-slots`) доработан под удобную работу на мобильном: карточка календаря зафиксирована и больше не уезжает при прокрутке слотов, добавлен компактный недельный вид и drag-handle для переключения жестом. Авто-выбор первой даты и слота убран — кнопка «Записаться» появляется только после того, как клиент осознанно выбрал и дату, и время.

## Что нового

### Возможности
- Фиксированная карточка календаря: `.cal-card` вынесена из прокручиваемой области в отдельную зону `.slots-cal-zone`; скроллится только список слотов.
- Два режима календаря: развёрнутый (полный месяц, по умолчанию при входе) и свёрнутый (одна строка-неделя с выбранной датой или текущей неделей).
- Drag-handle («черточка») между карточкой и списком слотов — визуальный намёк и кликабельный переключатель режима.

### Улучшения
- Переключение режимов свайпом по карточке: вверх — свернуть, вниз — развернуть; порог 30 пкс/400 мс/угол исключает срабатывание на тапах по ячейкам.
- Клик по drag-handle и нажатие Enter/Space (доступность) переключают режим как альтернатива свайпу.
- В свёрнутом виде показывается неделя выбранной даты с тёмным кружком; если дата не выбрана — текущая неделя с выделением «сегодня».
- Анимация перехода между режимами через CSS `max-height` + `transition` на `.cal-grid` (по аналогии с секциями периодов суток).
- Стрелки «‹ ›» в свёрнутом виде остаются рабочими и меняют месяц как прежде.

### Исправления
- Убран авто-выбор первой доступной даты (`_autoSelectFirstDate`) и авто-назначение первого слота в `loadTimes`; `state.dateISO` и `state.slot` при входе пусты.
- Кнопка «Записаться» скрыта (`display:none` через класс `.hidden`) до тех пор, пока не выбраны и дата, и слот; `updateStickyBottom` — единственная точка управления видимостью блока.
- Заголовок «Выберите время» (`.page-title`) удалён из `#s-slots` — освобождает вертикальное пространство.
- Якорь баннера переноса записи перепривязан к `.slots-cal-zone` (`insertAdjacentElement('beforebegin')`) взамен удалённого `.page-title`.

## Изменения публичного контракта

Внешний контракт не изменился: эндпоинты YCLIENTS (`/book_dates`, `/book_times`, `/book_record`), поток `startBooking` → `s-confirm`, структура `state` (`slot` / `dateISO` / `dateFull` / `_rescheduleId`) сохраняются.

Внутреннее изменение поведения UI: при входе на `s-slots` дата и слот больше не преселектятся; кнопка действия видима только после полного выбора. Миграция не требуется — статический фронтенд; кэш обновляется автоматически через поднятый `CACHE_VERSION`.

## Известные ограничения

- Листание по неделям в свёрнутом виде не реализовано: стрелки меняют месяц, а не неделю. Отложено до отдельного требования.
- Состояние (свёрнут/развёрнут) не сохраняется между заходами на экран — при входе всегда развёрнуто.
- Для недель, пересекающих границу месяца, доступность дней соседнего месяца загружается в фоне; до загрузки соседние дни показываются как недоступные.

## Проверка

**Линтер:** `npm run lint` — пройден без ошибок (ESLint, `docs/modules/` + `docs/app.js`).

**Автотесты:** в проекте отсутствуют; критерии приёмки проверены статически по коду и разметке.

**Критерии приёмки BA-035:**

- КП-1 (фиксация календаря): `.slots-cal-zone` имеет `flex-shrink:0`; `.scroll.slots-scroll` — собственный `overflow-y:auto`; `.cal-card` не входит в `.slots-scroll` — структура DOM соответствует. **Подтверждён по разметке и CSS.**
- КП-2 (вход — развёрнутый месяц): `loadDates()` устанавливает `_calCollapsed = false`; `_renderCalGrid` вызывает `_renderMonthGrid` при `_calCollapsed === false`. **Подтверждён по коду.**
- КП-3 (свайп вверх/вниз): обработчики `touchstart`/`touchend` на `.slots-cal-zone`; порог `absDy < 30 || dt > 400 || absDy <= absDx * 1.5`; `dy < 0 && !_calCollapsed → _calCollapsed = true`, `dy > 0 && _calCollapsed → _calCollapsed = false`. **Подтверждён по коду.** Фактическая работа на устройстве **требует ручной проверки в браузере.**
- КП-4 (свёрнутый вид — неделя выбранной/текущей даты): `_renderWeekRow` — `anchorISO = state.dateISO || today`; `_getWeekStart(anchorISO)` → 7 ячеек Пн–Вс; `sel` при совпадении с `state.dateISO`, `today` при отсутствии выбора. **Подтверждён по коду.**
- КП-5 (drag-handle виден): `<div class="cal-drag-handle-bar">` — 36×4 px, цвет `var(--border)`, переход на `var(--text-2)` при hover/focus; расположен в `.slots-cal-zone` между `.cal-card` и `.slots-scroll`. **Подтверждён по разметке и CSS.**
- КП-6 (клик по handle переключает режим): `onclick="toggleCalCollapsed()"` на `#calDragHandle`; `keydown` Enter/Space — `_initHandleKeyboard()`; `role="button"` + `tabindex="0"` + `aria-label`. **Подтверждён по разметке и коду.**
- КП-7 (нет авто-выбора при входе): `_autoSelectFirstDate` и её вызовы отсутствуют в `slots.js`; в `loadDates` `state.dateISO = ''`, `state.slot = ''`; авто-назначение `firstTime` в `loadTimes` отсутствует. **Подтверждён по коду (grep — совпадений нет).**
- КП-8 (кнопка появляется только после выбора даты и слота): `updateStickyBottom` — `state.dateISO && state.slot` → `bottom.classList.remove('hidden')`, иначе `bottom.classList.add('hidden')`; в `#slotsStickyBottom` инициальный класс `hidden`; CSS `.sticky-bottom.hidden { display:none }`. **Подтверждён по разметке, коду и CSS.**
- КП-9 (кнопка бронирует через startBooking, перенос — как в ADR-031): `onclick="startBooking()"` сохранён; `updateStickyBottom` устанавливает текст «Перенести запись» при `state._rescheduleId`. **Подтверждён по разметке и коду.** Сквозной сценарий **требует ручной проверки в браузере.**
- КП-10 (заголовок «Выберите время» отсутствует): в `#s-slots` нет `.page-title`; grep по `prototype.html` — совпадения только на других экранах. **Подтверждён по разметке.**
- КП-11 (свайп не конфликтует со скроллом и выбором): обработчик `touchend` проверяет `e.target.closest('.cal-cell')` и возвращает управление при тапе; `.slots-scroll` — отдельный контейнер, не вложен в `.slots-cal-zone`. **Подтверждён по коду и структуре DOM.**
- КП-12 (группировка слотов и фолбэки без регрессии): `_PERIODS`, `buckets`, `toggleSlotPeriod` не изменялись; фолбэки ошибки сети и «нет окон» сохранены в `loadTimes`. **Подтверждён по коду.**
- КП-13 (переход плавный): CSS `.cal-card .cal-grid { max-height: 500px; transition: max-height 0.25s ease, opacity 0.2s }`; `.cal-card.cal-collapsed .cal-grid { max-height: 44px }`. **Подтверждён по CSS.**
- КП-14 (тёмная тема, hapticTap, санитизация, CACHE_VERSION): все новые стили используют `var(--border)`, `var(--primary)`, `var(--text-2)` — токены темы; `hapticTap('select')` вызывается в `toggleCalCollapsed`, `calSelectDate`, `selectSlot`; время санитизируется `replace(/[^0-9:]/g,'')` до вставки в `onclick`; текст через `esc()`; `CACHE_VERSION` = `v87`; `version.json` = `v1.30.0 build 50`. **Подтверждён по коду, CSS и конфиг-файлам.** Ноль ошибок в консоли и Lighthouse installable **требуют ручной проверки в браузере.**

Итог: 14/14 критериев подтверждены статически; КП-3 (тач-свайп), КП-9 (сквозное бронирование), КП-14 (консоль/Lighthouse) дополнительно требуют ручной проверки в браузере.
