# Release Notes v1.21.0

- **Дата:** 2026-05-30
- **Тип релиза:** minor

## Кратко

Релиз выводит записи клиента на верхний уровень навигации: в нижнем таб-баре появляется вкладка «Записи» (→ экран истории), а дублирующая точка входа «Услуги» удаляется. Единственный вход в каталог услуг — FAB «Запись», кнопка «Записаться» на главной и блок мастеров.

## Что нового

### Возможности

- **Вкладка «Записи» в таб-баре.** Новая 4-я позиция (Главная · Лента · [+] · Записи · Профиль) — прямой доступ к истории и предстоящим визитам из любого клиентского экрана.

### Улучшения

- **Активное состояние «Записи».** На экранах `s-history` и `s-upcoming` вкладка «Записи» подсвечивается как активная. На `s-services`/`s-masters`/`s-slots` активной вкладки нет (каталог — транзитный экран, не точка назначения).
- **Переименование заголовка s-history.** Заголовок экрана изменён с «История» на «Записи» — соответствует подписи в таб-баре.
- **CACHE_VERSION** поднята с v73 до v74: пользователи получают обновлённый `prototype.html` из новой версии кэша.

### Исправления

- Удалена вкладка «Услуги» из таб-бара — устранено дублирование точки входа в каталог.

## Изменения публичного контракта

Внешнего API, CLI и схем БД нет. Навигационное изменение UI: вкладка «Услуги» в таб-баре удалена во всех 9 клиентских копиях. Вход в `s-services` остаётся доступен через FAB «Запись», кнопку «Записаться» на главной (`prototype.html:320`) и блок мастеров. Дополнительный вход из Профиля → «История посещений» (`go('s-history')`) сохранён. Миграционный путь не требуется.

## Известные ограничения

- КП-3, КП-10, КП-11 (тап по «Записи», регрессии фильтрации/действий, консоль / PWA installable) требуют браузера и живого YCLIENTS API — проверяются мейнтейнером после деплоя на GitHub Pages.
