# Release Notes v1.6.0

- **Дата:** 2026-05-28
- **Тип релиза:** minor

## Кратко

Клиент получает два реально работающих независимых тумблера в профиле: «Push-уведомления» (рекламные рассылки) и «Напоминания о визитах». Предпочтения хранятся на уровне клиента (применяются на всех его устройствах). Добавлено второе напоминание примерно за 2 часа до визита; из настроек профиля удалены устаревшие пункты «Публикация фото и видео» и «Публикация отзывов».

## Что нового

### Возможности

- **Два независимых тумблера push в профиле.** «Push-уведомления» управляет получением рекламных рассылок от студии; «Напоминания о визитах» — напоминаниями накануне и за ~2 ч до записи. Тумблеры работают независимо: можно отключить рассылки, оставив напоминания, и наоборот.
- **Предпочтения per-client.** Настройка сохраняется в Cloudflare Worker KV (`prefs:${clientId}`) и применяется ко всем устройствам клиента. При открытии профиля тумблеры загружают актуальное состояние с сервера.
- **Второе напоминание (soon).** Cron-джоб теперь запускается каждые 30 минут (`*/30 * * * *`) и отправляет напоминание «Сегодня в HH:MM: &lt;услуга&gt;. Скоро ждём вас!» для записей, начало которых попадает в окно \[now+90, now+150) минут. Первое напоминание (накануне в ~18:00) сохранено.
- **Новый Worker-эндпоинт `/preferences`.** `POST /preferences {clientId, promo, remind}` → `{ok:true}`; `GET /preferences?clientId=…` → `{promo, remind}`. Без `X-Admin-Secret` (клиентский эндпоинт).

### Улучшения

- Фильтрация рассылок по `promo`: клиенты с `promo: false` не получают административные рассылки через `/send` (broadcast и target). Кэш предпочтений на вызов исключает лишние обращения к KV при мультидевайс-клиентах.
- Часовой пояс студии UTC+3 (`STUDIO_TZ_OFFSET_MIN = 180`) учитывается при расчёте времени напоминаний. Записи хранятся в локальном времени студии — все сравнения выполняются корректно.
- Анти-дубли напоминаний: маркер `reminded:${recordId}:${kind}` с TTL 48 ч гарантирует одну доставку на запись и вид при любой частоте cron-запусков.
- При недоступности push (нет разрешения или поддержки браузером) тумблеры помечаются неактивными, показывается подсказка «Уведомления недоступны».
- Визуальный откат тумблера при ошибке сохранения (`savePreferences` вернул `false`).

### Исправления

- Дефолт «оба типа включены» для существующих подписчиков без записи `prefs:` в KV — миграция не требуется, доставка для действующих клиентов не прерывается.

## Изменения публичного контракта

**Новый эндпоинт Worker:**
- `POST /preferences` — тело `{clientId, promo, remind}`, ответ `{ok:true}`; при отсутствии `clientId` — `{error:'missing_fields'}` (400). Без `X-Admin-Secret`.
- `GET /preferences?clientId=…` — ответ `{promo, remind}`; при отсутствии `clientId` — 400.

**KV `SUBSCRIPTIONS`:** добавлены префиксы ключей `prefs:${clientId}` и `reminded:${recordId}:${kind}`. Префикс `sub:` (подписки) — без изменений.

**Cron-расписание Worker:** `0 15 * * *` → `*/30 * * * *` (внутреннее; применяется после деплоя Worker).

**PWA:** добавлены `getPreferences`/`savePreferences` на `window`. `subscribePush`/`unsubscribePush`, навигация, сценарий записи — без изменений.

**Миграционный путь:** отсутствует. Клиенты без записи `prefs:` получают дефолт `{promo:true, remind:true}` — поведение не меняется.

## Известные ограничения

- Независимость тумблеров (выключить рассылки, оставить напоминания) подтверждается только после деплоя на реальном устройстве — проверку без развёрнутого Worker провести невозможно.
- Состояние тумблеров между устройствами одного клиента синхронизируется через сервер; при отсутствии сети тумблеры показывают дефолт `{promo:true, remind:true}`.
- Реальное получение second-reminder (~за 2 ч) проверяется только после деплоя Worker с новым cron-расписанием.

## Порядок деплоя для мейнтейнера

1. **Worker** (сначала — новый `/preferences` должен ответить до обновления клиентов):
   ```
   cd cf-worker && npx wrangler deploy
   ```
   Требует `CLOUDFLARE_API_TOKEN`. Новое cron-расписание `*/30` применится только после деплоя Worker.

2. **PWA:**
   ```
   git push origin main
   ```
   GitHub Pages публикует `docs/` (~1 мин). Новый SW `v41` обновит клиентов.

3. **Проверка на реальном устройстве:** открыть профиль → «Настройки уведомлений» → убедиться, что видны два тумблера без «Фото/Отзывов». Выключить «Push-уведомления», отправить рассылку — не должна прийти. Напоминания оставить вкл — cron-напоминание должно прийти. Переоткрыть приложение — состояние сохранено. На втором устройстве клиента — то же состояние. DevTools → Application → Service Workers: SW `v41` активен.
