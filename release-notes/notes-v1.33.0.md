# Release Notes v1.33.0

- **Дата:** 2026-05-31
- **Тип релиза:** minor

## Кратко

Релиз закрывает технический долг по доступности, накопленный с v1.32.0 (ADR-034). Основной клиентский поток «логин → услуга → мастер → слот → бронь» теперь полностью проходим с клавиатуры и понятен скринридеру; оверлеи и sheet закрываются по Esc; пользователи с вестибулярной чувствительностью могут отключить анимации через системную настройку.

## Что нового

### Возможности

- **Клавиатурная навигация.** Все критичные клиентские элементы (tab-bar, карточки услуг/мастеров, строки выбора категорий, ячейки слотов, drag-handle календаря, баннер установки) получили `role="button"` + `tabindex="0"` + `aria-label`. Единый делегированный keydown-обработчик на корне `#phone` активирует их через Enter/Space (Space не скроллит страницу).
- **Esc-закрытие оверлеев.** `confirmSheet` и `errorSheet` (`ui.js`) закрываются по Esc через приватные хелперы `_mountSheet`/`_dismissSheet`: слушатель снимается при закрытии, стек активных оверлеев обеспечивает закрытие только верхнего. Оверлей выбора категории (`services.js`) тоже получил Esc-обработку.
- **Фокус-ловушка sheet.** При открытии sheet фокус переходит на первую кнопку; Tab/Shift+Tab циклит внутри `.ui-sheet`; при закрытии фокус возвращается на элемент, активный до открытия.
- **Перенос фокуса при смене экрана.** `go()` в `navigation.js` переносит фокус на заголовок нового экрана (`.page-title` / `.section-title`, `tabindex="-1"`) или на контейнер экрана как фолбэк. Перенос не перебивает точечные `.focus()` в `auth.js`/`review.js`.
- **Видимый focus-ring.** Глобальные правила `:focus-visible` с токеном `--focus-ring` (корректен в светлой и тёмной теме); кольцо отсутствует при кликах мышью/тачем (`:focus:not(:focus-visible)`).
- **prefers-reduced-motion.** При системной настройке «уменьшить движение»: переходы экранов, карусель фото услуг и scale-анимации (`pop`, `tapPulse`, `inboxBadgePulse`) мгновенны. JS-гейт `prefersReducedMotion()` блокирует inline-переходы в `setTx` и карусели.

### Улучшения

- `aria-label` на иконочных кнопках клиентского потока: «Уведомления», «Свернуть или развернуть календарь», «Лента», «Запись», «Записи», «Профиль», «Добавить на экран» и др.
- Экспортированы чистые функции `focusTargetSelector(screenEl)` (`navigation.js`) и `prefersReducedMotion()` (`utils.js`) — тестируемы без DOM.

### Исправления

- Нет исправлений багов вне области a11y.

## Изменения публичного контракта

Публичные сигнатуры `ui.js` (`toast`, `confirmSheet`, `errorSheet`, `setOfflineBanner`) не изменены.

Новые экспорты (только добавление, не замена):
- `navigation.focusTargetSelector(screenEl) → HTMLElement | null`
- `utils.prefersReducedMotion() → boolean`

Миграционный путь не требуется.

## Известные ограничения

- Доступность **админского слоя** (`admin.js`, `github.js`) — вне объёма релиза (технический долг, зафиксирован в ADR-035).
- Полный аудит WCAG AA с формальной сертификацией — вне объёма.
- PWA работает только по HTTPS (GitHub Pages) или localhost — ограничение платформы.

## Проверка

**Автотесты (npm test):** 385 тестов, 0 упало. Включая `a11y.test.js` (10 тестов):
- `focusTargetSelector`: возвращает `.page-title` при наличии, `.section-title` как второй приоритет, контейнер как фолбэк, не вызывает `.focus()`, возвращает `null` при `null`-вводе.
- `prefersReducedMotion`: возвращает `false`/`true` по мок-результату `matchMedia.matches`; тип — boolean.

**ESLint:** чист (0 предупреждений).

**Критерии приёмки:**

- КП-1 (клавиатурная проходимость): требует ручной проверки в браузере — подтверждено по коду: все критичные элементы клиентского потока получили `role="button"` + `tabindex="0"` + `aria-label`; делегированный keydown-обработчик зарегистрирован в `navigation.js`.
- КП-2 (Esc): требует ручной проверки в браузере — подтверждено по коду: `_mountSheet`/`_dismissSheet` в `ui.js`, Esc-обработчик в `_openCatFilter` `services.js`.
- КП-3 (перенос фокуса): подтверждено unit-тестом `focusTargetSelector` (10 кейсов) + чтением кода `navigation.js:go()`.
- КП-4 (aria-label): подтверждено чтением кода — все иконочные кнопки клиентского потока имеют `aria-label`; DevTools Accessibility tree — требует ручной проверки.
- КП-5 (aria-live): требует ручной проверки скринридером — контейнер `aria-live="polite"` (`#uiToastHost`) присутствует в разметке (ADR-034, v1.32.0).
- КП-6 (reduced-motion): требует ручной проверки DevTools Rendering — подтверждено по коду: CSS-блок расширен (`transition:none` для `.screen` и `.svc-carousel-track`, `animation:none` для scale-keyframes); JS-гейт `prefersReducedMotion()` в `setTx` и карусели; `prefersReducedMotion` покрыт unit-тестом.
- КП-7 (focus-visible): требует ручной проверки в браузере — подтверждено по коду: глобальные правила `:focus-visible` и `:focus:not(:focus-visible)` в `style.css`, токен `--focus-ring` в обеих темах.
- КП-8 (без регрессий мышь/тач): требует ручной проверки в браузере.
- КП-9 (тех. целостность): `CACHE_VERSION === 'v90'` — подтверждено; lint чист; новых рантайм-зависимостей нет.
- КП-10 (без дублирования): подтверждено чтением кода — Esc/фокус-ловушка только в `ui.js` (`_mountSheet`/`_dismissSheet`), публичные сигнатуры ADR-034 не изменены.
