# Release Notes v1.3.0

- **Дата:** 2026-05-28
- **Тип релиза:** minor

## Кратко

Оценки и текстовые отзывы за завершённые визиты теперь отправляются напрямую в YCLIENTS через `POST /comments`, а не сохраняются только локально. Отзывы сразу становятся видны всем пользователям приложения на карточке мастера. При ошибке сети введённые данные не теряются и доступна повторная попытка.

## Что нового

### Возможности

- Кнопка «Отправить отзыв» теперь выполняет `POST /comments/{company}` в YCLIENTS (`postComment` в `api.js`).
- На время отправки кнопка блокируется с подписью «Отправка…», исключая двойную отправку.
- При успехе показывается подтверждение «Спасибо! Ваш отзыв отправлен.», визит помечается оценённым.
- При ошибке — понятное русскоязычное сообщение, поля экрана не очищаются, кнопка снова доступна.

### Улучшения

- Признак «оценено» перенесён на новый ключ `yc_reviewed_ids` (массив `record_id` успешно отправленных отзывов). Старый `yc_reviews` больше не читается и не пишется в затронутых модулях.
- `history.js` и `profile.js` читают `reviewedIds` из `_loadReviewedIds()` (`yc_reviewed_ids`).
- В `openRateVisit` передаётся `ycStaffId || masterId`, обеспечивая корректный YCLIENTS staff id для привязки отзыва.
- Удалена функция `_avgRating` из `masters.js` — карточка мастера в списке больше не показывает локальный средний рейтинг из упразднённого `yc_reviews`.

### Исправления

- Рейтинг на карточке мастера (`s-master`) из `GET /comments` работает без изменений.

## Изменения публичного контракта

Внешнего API/CLI нет. Публичный контракт приложения не меняется:
- Сигнатуры `openRateVisit`, `submitReview`, `openMasterCard`, `back`, `go` — сохранены.
- Внешний вид экранов `s-review`, `s-history`, `s-master` — сохранён.
- Механика чаевых на `s-review` — без изменений.

**Хранилище:** ключ `yc_reviews` более не используется; признак «оценено» теперь в `yc_reviewed_ids`. Миграция старых данных `yc_reviews` не выполняется — они игнорируются. На новом устройстве кнопка «Оценить» появится для ранее оценённых визитов (принятый компромисс, серверный признак — техдолг).

**`CACHE_VERSION`: `v35` → `v36`** (изменены `review.js`, `history.js`, `masters.js`, `api.js`).

## Известные ограничения

- Признак «оценено» хранится локально на устройстве: при входе с другого устройства кнопка «Оценить» может снова появиться для уже оценённого визита.
- Дедупликация отзывов на стороне YCLIENTS не гарантирована; локальный признак исключает дубль только на том же устройстве.
- `POST /comments` использует тот же partner-токен, что и все остальные вызовы; при ограничениях прав API отзыв не создастся, пользователь получает fail-safe сообщение.
