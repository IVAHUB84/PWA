# Release Notes v1.29.0

- **Дата:** 2026-05-30
- **Тип релиза:** minor

## Кратко

Экран «Выберите время» (`s-slots`) полностью редизайнен под привычный вид мобильного приложения YCLIENTS: горизонтальная лента дат заменена на полноценный месячный календарь с навигацией по месяцам и реальной подгрузкой доступных дат. Слоты сгруппированы по периодам суток (Утро / День / Вечер) со сворачиваемыми секциями. Убрана карточка мастера и стороннее брендирование; кнопка действия переименована в «Записаться».

## Что нового

### Возможности
- Месячный календарь-карточка: заголовок месяца, стрелки «‹ ›», строка дней недели Пн–Вс, сетка чисел; горизонтальная лента дат удалена.
- Навигация стрелками реально подгружает доступные даты соседнего месяца через `book_dates`; ответы кэшируются в Map по ключу `YYYY-MM` на время жизни экрана.
- Визуальные состояния дат: доступная (кликабельна), недоступная/прошедшая (приглушена, некликабельна), выбранная (тёмный заполненный кружок `.sel`), «сегодня» (подчёркнуто).
- Стрелка «назад» неактивна на текущем месяце; «вперёд» неактивна за горизонтом доступных дат; фоновый prefetch следующего месяца для корректной блокировки.
- Слоты сгруппированы по периодам суток: Утро `< 12:00`, День `12:00–16:59`, Вечер `≥ 17:00`; пустые секции скрыты.
- Каждая непустая секция раскрыта по умолчанию; шеврон сворачивает/разворачивает секцию (CSS-переход по `max-height` и `opacity`).

### Улучшения
- Кнопка действия в обычном сценарии переименована в «Записаться»; сценарий переноса записи (`state._rescheduleId`) по-прежнему показывает «Перенести запись».
- Фоновый prefetch следующего месяца устраняет лишние запросы при листании туда-обратно.
- Авто-выбор первой доступной даты и первого слота при открытии экрана сохранён.
- Тактильный отклик (`hapticTap`) при выборе даты, слота и переключении месяца сохранён.

### Исправления
- Удалён блок `#masterStrip`; якорь баннера контекста переноса перепривязан к `.page-title` экрана — баннер корректно отображается без зависимости от удалённого элемента.
- В `booking.js` сброс подписи кнопки в `finally` учитывает `state._rescheduleId`, не перезаписывая «Перенести запись» на «Записаться» в сценарии переноса.

## Изменения публичного контракта

Нет. Эндпоинты YCLIENTS (`/book_dates`, `/book_times`, `/book_record`), структура `state` (`slot` / `dateISO` / `dateFull` / `_rescheduleId`), поток `startBooking` → `s-confirm` остаются прежними. Изменения — только во внутренней разметке, стилях и рендере экрана `s-slots`.

## Известные ограничения

- Горизонт навигации вперёд ограничен тем, что отдаёт API (`book_dates`); при исчерпании доступных дат стрелка «вперёд» блокируется.
- Состояние раскрытия секций периодов суток и отображаемый месяц не сохраняются между заходами на экран — сбрасываются при каждом открытии.
- Анимация раскрытия секций реализована через CSS `max-height`; при очень большом числе слотов в одной секции переход может быть неточным по времени.

- КП-1 (месячный календарь, лента удалена): `prototype.html` — разметка `#s-slots` содержит `.cal-card` с `#calMonthLbl`, `#calPrev`/`#calNext`, `.cal-weekdays`, `#calGrid`; блок `.date-picker` отсутствует — **подтверждён по разметке**.

- КП-3 (состояния дат): `_renderCalGrid` выставляет классы `sel` (выбранная), `today` (подчёркнуто через CSS), `unavail` (приглушена, `pointer-events:none`) — **подтверждён по коду и CSS**.
- КП-4 (выбор даты перезагружает слоты): `calSelectDate(iso)` вызывает `loadTimes(iso)` — **подтверждён по коду**.
- КП-5 (авто-выбор первой даты и слота): `loadDates` → `_autoSelectFirstDate` → `calSelectDate(sorted[0])` → `loadTimes` → первый слот `state.slot = firstTime` + `_setConfirmBtnEnabled(true)` — **подтверждён по коду**.
