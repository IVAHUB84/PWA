# Release Notes v1.1.0

- **Дата:** 2026-05-28
- **Тип релиза:** minor

## Кратко

Экран выбора услуг (`s-services`) переработан с текстового списка на визуальный каталог карточек с фотографиями. Каждая карточка показывает обложку (фото или градиентный плейсхолдер с эмодзи категории), название, цену и длительность процедуры. Поиск, фильтр категорий и вся навигация сохранены без изменений.

## Что нового

### Возможности

- Каталог услуг отображается сеткой карточек-обложек (2 колонки, соотношение 4:3 для области изображения).
- Новый модуль `serviceImages.js`: карта привязки «название услуги → фото», карта категорий с эмодзи и CSS-градиентами, детерминированная нормализация названий (регистр, ё→е, тире, пробелы), чистая функция-резолвер `resolveServiceImage`.
- Плейсхолдеры категорий: Брови — розово-малиновый градиент, Губы — красно-коралловый, Глаза — сиреневый, Эпиляция — мятно-зелёный, остальные — сиреневый дефолт.
- `docs/images/services/` — каталог для фотографий услуг (MVP: пустой, контент поставляет заказчик).

### Улучшения

- `services.js`: обе ветки рендера (обычная и `masterPreSelected`) используют резолвер; фото рендерится с `loading="lazy"` и `onerror`-фолбэком на плейсхолдер.
- `scenarios.js`: `tapTarget` сценария записи обновлён на `.svc-catalog-card`.
- `style.css`: новые классы `.svc-catalog`, `.svc-catalog-card`, `.svc-cover`, `.svc-cover-img`, `.svc-cover-placeholder`, `.svc-info`; корректный вид в светлой и тёмной теме.
- `eslint.config.js`: добавлен override для `*.test.js` (разрешены `console`, `process`).
- `.gitignore`: восстановлены правила защиты секретов, редакторные артефакты, `cf-worker/node_modules/`, `deploy.zip`.

### Исправления

- Нет регрессий публичного контракта: `selectService`, поиск, фильтр категорий, баннер мастера — без изменений.

## Изменения публичного контракта

Публичный контракт не меняется. Источник и поля данных услуг (YCLIENTS: название, цена, длительность, категория) остаются прежними. Навигация и переходы (`selectService`, режим `masterPreSelected`) не изменены. Меняется только внутренняя вёрстка карточек на экране `s-services`.

### Service Worker

`CACHE_VERSION` поднят с `v33` до `v34`. Пользователи получат новый кэш при следующем открытии приложения.

## Известные ограничения

- `SERVICE_PHOTO_MAP` пуст: все карточки показывают плейсхолдеры до передачи фотоконтента заказчиком (допущение 6.1 BA-требований).
- Фотографии не добавлены в `APP_SHELL` — при первом офлайн-визите непросмотренные фото деградируют до плейсхолдера (поведение согласовано ADR-001, вариант 3A).
- Фотографии хранятся в git-репозитории (Вариант 1A, MVP): при накоплении контента потребуется ручная оптимизация веса перед коммитом. Миграция на S3 — технический долг, локализованный в резолвере (ADR-001, раздел «Путь миграции»).
- КП-12 (подтверждение заказчика о премиум-виде) не может быть закрыт без участия заказчика.
