# Release Notes v1.25.0

- **Дата:** 2026-05-30
- **Тип релиза:** minor

## Кратко

Администратор студии теперь может открывать детали записи или клиента одним тапом, не уходя в веб-кабинет YCLIENTS. Из всплывающего шита сразу доступна кнопка «Позвонить», открывающая системный диалер.

## Что нового

### Возможности

- **Drill-down шит записи (Сводка).** Тап по строке записи открывает bottom-sheet с именем клиента, названием услуги, именем мастера, временем и стоимостью. При наличии `client.phone` — кнопка «Позвонить» с `tel:`-ссылкой на нормализованный номер. Телефон берётся из ответа `/records` без дополнительных запросов (ограничение Огр-3 соблюдено).
- **Drill-down шит клиента (База).** Тап по строке клиента открывает bottom-sheet с именем, телефоном и числом визитов. Кнопка «Позвонить» при наличии телефона.
- **Выделяемый телефон.** Номер в обоих шитах можно выделить и скопировать: точечное исключение из глобального `user-select:none` через CSS-класс `.sheet-phone` (по образцу `.studio-contacts`, ADR-024).

### Улучшения

- `_telHref` в `studio.js` помечен `export` — единый источник нормализации `tel:` переиспользуется в `admin.js` без дублирования регулярного выражения.
- `.booking-row` и `.client-row` получили `cursor: pointer` — визуальный признак кликабельности.
- Кликабельность строк сохраняется после фильтрации клиентов (`filterAdminClients`): `onclick` задаётся в шаблоне строки `_renderClientRows`, индекс ссылается на актуально отрисованный массив.

### Исправления

- Нет.

## Изменения публичного контракта

- **`docs/modules/studio.js`:** функция `_telHref` стала экспортируемой. Обратимо-совместимое расширение — существующие импортёры не затронуты, внутреннее использование в `renderStudioContacts` сохраняется.
- **`window`-API `admin.js`:** добавлены глобальные обёртки `_openBookingSheet(index)` и `_openClientSheet(index)` для `onclick`-вызовов из разметки строк. Следуют принятому в модуле стилю.
- **Внешний API / YCLIENTS / схема БД:** без изменений. Новых сетевых запросов нет.

## Известные ограничения

- Действие «Открыть в YCLIENTS» (deep-link на запись/клиента) отсутствует — вне объёма v1.25.0 (решение по ОВ-2, BA-030). Возможность сформировать стабильный URL не подтверждена; зафиксировано кандидатом на будущий релиз.
- Деструктивные операции с записями (отмена, перенос) — отдельное требование, не входят в этот релиз.
- Браузерный прогон (DevTools, реальный диалер, сервис-воркер) в CI-среде недоступен — проверен статически по коду.
