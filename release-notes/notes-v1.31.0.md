# Release Notes v1.31.0

- **Дата:** 2026-05-31
- **Тип релиза:** minor

## Кратко

Релиз вводит персистентный кэш каталога услуг, мастеров и цен в localStorage с моделью stale-while-revalidate: при повторном холодном старте каталог и диапазоны цен отображаются мгновенно из снимка, а данные обновляются из сети в фоне. Дополнительно: троттлинг фоновых обновлений главной (60 с), снятие cache-bust в `_ghRead`, сокращение дублирующих ре-рендеров каталога при старте. Видимый UI, поток бронирования и внешний контракт не меняются.

## Что нового

### Возможности
- Новый модуль `docs/modules/dataCache.js`: чтение/запись снимка каталога (`yc_catalog_cache_v1`) и цен (`yc_prices_cache_v1`) в localStorage с версионированием ключей и безопасной обработкой ошибок (`try/catch`).
- Синхронная гидрация `SERVICES_DATA`, `MASTERS_DATA`, `servicePriceRange`, `staffServicePrice` из снимка при старте — каталог и цены рендерятся до прихода сетевого ответа.
- Функция `hydratePrices` в `state.js`: наполняет/очищает объекты цен из снимка с фильтрацией невалидных диапазонов (NaN, Infinity, null).
- Запись снимка каталога после успешного `Promise.all` в `initApp`; запись снимка цен в конце `_collectStaffPrices`.

### Улучшения
- Троттлинг фоновых обновлений на `s-home`: `_fetchAndMergeServerRecords` и `_ghRead` при повторных входах не вызываются чаще одного раза в 60 с (timestamp `_lastHomeRefresh` хранится в памяти модуля).
- Убран `?_=Date.now()` из `_ghRead` (`github.js`): актуальность данных регулируется троттлингом, а не принудительным обходом HTTP-кэша.
- Сокращение дублирующих ре-рендеров каталога при старте (ФТ-11): число полных вызовов `renderServices()` уменьшено относительно прежних ≥4.
- `CACHE_VERSION` поднята `v87` → `v88`; новый модуль `./modules/dataCache.js` добавлен в `APP_SHELL`.

### Исправления
- Кэш цен мастеров (`servicePriceRange`, `staffServicePrice`), прежде живший только в памяти сессии (ADR-012, техдолг), теперь персистируется и восстанавливается при последующих запусках.

## Изменения публичного контракта

Внешний API YCLIENTS, поток бронирования (`логин → услуга → мастер → слот → бронь`), состав экранов, контент и видимый UI **не меняются**.

Внутренние изменения:
- Новый модуль `docs/modules/dataCache.js` — часть APP_SHELL.
- Новые ключи в localStorage: `yc_catalog_cache_v1`, `yc_prices_cache_v1` (снимки каталога и цен). Ключи содержат только нормализованные поля каталога, токены и секреты не кэшируются.
- Новый экспорт `hydratePrices` из `docs/modules/state.js`.

Миграция не требуется: при несовпадении версии схемы снимок игнорируется (forward-compatible). Старый SW автоматически инвалидируется подъёмом `CACHE_VERSION`.

## Известные ограничения

- ФТ-12 (ленивая загрузка EmailJS) **не реализована** в этом релизе — вынесена в отдельное требование/релиз (решение ADR-033, Вопрос 6).
- При первом запуске (нет снимка) или после очистки хранилища поведение совпадает с прежним: каталог ждёт сетевого ответа.
- Снимок каталога содержит состояние мастеров на момент последней загрузки (`avail`/`availText`): при повторном старте без сети могут отображаться устаревшие статусы доступности до прихода сетевого обновления.

## Проверка

**Линтер:** `npm run lint` — пройден без ошибок (ESLint, `docs/modules/` + `docs/app.js`).

**Автотесты:** `npm test` — 347 тестов, 0 упало (10 наборов: install, masters-filter, serviceImages, review-post, post-comment-worker, feed-typed-posts, inboxTarget, **dataCache** — новый, cf-worker/subscriptions, cf-worker/preferences).

**Критерии приёмки BA-036:**

- КП-1 (мгновенный каталог при повторном старте): `_hydrateFromCache()` вызывается синхронно до `initApp()`; `readCatalogSnapshot()` возвращает данные без `await`; `renderServices()` вызывается сразу после гидрации. **Подтверждён по коду. Реальная мгновенность рендера до ответа сети — требует ручной проверки в браузере (DevTools → Network).**
- КП-2 (фоновое обновление без мигания): после `Promise.all` в `initApp` данные обновляются и `renderServices()` вызывается — то же поведение, что прежде. **Подтверждён по коду. Отсутствие видимого мигания — требует ручной проверки в браузере.**
- КП-3 (каталог без ожидания N запросов цен): `_collectStaffPrices()` вызывается после `renderServices()` в `initApp`, не блокируя рендер; диапазоны цен применяются из снимка немедленно. **Подтверждён по коду (порядок вызовов в `initApp`). Визуальная проверка — в браузере.**
- КП-4 (офлайн с кэшем / без кэша): `readCatalogSnapshot()` и `readPricesSnapshot()` не зависят от сети; `initApp` обёрнут в `try/catch`. **Подтверждён по коду. Эмуляция офлайн — требует ручной проверки в браузере (DevTools → Network: Offline).**
- КП-5 (троттлинг `s-home` 60 с): обработчик `s-home` проверяет `Date.now() - _lastHomeRefresh < HOME_REFRESH_THROTTLE_MS (60000)` и возвращается досрочно; `renderHomeHero()`/`updateInboxBadge()` выполняются всегда. **Подтверждён по коду. Отсутствие сетевых запросов — требует ручной проверки в браузере (DevTools → Network).**
- КП-6 (нет `?_=` в `_ghRead`): `github.js` строка 11 — `fetch(_GH_RAW)` без параметров. **Подтверждён по коду (статически).**
- КП-7 (сквозной сценарий без регрессий): поток бронирования, слоты, инбокс, auth — файлы не изменялись в этом релизе. **Подтверждён статически (diff затрагивает только dataCache.js, app.js boot/initApp/s-home, state.js, github.js, sw.js, version.json). Сквозной прогон логин → бронь и ноль ошибок в консоли — требуют ручной проверки в браузере.**
- КП-8 (устойчивость при ошибке хранилища): все `setItem`/`getItem` в `dataCache.js` обёрнуты в `try/catch`; автотест `dataCache.test.js` явно проверяет, что `QuotaExceededError` и `storage error` не пробрасываются. **Подтверждён автотестом.**
- КП-9 (нет токенов в снимке): `_pickCatalogService` в `dataCache.js` копирует только `SAFE_CATALOG_FIELDS = ['id','name','cat','dur','price_min','priceStr','comment','photos']`; автотест проверяет, что `userToken` и `YC_TOKEN` не попадают в снимок услуг и мастеров. **Подтверждён по коду и автотесту. Просмотр DevTools → Application → Local Storage — требует ручной проверки.**

(сокращение ре-рендеров): в `initApp` `renderServices()` вызывается один раз после успешного сетевого ответа и один раз в конце `_collectStaffPrices`; boot вызывает ещё один `renderServices()` после гидрации из кэша. **Подтверждён по коду — число снижено. Точный счёт через инструментирование — ручная проверка.**
- КП-11 опц. (ленивый EmailJS): **не применимо** — ФТ-12 исключена из релиза.
- КП-12 (CACHE_VERSION поднята): `sw.js` строка 1 — `CACHE_VERSION = 'v88'`; `./modules/dataCache.js` присутствует в `APP_SHELL`. **Подтверждён по коду.**
