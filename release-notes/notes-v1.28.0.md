# Release Notes v1.28.0

- **Дата:** 2026-05-30
- **Тип релиза:** minor

## Кратко

Уведомления стали точками входа: клик по карточке инбокса или по системному push открывает нужный раздел или карточку услуги напрямую. Отметка «прочитано» стала точечной — только по кликнутому уведомлению. Управление историей разведено на два явных действия: иконка корзины удаляет всю историю, кнопка «Прочитать всё» убирает бейдж без удаления.

## Что нового

### Возможности
- Клик по уведомлению-напоминанию о визите (в инбоксе и по системному push) открывает раздел «Записи».
- Клик по уведомлению об услуге открывает карточку этой услуги через `openServiceCard(id)`.
- Клик по уведомлению о категории открывает каталог с применённым фильтром категории.
- Уведомление без цели при клике только помечается прочитанным — никакого перехода, никаких ошибок на старых записях.
- Кнопка «Прочитать всё» над списком инбокса: помечает все уведомления прочитанными без удаления, обнуляет бейдж. Скрыта/`disabled` при отсутствии непрочитанных и при пустом списке.
- В композере пуша (админка) появился выбор цели рассылки: «без цели» (дефолт), «услуга» (выбор из каталога), «категория» (выбор из существующих категорий). Цель доезжает в payload и сохраняется в IDB-записи получателя.

### Улучшения
- Единый маршрутизатор цели `routeToTarget` (модуль `inboxTarget.js`) — одинаковая цель даёт одинаковый переход из инбокса и из системного клика, в т. ч. при закрытом приложении (открытие через `?n=<encoded>` с разбором при старте).
- Поле `target` добавлено в IDB-запись (`inboxStore.js`) и в инлайн-контракт SW (`sw.js`), синхронизировано. Старые записи без `target` читаются как «без цели» — без миграции, `DB_VERSION` = `1`.
- Payload напоминаний в Cloudflare Worker теперь включает `target:{type:'records'}`.

### Исправления
- Заход в инбокс через колокольчик больше не помечает все уведомления прочитанными — `markAllRead()` убран из `enterInbox()`.
- Иконка метёлки в шапке инбокса заменена на иконку корзины с aria-меткой; действие (удалить всю историю с подтверждением) сохранено.

## Изменения публичного контракта

- **Payload push** (рассылки и напоминания): добавлено необязательное поле `target:{type,id?}` (`type ∈ {service, records, category}`). Старые клиенты игнорируют его. Миграция не требуется.
- **IDB `studio-inbox`, store `notifications`**: запись расширена необязательным полем `target`. `DB_VERSION` = `1` (без изменений); старые записи без `target` валидны (= «без цели»).
- **`sendAdminPush(title, body, targetClientId, target?)`** — новый необязательный 4-й параметр `target`.
- **Worker `/send`** — принимает необязательное поле `target` в теле запроса.
- **Глубокая ссылка** `./?n=<encoded target>` — новый query-параметр запуска приложения; разбирается и очищается при старте (`history.replaceState`).
- **`CACHE_VERSION`**: поднята `v83 → v84`; модуль `./modules/inboxTarget.js` добавлен в `APP_SHELL`.

## Известные ограничения

- Цель `category` ключуется по имени-строке. При переименовании категории старые уведомления деградируют в «без цели» — ожидаемое поведение, без ошибок.
- Детальная карточка визита отсутствует; целью напоминания о визите остаётся раздел «Записи».
- Cloudflare Worker деплоится отдельным шагом (см. ниже).
