# Release Notes v1.15.0

- **Дата:** 2026-05-29
- **Тип релиза:** minor

## Кратко

На главном экране плашка «Оцените прошлый визит» получила крестик «×», позволяющий закрыть подсказку без выставления оценки. Скрытие персистентно — при повторных заходах для того же визита плашка не появляется; возможность оценить визит через историю визитов сохраняется.

## Что изменено

### Возможности

- **Скрытие плашки «Оцените прошлый визит» без оценки (BA-020, ADR-017).**
  - На карточке плашки добавлен крестик «×» (`rate-dismiss-btn`) в правом верхнем углу — абсолютное позиционирование поверх контента, тач-зона 32×32px, `aria-label="Скрыть"`, `type="button"`. Контент карточки (заголовок, услуга, мастер/дата, кнопка «Оставить отзыв →») не сдвигается.
  - После нажатия крестика блок `#homeHero` немедленно перерисовывается в штатный фолбэк «Нет предстоящих записей» через `renderHomeHero()`.
  - Факт скрытия сохраняется в `localStorage` под ключом `yc_dismissed_rate_ids` (массив строковых id). При повторных заходах для того же визита плашка не появляется.
  - Скрытый визит остаётся **неоценённым**: его id не попадает в `yc_reviewed_ids`, кнопка «Оценить» в истории визитов остаётся доступной.
  - Для нового прошедшего неоценённого визита (другой `recordId`) плашка появляется снова — дисмисс пер-`recordId`.

### Служебное

- `CACHE_VERSION` поднята `v63 → v64` (изменены файлы `APP_SHELL`: `review.js`, `profile.js`, `style.css`).
- `docs/version.json`: build 30 → 31, version v1.15.0.

## Изменения публичного контракта

Внешний API / YCLIENTS / схемы данных не меняются. Скрытие не отправляет ничего на бэкенд.

Новый ключ `localStorage`: **`yc_dismissed_rate_ids`** — массив строковых id, внутреннее клиентское состояние. Отдельный от `yc_reviewed_ids`. Отсутствие ключа при первом запуске трактуется как `[]` — миграция не требуется.

Наблюдаемое изменение UI: на плашке «Оцените прошлый визит» появился крестик «×». Поток отзывов (`openRateVisit` → `s-review` → `submitReview`) и экран истории визитов не изменены.

## Известные ограничения

- Список `yc_dismissed_rate_ids` монотонно растёт: очистка id, выпавших из 7-дневного окна, не реализована (вне объёма BA-020). При необходимости — отдельным требованием.
- Скрытие локально: синхронизация между устройствами не предусмотрена.

## Проверка

**Статическая (код), пройдено:**

- КП-1: крестик присутствует в разметке (`type="button"`, `class="rate-dismiss-btn"`, `aria-label="Скрыть"`); кнопка «Оставить отзыв →» не тронута. — **пройдено статически**.
- КП-2: `dismissRateVisit` вызывает `_saveDismissedRateId` + `renderHomeHero()` — перерисовка через штатный путь. — **пройдено статически**.
- КП-3/КП-4: `_loadDismissedRateIds()` читает `yc_dismissed_rate_ids`; фильтр `.filter(r => !dismissedIds.has(String(r.id)))` параллелен фильтру по `reviewedIds`; дисмисс пер-`recordId`. — **пройдено статически**.
- КП-5: `yc_dismissed_rate_ids` и `yc_reviewed_ids` — отдельные ключи; `_saveReviewedId` не изменён; `history.js` не тронут. — **пройдено статически**.
- КП-6: `openRateVisit`, `submitReview`, `_loadReviewedIds`, `_saveReviewedId` — без изменений. — **пройдено статически**.
- КП-7: `_loadDismissedRateIds()` возвращает `[]` в `catch`; `_saveDismissedRateId` пишет в `try/catch`. — **пройдено статически**.
- КП-8: все данные визита в плашке через `esc()`; крестик пользовательских данных не выводит. — **пройдено статически**.
- КП-9: `CACHE_VERSION` v64 подтверждена в `sw.js`; новых рантайм-зависимостей нет. — **пройдено статически**.
- Линт (`npm run lint`): чисто, без ошибок. — **пройдено**.

**Требует ручной проверки в браузере:**

- C1 — на главном при наличии прошедшего неоценённого визита показывается плашка с крестиком; нажатие крестика скрывает плашку, показывается «Нет предстоящих записей»; на `s-review` перехода нет.
- C2 — после скрытия: перезагрузка/повторное открытие → плашка не появляется; в `localStorage` есть `yc_dismissed_rate_ids` с этим id; в `yc_reviewed_ids` этого id нет.
- C3 — при наличии другого прошедшего неоценённого визита (другой `recordId`) плашка появляется снова.
- C4 — для скрытого визита кнопка «Оценить» в истории визитов доступна и открывает `s-review`.
- C5 — кнопка «Оставить отзыв →» открывает `s-review`, поток отправки работает.
- Деградация — при недоступном `localStorage` (эмуляция исключения) главный экран рендерится без ошибок в консоли.
- Тач/вёрстка — крестик удобен на тач; контент карточки не сдвигается; узкий экран и тёмная тема.
- PWA — SW переустановился на `v64` (DevTools → Application → Service Workers: статус active); устанавливаемость и оффлайн-поведение не регрессировали.
