# Release Notes v1.20.0

- **Дата:** 2026-05-30
- **Тип релиза:** minor

## Кратко

Релиз убирает «вебовое» поведение интерфейса при долгом нажатии: текст кнопок, карточек и меток больше не выделяется, изображения не вызывают нативное контекстное меню на iOS и не перетаскиваются на десктопе. Поля ввода полностью сохраняют ввод, выделение и вставку, в том числе вставку кода из SMS.

## Что нового

### Улучшения

- **Глобальный запрет выделения текста.** `user-select: none` (со всеми вендорными префиксами) на `body` — long-press по любому элементу UI не вызывает подсветку и маркеры выделения.
- **Запрет нативного callout на iOS.** `-webkit-touch-callout: none` на `body` и `img` — long-press по изображениям и элементам не открывает меню «Сохранить / Скопировать / Поделиться».
- **Запрет drag по изображениям.** `-webkit-user-drag: none; user-drag: none` глобально на `img` — картинку нельзя «утащить» мышью на десктопе.
- **Исключение для полей ввода.** `input`, `textarea`, `[contenteditable]` сохраняют `user-select: text` и `-webkit-touch-callout: default` — каретка, выделение, копирование и вставка работают штатно.

### Исправления

- Регресс press feedback (v1.19.x): теперь при удержании элемента управления пользователь видит только визуальную реакцию нажатия, без системного выделения текста.

## Изменения публичного контракта

Внешнего API, CLI и схем БД нет. Поведенческое изменение UI: текст вне полей ввода больше нельзя выделить вручную. Следствие — теряется возможность скопировать телефон/адрес студии/промокод нативным выделением. Потеря принята заказчиком осознанно (ОВ-2, ADR-022); компенсирующий функционал (кнопки «Скопировать») не входит в данный релиз. Миграционный путь не требуется.

## Известные ограничения

- Ручное копирование телефона/адреса студии и промокодов через нативное выделение текста более недоступно. Компенсация (отдельная задача) — вне рамок этого релиза.
