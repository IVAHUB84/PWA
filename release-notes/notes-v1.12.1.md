# Release Notes v1.12.1

- **Дата:** 2026-05-29
- **Тип релиза:** patch (hotfix)

## Кратко

Исправление дефекта v1.12.0: галерея фото услуги не появлялась — карусель показывала одно фото, даже когда в YCLIENTS у услуги несколько фото.

## Исправления

- **Карусель фото услуги не отображалась (источник галереи).** В v1.12.0 каталог дочитывал галерею из списочного management-эндпойнта `GET /company/{company}/services/`, но на живых данных подтверждено, что он **схлопывает `image_group.images` до одного фото** (объект `{basic}`) — массив фото отдаёт только одиночный эндпойнт `/{id}`. Из-за этого `photos` всегда содержал 1 элемент и ветка карусели (`photos.length >= 2`) не срабатывала.
- **Источник галереи изменён на `GET /book_services/{company}`** → поле `data.services[].images` (массив URL-строк, main первым + gallery). Одним запросом для всего каталога. Подтверждено на услуге «Маникюр с покрытием гель-лак» (3 фото). Маппинг прочих полей каталога (название, категория, длительность, диапазон цен, описание) и поток записи не затронуты.
- `CACHE_VERSION` поднята `v58 → v59` — гарантированный подхват фикса клиентами.

## Связанные артефакты

- ADR-014 (раздел «ПОПРАВКА (v1.12.1)») — корректировка решения G1.
- Изменён `docs/app.js` (источник галереи), `docs/sw.js` (CACHE_VERSION).
