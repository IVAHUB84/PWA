# Release Notes v1.27.0

- **Дата:** 2026-05-30
- **Тип релиза:** minor

## Кратко

Полная переработка плашки «О студии» на экранах подтверждения записи (`s-confirm`) и профиля (`s-profile`). Данные берутся исключительно из YCLIENTS — пустые поля показываются прочерком, а не выдуманными плейсхолдерами. Кнопка навигации теперь открывает выбор картографического приложения с построением маршрута по реальным координатам студии.

## Что нового

### Возможности
- Кнопка «Показать на карте» открывает bottom-sheet с выбором приложения: Apple Maps, Google Maps, Яндекс Навигатор, Яндекс Карты. При наличии координат строится маршрут к точке студии (`55.686006, 37.85368`); при отсутствии координат — поиск по тексту адреса. Кнопка скрывается, если нет ни координат, ни адреса.
- В плашке добавлены строки Сайт и Telegram (скрываются при пустом значении).
- Адрес отображается с городом: «Люберцы, улица Генерала Кузнецова, 22».

### Улучшения
- Источник данных компании переключён с `GET /company/{id}` на `GET /companies?id={id}` — этот endpoint отдаёт реальные координаты и полный набор полей.
- Нормализованная схема кэша `localStorage yc_company` расширена: добавлены `publicTitle`, `city`, `site`, `telegram`, координаты — плашка корректно работает в оффлайне.
- Заголовок плашки берётся из `public_title` (имя для онлайн-записи) с фолбэком на `title`.
- Пустые поля (телефон, часы, адрес, заголовок) показываются прочерком `—` с сохранением метки строки — никаких фейковых данных.
- Иконки SVG у каждой строки; bottom-sheet оформлен через токены темы (корректно в светлой и тёмной теме).
- Контакты выделяются и копируются (`user-select: text` сохранено).

### Исправления
- Устранён баг: кнопка «Маршрут» вела на форму отзыва чужой студии (Троицк) из-за нулевых координат и фолбэка на `REVIEW_URLS.yandex`.
- Устранён баг: отображался фейковый телефон `+7 495 123-45-67` из плейсхолдера `STUDIO`; теперь при пустом API — `—`.
- Удалены две отдельные ссылки «Маршрут» и «2ГИС», заменены единой кнопкой выбора.
- Строка метро не выводится (поля метро в API YCLIENTS нет).

## Изменения публичного контракта

- **YCLIENTS API:** запрос данных компании — `GET /companies?id={id}` вместо `GET /company/{id}`.
- **`localStorage yc_company`:** расширенная нормализованная схема. Обратно совместима по чтению: старый формат v1.23.0 читается, отсутствующие поля трактуются как пустые (прочерк/скрытие до первого онлайн-старта). Сброс/миграция ключа не требуется.
- **`docs/modules/constants.js`:** удалён экспорт `STUDIO`. Других потребителей в проекте не было — ломающее изменение только в рамках внутреннего кода.
- **UI:** одна кнопка «Показать на карте» вместо «Маршрут» + «2ГИС»; новые строки Сайт и Telegram; прочерк для пустых полей.

## Известные ограничения

- Телефон студии в API YCLIENTS по-прежнему пуст (partner-токен не отдаёт телефон) — в плашке отображается `—`. Заполнится автоматически при появлении данных в YCLIENTS.
- App-схема `yandexnavi://` на desktop может не сработать — ожидаемое поведение; Google/Apple/Яндекс Карты открываются веб-версией.
- `social.viber` заполнен в API, но не выводится в плашке — сознательно вне scope по решению заказчика.
