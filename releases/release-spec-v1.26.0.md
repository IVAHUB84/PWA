# Спецификация релиза v1.26.0

- **Дата планирования:** 2026-05-30
- **Тип релиза:** minor (обратимо-совместимое изменение админ-UI: реорганизация навигации и метрик режима студии; публичный клиентский контракт не меняется)
- **Цели:** убрать случайный выход из режима студии (вынести «Выйти» из tab-bar в шапку с подтверждением) и убрать фейковую метрику сводки, заменив рейтинг честным «средним чеком сегодня».

## Источники
- BA-требования: `ba-req/ba-requirements-031.md`
- ADR: `adrs/adr-028.md`

## Объём релиза

### Задача 1: Убрать пятую вкладку «Выйти» из tab-bar (все 4 экрана)
- **Что:** удалить `<div class="tab" onclick="go('s-login','tab')">…</div>` (вкладка «Выйти» с иконкой двери) из каждой из четырёх копий `tab-bar`. Оставшиеся 4 вкладки (Сводка/Лента/Клиенты/Рассылка) и подсветка `active` не меняются.
- **Где (файлы/модули):** `docs/prototype.html` — строки 708 (`s-admin`), 727 (`s-admin-feed`), 785 (`s-admin-clients`), 814 (`s-admin-push`). Удалить ровно эти `<div class="tab" onclick="go('s-login','tab')">`.
- **Контракт:** на каждом из 4 экранов `tab-bar` содержит ровно 4 `.tab`. Никакие `onclick` других вкладок не меняются.
- **Критерии готовности:** КП-1, КП-2. На всех 4 экранах ровно 4 вкладки; вкладки «Выйти» нет; переключение и подсветка активной работают.
- **Связь с ADR:** ADR-028, Решение п.1.

### Задача 2: Кнопка выхода в шапке + обработчик с подтверждением (все 4 экрана)
- **Что:**
  1. В `admin.js` добавить функцию `_adminLogout()`: `if (confirm('Выйти из режима студии?')) go('s-login');` — `clearSession()`/`logout()` НЕ вызывать. Добавить `_adminLogout` в существующий блок глобального экспорта (admin.js строки 538–548).
  2. В `admin-header` каждого из 4 экранов добавить иконку-кнопку выхода в **левой части шапки, перед блоком брендинга**, с `onclick="_adminLogout()"`. На `s-admin-clients` (prototype.html:775) шапку привести к flex-паттерну остальных (`display:flex;align-items:center;`), сохранив порядок: кнопка выхода → блок брендинга.
  3. Правые кнопки шапки НЕ трогать: «🔑 Токен» (`_adminResetToken`, prototype.html:691), «+ Новый» (prototype.html:717), «+ Новая» (prototype.html:794).
- **Где (файлы/модули):** `docs/modules/admin.js` (новая функция + экспорт); `docs/prototype.html` — `admin-header` строк 691 (`s-admin`), 715–718 (`s-admin-feed`), 775 (`s-admin-clients`), 792–795 (`s-admin-push`).
- **Контракт:** глобально доступная `window._adminLogout`; единый паттерн размещения кнопки выхода (слева) на всех 4 экранах; правые кнопки сохранены и работают.
- **Критерии готовности:** КП-3, КП-4, КП-5. Кнопка выхода есть на всех 4 экранах слева, не перекрывает «Токен»/«+ Новый»/«+ Новая»; нажатие показывает `confirm`; отмена оставляет на экране; подтверждение делает `go('s-login')`; сессия не сбрасывается.
- **Связь с ADR:** ADR-028, Решения п.2, п.3, п.4 (и обоснование вариантов: размещение слева, `window.confirm`, единая функция `_adminLogout`).

### Задача 3: Заменить фейковую карточку рейтинга на «Средний чек сегодня»
- **Что:**
  1. В `prototype.html:696` карточку `<div class="stat-card"><div class="stat-val">4.8 ★</div><div class="stat-lbl">Средняя оценка</div><div class="stat-diff up">за 30 дней</div></div>` заменить на карточку с `id`-элементом значения (например `<div class="stat-val" id="adminStatAvg">—</div>`) и подписью `<div class="stat-lbl">Средний чек сегодня</div>`. Захардкоженного «4.8 ★» и подписей «Средняя оценка»/«за 30 дней» в HTML не остаётся. Сетка `stat-grid` сохраняет 4 карточки.
  2. В `renderAdminDashboard()` (`admin.js`, после расчёта `rev` ~строка 141) проставить значение: при `recs.length > 0` — `Math.round(rev / recs.length).toLocaleString('ru-RU') + ' ₽'`; при `recs.length === 0` — `'—'`. Запись в элемент по `id` через `if (el) el.textContent = …` (паттерн как у `adminStatRevenue`). Дополнительных запросов к API не делать.
- **Где (файлы/модули):** `docs/prototype.html` (карточка сводки), `docs/modules/admin.js` (`renderAdminDashboard`).
- **Контракт:** значение карточки формируется только в JS из уже загруженных `rev` и `recs`; при нуле записей — «—», без деления на ноль; в HTML нет захардкоженного значения.
- **Критерии готовности:** КП-6, КП-7. Нет «4.8 ★ / Средняя оценка / за 30 дней» в HTML; карточка «Средний чек сегодня»; сетка 4 карточки; значение = `rev/recs.length` из загруженных данных; «—» при нуле записей.
- **Связь с ADR:** ADR-028, Решение п.5.

### Задача 4: Убрать пустые `stat-diff` из карточек сводки
- **Что:** удалить элементы `<div class="stat-diff up"></div>` (пустые) из карточек статистики. У оставшихся карточек после перепрофилирования рейтинга (Задача 3) `stat-diff` тоже не остаётся.
- **Где (файлы/модули):** `docs/prototype.html` — карточки `stat-grid` строки 694, 695, 697 (пустые `stat-diff`); элемент `stat-diff` в карточке рейтинга устраняется в рамках Задачи 3.
- **Контракт:** в разметке сводки `s-admin` элементов `class="stat-diff"` не остаётся.
- **Критерии готовности:** КП-8. Нет пустых `stat-diff` в интерфейсе сводки.
- **Связь с ADR:** ADR-028, Решение п.6.

### Задача 5: Поднять CACHE_VERSION
- **Что:** в `docs/sw.js:1` поднять `CACHE_VERSION` с `'v80'` на `'v81'` (изменяются `prototype.html` и `modules/admin.js`, оба в `APP_SHELL`).
- **Где (файлы/модули):** `docs/sw.js`.
- **Критерии готовности:** КП-9 (часть про CACHE_VERSION). `CACHE_VERSION === 'v81'`.
- **Связь с ADR:** ADR-028, Решение п.7.

## Изменения публичного контракта
- Клиентский (не-админский) контракт PWA — без изменений.
- Внутренний UI-контракт режима студии: tab-bar становится 4-вкладочным; действие выхода переезжает в шапку. Миграция не требуется — отдельной админ-сессии в хранилище нет (BA-031, раздел 6), поведение выхода (`go('s-login')` без сброса сессии) сохраняет прежний эффект пятой вкладки.
- Новый глобальный обработчик `window._adminLogout`.
- `reference.html` — техническая справка по YCLIENTS API, описаний tab-bar/рейтинга админки не содержит, правок не требует (ADR-028, «Влияние на код»).

## Тесты
Автотестов в проекте нет (CLAUDE.md). Ручная приёмка тестировщиком по критериям BA-031 (КП-1…КП-9):
- На всех 4 админских экранах в tab-bar ровно 4 вкладки, «Выйти» отсутствует; переключение и подсветка работают.
- Кнопка выхода в шапке на всех 4 экранах слева, не перекрывает правые кнопки; «🔑 Токен», «+ Новый», «+ Новая» на месте и работают.
- Нажатие выхода → `confirm`; «Отмена» — остаёмся; «ОК» — переход на `s-login`; клиентская сессия не сброшена.
- Сводка: карточка «Средний чек сегодня» = выручка/записи; при отсутствии записей на сегодня — «—»; нет «4.8 ★»; сетка 4 карточки.
- В разметке сводки нет пустых `stat-diff`.
- DevTools → Console: ноль ошибок; Service Worker active; `CACHE_VERSION === 'v81'`.

## Риски и митигации
- **Риск:** рассогласование 4 копий tab-bar (вкладку удалили не во всех). → **Митигация:** в спецификации указаны все 4 строки (708/727/785/814); приёмка проверяет все 4 экрана (КП-1, НФТ-1).
- **Риск:** случайное удаление/повреждение рабочей кнопки «🔑 Токен»/«+ Новый»/«+ Новая» при правке шапки. → **Митигация:** кнопка выхода добавляется слева, правые кнопки не трогаются (ADR-028 вариант B); приёмка отдельно проверяет наличие и работу правых кнопок (КП-3).
- **Риск:** деление на ноль при нуле записей за сегодня. → **Митигация:** явная ветка `recs.length === 0 → '—'` до деления (Задача 3, КП-7).
- **Риск:** забыли поднять CACHE_VERSION → клиенты получают старый shell. → **Митигация:** Задача 5 + проверка `v81` в приёмке (КП-9).
- **Риск:** на `s-admin-clients` шапка без flex — кнопка выхода съедет. → **Митигация:** Задача 2 явно требует привести шапку к flex-паттерну остальных экранов.
