# Спецификация релиза v1.15.0

- **Дата планирования:** 2026-05-29
- **Тип релиза:** minor
- **Цели:** дать клиенту возможность скрыть плашку «Оцените прошлый визит» на главном экране **без выставления оценки** (крестик «×»), после чего плашка для этого визита больше не показывается, а на её место выводится штатный фолбэк «Нет предстоящих записей». Это точечная UX-правка одного блока `s-home`, не пересмотр логики отзывов.

## Источники

- BA-требования: `ba-req/ba-requirements-020.md` (скрытие плашки «Оцените прошлый визит» без оценки; ОВ-1 РЕШЕН заказчиком — вариант «а», пер-`recordId`-дисмисс; ОВ-2/ОВ-3 — за архитектором; ОВ-4 подтверждён — история не затрагивается; ОВ-5 закрыт анализом — счётчиков нет).
- ADR: `adrs/adr-017.md` (N1 — ключ `yc_dismissed_rate_ids` + помощники в `review.js`; O1 — крестик «×» на карточке; P1 — перерисовка через `window.renderHomeHero()`).

## Объём релиза

Все задачи строго в пределах объёма BA-020 (§2) и границ ADR-017. **Не трогаются:** поток «Оставить отзыв» (`openRateVisit` → `s-review` → `submitReview`, `review.js`) и `yc_reviewed_ids`; кнопка «Оценить» в истории визитов (`history.js:206`, завязана на `reviewedIds`); логика предстоящих записей (`next`/`upcoming`, `profile.js:85-88,116-129`); сам фолбэк «Нет предстоящих записей» (`profile.js:109-112`); правила формирования `lastUnreviewed` (окно 7 дней, фильтры статуса/даты) кроме добавления параллельного фильтра по дисмиссу; логика `go`/`back`/`navHistory`; прочие экраны и карточки. Серверная/YCLIENTS-сторона при скрытии не задействуется (состояние только локальное). YCLIENTS-токен не трогается. Ветка `backup` запрещена. Без новых рантайм-зависимостей.

### Изменяемые файлы (сводка)

- `docs/modules/review.js` — два новых помощника `_loadDismissedRateIds()`/`_saveDismissedRateId()` (Задача 1).
- `docs/modules/profile.js` — фильтр по дисмиссу в условии показа, крестик в разметке плашки, обработчик скрытия (Задачи 1, 2, 3).
- `docs/style.css` — при необходимости стили крестика/тач-зоны (Задача 2).
- `docs/sw.js` — поднятие `CACHE_VERSION` `v63 → v64` (общий пункт).
- `docs/version.json` — мейнтейнер: `build` 30 → 31, `version` → `v1.15.0` (этап `/release`/`/deploy`).
- `docs/modules/history.js` — **без изменений** (ОВ-4).
- `docs/app.js` — **без изменений** (`registerOnEnter('s-home')` уже вызывает `renderHomeHero`).

### Задача 1: Персистентность дисмисса — новый ключ `yc_dismissed_rate_ids` и фильтр в условии показа

- **Что:**
  1. В `docs/modules/review.js` добавить два экспортируемых помощника **рядом** с `_loadReviewedIds`/`_saveReviewedId` (`review.js:8-21`), структурно идентичных им:
     - `_loadDismissedRateIds()` — читает `localStorage.getItem('yc_dismissed_rate_ids')`, парсит JSON, возвращает массив; при любой ошибке (`JSON.parse`/недоступность хранилища) или не-массиве — `[]` (через `try/catch`, как `_loadReviewedIds`).
     - `_saveDismissedRateId(recordId)` — берёт текущий массив, добавляет `String(recordId)` если его ещё нет, пишет обратно в `try/catch` (как `_saveReviewedId`).
  2. В `docs/modules/profile.js`, функция `renderHomeHero`, блок формирования `lastUnreviewed` (`profile.js:93-99`): импортировать новые помощники из `./review.js` (рядом с `_loadReviewedIds`, `profile.js:6`); добавить `const dismissedIds = new Set(_loadDismissedRateIds());` и параллельный фильтр `.filter(r => !dismissedIds.has(String(r.id)))` в цепочку (рядом с существующим `.filter(r => !reviewedIds.has(String(r.id)))`).
- **Где (файлы/модули):** `docs/modules/review.js` (новые помощники у `review.js:8-21`); `docs/modules/profile.js` (`profile.js:6` импорт, `profile.js:93-99` фильтр).
- **Контракт:**
  - Новый ключ `localStorage`: **`yc_dismissed_rate_ids`**, значение — JSON-массив строковых id (`String(recordId)`). Отдельный от `yc_reviewed_ids` (не смешивать — Д-1).
  - `yc_reviewed_ids`, `_loadReviewedIds`/`_saveReviewedId`, `submitReview` и поток отзывов — **не меняются**.
  - Фильтр по дисмиссу **параллелен** фильтру по `reviewedIds`, не заменяет его: визит исключается из плашки, если он оценён **или** скрыт.
- **Критерии готовности:**
  - После скрытия и перезагрузки страницы / повторного открытия приложения плашка для того же визита снова не появляется; вместо неё — «Нет предстоящих записей». (BA-020 КП-3)
  - Для нового прошедшего неоценённого визита (другой `recordId`, не в `yc_dismissed_rate_ids`) плашка появляется снова. (BA-020 КП-4)
  - Скрытый визит остаётся неоценённым: его id нет в `yc_reviewed_ids`; кнопка «Оценить» в истории визитов для него доступна и работает. (BA-020 КП-5)
  - При недоступности `localStorage` (исключение чтения/записи) рендер главного экрана не падает, ошибок в консоли нет (`_loadDismissedRateIds()` возвращает `[]`). (BA-020 КП-7)
- **Связь с ADR:** ADR-017, Вопрос N / N1. **BA-020: ФТ-3, ФТ-4, ФТ-5, ФТ-7; КП-3, КП-4, КП-5, КП-7.**

### Задача 2: UI действия скрытия — крестик «×» на карточке плашки

- **Что:** в разметку карточки плашки «Оцените прошлый визит» (`docs/modules/profile.js:101-106`) добавить кнопку-крестик «×»:
  - Расположение: правый верхний угол карточки, **абсолютным позиционированием** поверх контента; самой карточке задать `position:relative`.
  - Тач-зона ≥ 32×32px (визуальный глиф может быть меньше); `type="button"`; `aria-label="Скрыть"`.
  - Крестик **не сдвигает** контент карточки (заголовок «Оцените прошлый визит», название услуги, мастер/дата, кнопка «Оставить отзыв →»). При необходимости — `padding-right`/`max-width` заголовку, чтобы текст не подползал под крестик на узких экранах.
  - Кнопка «Оставить отзыв →» (`profile.js:105`) сохраняется как основное действие; крестик — второстепенное, ненавязчивое.
  - Без подтверждающей модалки.
  - Стили крестика — инлайн (как остальной стиль карточки) либо классом в `docs/style.css`; цвета — через переменные тем (`var(--text-2)` и т.п.), без захардкоженных цветов, ломающих тёмную тему.
- **Где (файлы/модули):** `docs/modules/profile.js:101-106` (разметка карточки); при необходимости `docs/style.css` (класс крестика/тач-зона).
- **Контракт:** добавление крестика не меняет существующую кнопку «Оставить отзыв →», её `data-*`-атрибуты и `onclick="openRateVisit(...)"`. Данные визита, выводимые в плашке (`svcName`, `masterName`, `data-*`), — только через `esc()`, как в текущем рендере (`profile.js:103-105`); крестик пользовательских данных не выводит. Карточка плашки остаётся некликабельной целиком (клик только по крестику и по кнопке отзыва).
- **Критерии готовности:**
  - На плашке есть отдельное явное действие (крестик «×»), скрывающее её **без оставления отзыва** и без перехода на `s-review`. (BA-020 КП-1)
  - Крестик корректно работает на тач-устройствах; зона нажатия достаточна; контент карточки не сдвигается. (BA-020 НФТ-4)
  - Выводимые данные визита экранированы; в консоли нет ошибок при показе плашки. (BA-020 КП-8, НФТ-2)
- **Связь с ADR:** ADR-017, Вопрос O / O1. **BA-020: ФТ-1; КП-1, КП-8.**

### Задача 3: Обработчик скрытия и немедленная перерисовка блока

- **Что:** добавить в `docs/modules/profile.js` новую функцию-обработчик скрытия, экспортируемую в `window` (через `Object.assign(window, {...})` — `profile.js:318`), которая:
  1. принимает `recordId` (передаётся из `onclick` крестика через `data-rid`, как у существующей кнопки отзыва — `profile.js:105`);
  2. вызывает `_saveDismissedRateId(recordId)` (персист дисмисса);
  3. вызывает `renderHomeHero()` — полная перерисовка блока `#homeHero`; на втором проходе фильтр по `dismissedIds` (Задача 1) отсеивает скрытый визит → выводится фолбэк «Нет предстоящих записей» (или другой нескрытый неоценённый визит, если такой есть).
- **Где (файлы/модули):** `docs/modules/profile.js` (новая функция + регистрация в `window` на `profile.js:318`; `onclick` крестика из Задачи 2 указывает на неё).
- **Контракт:** перерисовка — **только** через существующий `renderHomeHero()` (не ручное удаление DOM-узла), чтобы фолбэк «Нет предстоящих записей» отрисовался штатным путём и не возникло второго источника правды. `registerOnEnter('s-home')` и `docs/app.js` — **не меняются**.
- **Критерии готовности:**
  - После нажатия крестика плашка немедленно исчезает, и на её месте отображается карточка «Нет предстоящих записей». (BA-020 КП-2)
  - Действие «Оставить отзыв →» по-прежнему открывает экран отзыва и поток отправки работает; показ предстоящих записей и карточка «Нет предстоящих записей» не затронуты. (BA-020 КП-6)
  - При показе плашки, скрытии, перерисовке и повторных заходах в консоли нет ошибок. (BA-020 КП-8)
- **Связь с ADR:** ADR-017, Вопрос P / P1. **BA-020: ФТ-2, ФТ-6; КП-2, КП-6, КП-8.**

## Изменения публичного контракта

- Внешний API / CLI / YCLIENTS-схемы — **не меняются**. Скрытие не отправляет ничего на бэкенд.
- Новый ключ `localStorage` `yc_dismissed_rate_ids` (массив строковых id) — внутреннее клиентское состояние, отдельное от `yc_reviewed_ids`. Миграционный путь не требуется (новый ключ, отсутствие = `[]`).
- Наблюдаемое поведение UI: на плашке «Оцените прошлый визит» появляется крестик «×»; нажатие скрывает плашку для конкретного визита (персистентно). Поток отзывов и история не меняются.
- **`CACHE_VERSION` / `APP_SHELL`:** изменяются файлы из `APP_SHELL` — `docs/modules/review.js`, `docs/modules/profile.js` и (при правках стилей) `docs/style.css`. Обязательно поднять `CACHE_VERSION` в `docs/sw.js` `v63 → v64`.
- **`docs/version.json`:** мейнтейнер на этапе `/release`/`/deploy` обновляет `build` 30 → 31 и `version` → `v1.15.0` (дату — на день релиза). Версию `package.json` не трогать.

## Тесты

Автотестов в проекте нет (ручная приёмка). Тестировщик проходит и фиксирует сценарии:

- **Скрытие (С1):** на главном при отсутствии предстоящих записей и наличии прошедшего неоценённого визита показывается плашка с крестиком «×»; нажатие крестика → плашка исчезает, появляется «Нет предстоящих записей»; на `s-review` перехода нет; отзыв не отправлен.
- **Персист (С2):** после скрытия — перезагрузка страницы / закрытие и повторное открытие приложения → плашка для того же визита не появляется; в `localStorage` есть `yc_dismissed_rate_ids` с этим id; в `yc_reviewed_ids` этого id нет.
- **Новый визит (С3):** при наличии другого прошедшего неоценённого визита (другой `recordId`) плашка появляется снова.
- **История (С4, ОВ-4):** для скрытого визита кнопка «Оценить» в истории визитов доступна и открывает экран отзыва.
- **Отзыв (С5):** кнопка «Оставить отзыв →» открывает `s-review`, поток отправки работает как прежде.
- **Деградация:** при недоступном `localStorage` (эмуляция исключения) главный экран рендерится без ошибок в консоли.
- **Тач/вёрстка:** крестик удобен на тач; контент карточки не сдвигается; проверка на узком экране и в тёмной теме.
- **PWA:** установка/оффлайн не регрессируют; SW переустановился на новый `CACHE_VERSION` (DevTools → Application → Service Workers — статус active).

## Риски и митигации

- **Риск:** скрытие попадает в `yc_reviewed_ids` → исчезает «Оценить» в истории. → **Митигация:** дисмисс строго в `yc_dismissed_rate_ids`; `yc_reviewed_ids`/`_saveReviewedId` не трогать; фильтр по дисмиссу параллелен фильтру по `reviewedIds`.
- **Риск:** после скрытия блок пуст вместо фолбэка. → **Митигация:** перерисовка через `renderHomeHero()` (P1), не ручное удаление узла.
- **Риск:** глобальное подавление плашки для будущих визитов. → **Митигация:** дисмисс пер-`recordId`; фильтр исключает только конкретный `String(r.id)`.
- **Риск:** недоступность `localStorage` ломает рендер. → **Митигация:** `_loadDismissedRateIds()` → `[]` при ошибке; `_saveDismissedRateId` в `try/catch`.
- **Риск:** крестик сдвигает контент / мелкая тач-зона. → **Митигация:** абсолютное позиционирование поверх карточки; тач-зона ≥ 32×32px; `padding-right` заголовка.
- **Риск:** XSS через данные визита в плашке. → **Митигация:** `svcName`/`masterName`/`data-*` — только через `esc()`.
- **Риск:** старый кэш SW отдаёт прежние файлы. → **Митигация:** поднять `CACHE_VERSION` `v63 → v64`.
