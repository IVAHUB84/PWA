# Спецификация релиза v1.20.0

- **Дата планирования:** 2026-05-30
- **Тип релиза:** minor
- **Цели:** убрать «вебовость» интерфейса — запретить выделение текста во всём UI (кроме полей ввода) и отключить нативное контекстное меню/перетаскивание по изображениям, не ломая ввод в формах и функциональные жесты. Чистый CSS, без новых зависимостей.

> Тип релиза — **minor**: поведенческое улучшение UX без ломающих изменений публичного контракта (API/CLI/схем БД нет; CSS-поведение выделения — не публичный контракт). Осознанная потеря ручного копирования контактов (ОВ-2) принята бизнесом и не считается ломающим изменением контракта.

## Источники

- BA-требования: `ba-req/ba-requirements-025.md`
- ADR: `adrs/adr-022.md`

## Объём релиза

### Задача 1: Глобальный запрет выделения и анти-callout/анти-drag в `style.css`

- **Что:** добавить в `docs/style.css` блок CSS-правил согласно ADR-022, раздел «Решение»:
  1. На `body` — `user-select: none` (+ `-webkit-/-moz-/-ms-` префиксы) и `-webkit-touch-callout: none`.
  2. Исключение `input, textarea, [contenteditable]` — `user-select: text` (+ префиксы) и `-webkit-touch-callout: default` (приоритет над п.1).
  3. Глобально `img` — `-webkit-user-drag: none; user-drag: none; -webkit-touch-callout: none;`.
- **Где (файлы/модули):** `docs/style.css` — добавить блок в начало файла (после строки 9, рядом с глобальными правилами `*`/`html`/`input`), чтобы исключение для полей шло после правила на `body`. Существующие селекторы `.tab` (142), `.pin-key` (819), `.svc-carousel-track` (852-853), `.svc-carousel-slide img` (856) **не удалять и не изменять** (ФТ-5, ФТ-6, ADR-022 §Решение п.4).
- **Контракт:** см. ADR-022, раздел «Решение». Без вендорно-нестандартных хаков; только перечисленные свойства.
- **Критерии готовности:**
  - На `body` присутствует `user-select: none` со всеми префиксами `-webkit-/-moz-/-ms-` и `-webkit-touch-callout: none`.
  - Селектор `input, textarea, [contenteditable]` задаёт `user-select: text` (+ префиксы) и `-webkit-touch-callout: default`; объявлен после правила на `body`.
  - Селектор `img` задаёт `-webkit-user-drag: none; user-drag: none; -webkit-touch-callout: none;`.
  - Существующее правило `.svc-carousel-slide img { pointer-events: none; ... }` (стр. 856) сохранено без изменений.
  - Правок `docs/prototype.html` нет.
- **Связь с ADR:** ADR-022, раздел «Решение» (п.1–5), «Влияние на код».

## Изменения публичного контракта

Нет изменений внешнего API / CLI / схем БД. Поведенческое изменение UI: текст в неинтерактивных и интерактивных элементах (кроме полей ввода) больше нельзя выделить → теряется ручное копирование телефона/адреса/промокода. Миграционный путь не требуется — потеря принята бизнесом (ОВ-2, ADR-022). Поля ввода (`input`/`textarea`/`[contenteditable]`) полностью сохраняют ввод/выделение/вставку.

## Тесты

Автотестов в проекте нет; проверка ручная по критериям приёмки (раздел ниже выполняет тестировщик). Архитектор контролирует лишь физическое наличие изменений CSS. Обязательный минимум проверок для этого релиза (ручных):

- Long-press по кнопкам/карточкам/чипам/иконкам/меткам не выделяет текст (мобильная эмуляция + реальное устройство).
- Поля ввода (OTP `input.otp-box`, телефон, имя, e-mail, отзыв, поля админа push) сохраняют ввод/выделение/копирование/вставку (вкл. вставку кода из SMS).
- Long-press по изображениям на iOS Safari / Android Chrome не открывает callout-меню сохранения.
- Изображения нельзя выделить и нельзя перетащить (на десктопе нет drag-«призрака»).
- Свайп карусели фото услуг, скролл, тапы, press feedback (req-023) — без регрессий.

## Риски и митигации

- **Риск:** исключение для полей не перебивает глобальное правило (специфичность/порядок), ломается ввод в OTP/формах. → **Митигация:** объявить селектор `input, textarea, [contenteditable]` ПОСЛЕ правила на `body`; проверить КП-2 на экране OTP (`prototype.html:406-409`) и формах.
- **Риск:** регресс свайпа карусели при обобщении правил на `img`. → **Митигация:** не трогать `.svc-carousel-slide img` (стр. 856); глобальное `img`-правило добавляет только drag/callout, не `pointer-events`. Проверить КП-5.
- **Риск:** забыто поднятие `CACHE_VERSION` → пользователи получат старый CSS из кэша. → **Митигация:** Задача 2 обязательна; проверка КП-7.
- **Риск:** случайное удаление точечных `user-select`/`user-drag` создаёт регресс. → **Митигация:** существующие селекторы только дополняются, не удаляются (ADR-022 §Решение п.4).
