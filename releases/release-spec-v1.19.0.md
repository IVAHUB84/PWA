# Спецификация релиза v1.19.0

- **Дата планирования:** 2026-05-29
- **Тип релиза:** minor (по содержанию это багфикс CSS; версия задана продуктово как очередной minor-инкремент серии UI-polish после v1.18.0 — публичный контракт не меняется, ломающих изменений нет)
- **Цели:** устранить два визуально заметных мобильных дефекта Studio PWA — самопроизвольное увеличение шрифтов на iOS Safari и откат к desktop-макету «телефон в рамке» при повороте телефона в landscape. После релиза приложение остаётся в стабильном вертикальном мобильном полноэкранном виде в любой ориентации.

## Источники

- BA-требования: `ba-req/ba-requirements-024.md`
- Сырое требование: `requirements/req-mobile-fonts-orientation-024.md`
- ADR: `adrs/adr-021.md`

## Объём релиза

### Задача 1: фиксация масштаба текста (баг 1)

- **Что:** отключить авто-инфляцию шрифтов мобильным Safari, добавив фиксацию масштаба текста на корневом элементе. Закрывает ФТ-1, ФТ-2.
- **Где (файлы/модули):** `docs/style.css` — глобальный блок в начале файла (рядом со строками 1–8, где уже живут `box-sizing`, `touch-action` и защита инпутов).
- **Контракт:** на `html` (корневой элемент) добавляются два свойства:
  - `-webkit-text-size-adjust: 100%;`
  - `text-size-adjust: 100%;`
  Существующее правило `input, textarea, select { font-size: max(16px, 1em); }` (`docs/style.css:8`) **остаётся без изменений**.
- **Критерии готовности:**
  - В `docs/style.css` присутствует правило с `-webkit-text-size-adjust: 100%` и `text-size-adjust: 100%`, применённое к корневому элементу (`html`).
  - Правило `input, textarea, select { font-size: max(16px, 1em); }` сохранено и не изменено.
  - Базовые размеры шрифтов в дизайне не изменены (новое правило только фиксирует масштаб, не задаёт размеры).
- **Связь с ADR:** ADR-021, раздел «Решение» п. 1.

### Задача 2: ориентационно-нейтральный критерий мобильного полноэкранного режима (баг 2)

- **Что:** изменить условие единственного мобильного media-query так, чтобы полноэкранный мобильный режим срабатывал на телефоне в **любой** ориентации (portrait и landscape), но **не** срабатывал на настоящем десктопе. Тело media-query не меняется. Закрывает ФТ-3, ФТ-4, ФТ-5, ФТ-6, ФТ-8.
- **Где (файлы/модули):** `docs/style.css` — мобильный media-query (текущая строка ~623: `@media (max-width:500px) {`).
- **Контракт:** условие media-query меняется с
  `@media (max-width:500px)`
  на
  `@media (max-width: 500px), (max-height: 500px) and (pointer: coarse)`.
  Всё тело media-query (полноэкранный режим `.phone-wrap`/`.phone`, скрытие `.scenario-bar`/`.scenario-hint`/`.step-indicator`, `.status-bar`/`.tab-bar` с `env(safe-area-inset-*)`, `.sticky-bottom`, `.fab`, `.sheet-overlay`, `.sheet*`) остаётся **без изменений**.
- **Критерии готовности:**
  - Условие мобильного media-query соответствует контракту выше (один media-query, составное условие).
  - Тело media-query не изменено (только селектор условия `@media`).
  - На телефоне в landscape (ширина 700–900px, высота < 500px, `pointer: coarse`) активируется мобильный полноэкранный режим — desktop-рамка «телефон» не появляется (ФТ-3, ФТ-4).
  - На настоящем десктопе (ширина > 1000px, `pointer: fine`) сохраняется превью «телефон в рамке»; мобильный режим не активируется ошибочно даже в невысоком десктоп-окне (ФТ-6).
  - Portrait-режим на телефоне визуально идентичен текущему (ФТ-5).
  - Интерфейс в landscape не поворачивается, контент не переверстывается под горизонталь, никакие подсказки/оверлеи не добавляются (ФТ-4, НФТ-5).
- **Связь с ADR:** ADR-021, раздел «Решение» п. 2 (вариант B), ОВ-2/ОВ-3.

### Задача 3: поднятие CACHE_VERSION

- **Что:** так как `docs/style.css` (а это файл из `APP_SHELL`) изменяется, поднять `CACHE_VERSION` в service worker. Закрывает ФТ-9, НФТ-4.
- **Где (файлы/модули):** `docs/sw.js`, строка 1.
- **Контракт:** `const CACHE_VERSION = 'v67';` → `const CACHE_VERSION = 'v68';`
- **Критерии готовности:** `CACHE_VERSION` в `docs/sw.js` равен `'v68'`.
- **Связь с ADR:** ADR-021, раздел «Влияние на код».

### Задача 4: подтверждение PWA/iOS и safe-area (без изменений кода)

- **Что:** подтвердить, что `docs/manifest.json` (`"orientation": "portrait-primary"`) и safe-area-правила остаются корректными — **изменения кода не требуются**, задача фиксирует проверяемое отсутствие регрессий. Закрывает ФТ-7, ФТ-8, ОВ-4/ОВ-5.
- **Где (файлы/модули):** `docs/manifest.json` — без изменений; safe-area-правила в теле media-query `docs/style.css` — без изменений.
- **Контракт:** `manifest.json` остаётся с `"orientation": "portrait-primary"`; `env(safe-area-inset-top/bottom)` правила `.status-bar`/`.tab-bar`/`.sticky-bottom` не трогаются.
- **Критерии готовности:**
  - `docs/manifest.json` не изменён, `"orientation": "portrait-primary"` присутствует.
  - safe-area-правила в `docs/style.css` не изменены; в вертикальном виде (в т.ч. при физическом повороте устройства, где контент остаётся вертикальным) контент не перекрывается вырезом (КП-7).
- **Связь с ADR:** ADR-021, раздел «Решение» п. 3–4.

## Изменения публичного контракта

Нет. Внешний API (YCLIENTS), CLI, схемы данных и манифест PWA не меняются. Меняется только адаптивная вёрстка (CSS) и версия кэша SW. Миграционный путь не требуется — обновление SW (`v67 → v68`) подхватит новый `style.css` штатным механизмом кэширования.

## Тесты

Автотестов в проекте нет (см. `CLAUDE.md`). Минимальный набор ручных проверок, обязательных для этого релиза (выполняет тестировщик на реальном устройстве/эмуляторе iOS Safari — оба бага специфичны для мобильного Safari):

- Текст не увеличивается самопроизвольно в portrait, в landscape и при смене ориентации; размеры соответствуют дизайну (КП-1).
- Поля ввода не вызывают авто-зум iOS при фокусе (КП-2).
- При повороте в landscape desktop-рамка «телефон» не появляется — ни во вкладке браузера, ни в standalone PWA (КП-3).
- В landscape интерфейс не поворачивается, контент не переверстывается, подсказок/оверлеев нет (КП-4).
- Portrait на телефоне визуально идентичен текущему (КП-5).
- Desktop-превью «телефон в рамке» в широком окне (> ~1000px) сохранено, в т.ч. в невысоком десктоп-окне (КП-6).
- Safe-area корректна, контент не перекрыт вырезом (КП-7).
- В standalone ориентация зафиксирована манифестом; на iOS поведение landscape обеспечено CSS (КП-8).
- Консоль без ошибок; PWA устанавливаема; новых рантайм-зависимостей нет; `CACHE_VERSION` поднята (КП-9).

## Риски и митигации

- **Риск:** `(pointer: coarse)` / `max-height: 500px` ведут себя не так на конкретной модели iOS → **Митигация:** обязательная ручная проверка на реальном iOS Safari (НФТ-2); порог и условие изолированы в одном media-query и легко корректируются.
- **Риск:** агрессивное кэширование SW отдаёт старый `style.css` → **Митигация:** bump `CACHE_VERSION` до `v68` (Задача 3); проверка DevTools → Application → Service Workers (status active, новая версия).
- **Риск:** регрессия portrait или desktop-превью из-за изменения условия media-query → **Митигация:** тело media-query не меняется (только селектор); явные проверки КП-5 и КП-6.
- **Риск:** крупный планшет в landscape с высотой > 500px попадёт в desktop-превью → **Митигация:** принято осознанно (планшеты вне scope, BA §2); зафиксировано в ADR-021 «Последствия».
