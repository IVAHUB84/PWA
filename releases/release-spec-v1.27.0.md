# Спецификация релиза v1.27.0

- **Дата планирования:** 2026-05-30
- **Тип релиза:** minor (обратимо-совместимая переработка функциональности)
- **Цели:** Сделать плашку «О студии» достоверным источником контактов: данные
  только из YCLIENTS, честный прочерк при отсутствии, рабочая навигация к
  студии с выбором картографического приложения.

## Источники
- BA-требования: `ba-req/ba-requirements-032.md` (источник
  `requirements/req-studio-contacts-data-031.md`)
- ADR: `adrs/adr-029.md` (вытесняет соответствующую часть `adrs/adr-025.md`)

## Объём релиза

Все задачи реализуют решения ADR-029. Контракт строк, схема данных и
построение ссылок карт описаны в ADR — здесь только что и где сделать.

### Задача 1: Сменить источник данных компании на `/companies?id=`
- **Что:** В `Promise.all` загрузки справочников заменить запрос
  `GET /company/{id}` на `GET /companies?id={id}`. Результат передаётся в
  `setCompanyData()`.
- **Где:** `docs/app.js` (~167-172, элемент `companyRes`).
- **Контракт:** см. ADR-029, «Загрузка endpoint». Сохранить
  `.catch(() => ({}))`.
- **Критерии готовности:** в коде нет обращения к `/company/${YC.company}` для
  плашки; запрос идёт на `/companies?id=${YC.company}`.
- **Связь с ADR:** ADR-029, разделы «Решение → Загрузка endpoint», «Влияние на
  код».

### Задача 2: Новая нормализация и источники данных в `studio.js`
- **Что:** Переписать `setCompanyData(raw)` под нормализованную схему (все
  поля: `title`, `publicTitle`, `address`, `city`, `phone`, `schedule`, `site`,
  `telegram`, `lat`, `lon`); корректно достать единственную компанию из
  ответа `/companies?id=` (объект, `{data:...}` или массив с одним элементом).
  Переписать `getCompanyContacts()` без слияния со `STUDIO` и без построения
  карт-URL из `REVIEW_URLS`: приоритет «память → кэш `yc_company` → пусто».
  Перезаписывать кэш при наличии хотя бы одного непустого поля/координат.
  Чтение старого формата кэша v1.23.0 — мягкая деградация (отсутствующие поля =
  пусто). Удалить импорт `STUDIO`.
- **Где:** `docs/modules/studio.js`.
- **Контракт:** схема данных и правила источников — ADR-029, «Нормализованная
  схема данных», «Источники и оффлайн», «ОВ-5».
- **Критерии готовности:** `getCompanyContacts()` не ссылается на `STUDIO` и
  `REVIEW_URLS`; кэш `yc_company` содержит расширенный набор; при пустом API и
  наличии кэша возвращаются кэшированные значения; `telegram` хранится без
  ведущего `@`.
- **Связь с ADR:** ADR-029, «Решение».

### Задача 3: Новый рендер плашки (состав, прочерки, скрытие соцсетей)
- **Что:** Переписать `renderStudioContacts({variant})`: заголовок
  (`publicTitle`/`title`), адрес (`[city, address]` через запятую), телефон
  (текст + `tel:` при непустом, иначе `—` без ссылки), часы (или `—`), сайт
  (ссылка, текст без схемы/завершающего слэша; скрыть при пустом), Telegram
  (ссылка `https://t.me/<handle>`, текст `@handle`; скрыть при пустом). Метро
  не выводить. Пустые адрес/телефон/часы → `—` с сохранением метки. Иконки у
  строк. `variant: 'confirm' | 'profile'` сохраняется — разметка данных общая,
  отличается только обёртка (как сейчас). Все значения через `esc`; внешние
  ссылки — `target="_blank" rel="noopener"`.
- **Где:** `docs/modules/studio.js` (`renderStudioContacts`); вызовы в
  `docs/modules/slots.js` (~97-98) и `docs/modules/profile.js` (~63-64) не
  меняются по сигнатуре.
- **Контракт:** таблица состава строк — ADR-029, «Состав и правила рендера».
- **Критерии готовности:** соответствует КП-2…КП-6, КП-10 BA-032; в плашке нет
  строки метро; пустые соцсети не рендерятся; пустые адрес/телефон/часы дают `—`.
- **Связь с ADR:** ADR-029, «Состав и правила рендера», «ОВ-2», «ОВ-4».

### Задача 4: Кнопка «Показать на карте» + bottom-sheet выбора приложения
- **Что:** Заменить ссылки «Маршрут» + «2ГИС» одной кнопкой «Показать на
  карте». Кнопка видна при валидных координатах **или** непустом адресе, иначе
  скрыта. По нажатию открывается bottom-sheet (паттерн `#consentModal`) с
  пунктами Apple Maps / Google Maps / Yandex Navigator / Yandex Maps + «Отмена»
  (все 4 всегда). Добавить построитель URL карт по приоритету «координаты →
  адрес». Web-ссылки — `target="_blank" rel="noopener"`; `yandexnavi://` —
  обычный `href`. Закрытие листа — по «Отмена» и клику на фон.
- **Где:** `docs/modules/studio.js` (построитель URL + логика открытия/наполнения
  листа); контейнер листа — `docs/prototype.html`; стили — `docs/style.css`.
- **Контракт:** форматы всех 8 URL (координаты и фолбэк по адресу), правило
  видимости и валидности координат — ADR-029, «Кнопка карты и bottom-sheet»,
  «ОВ-1», «ОВ-3».
- **Критерии готовности:** КП-7, КП-8, КП-9 BA-032; при координатах
  `55.686006, 37.85368` ссылки ведут на эту точку, Navigator строит маршрут;
  без координат, но с адресом — поиск по тексту; без координат и адреса кнопка
  скрыта.
- **Связь с ADR:** ADR-029, «Кнопка карты и bottom-sheet».

### Задача 5: Удалить `STUDIO` как источник из `constants.js`
- **Что:** Удалить экспорт `STUDIO` из `constants.js` (единственный потребитель

- **Связь с ADR:** ADR-029, «Влияние на код».

## Изменения публичного контракта
- **YCLIENTS API:** источник данных компании — `GET /companies?id={id}` вместо
  `GET /company/{id}`. Внешний контракт PWA не меняется.
- **`localStorage` `yc_company`:** расширенная нормализованная схема (добавлены
  `publicTitle`, `city`, `site`, `telegram`; координаты сохраняются). Обратно
  совместимо по чтению: старый формат v1.23.0 читается, отсутствующие поля
  трактуются как пустые. Миграция/сброс ключа не требуется.
- **Внутренний модуль `constants.js`:** удалён экспорт `STUDIO` (ломающее
  изменение только для несуществующих внешних потребителей — внутри проекта
  единственным был `studio.js`).
- **UI-контракт плашки:** одна кнопка «Показать на карте» вместо «Маршрут» +
  «2ГИС»; новые строки Сайт и Telegram; прочерк `—` для пустых полей.

Автотестов в проекте нет — проверки ручные (см. CLAUDE.md). Релиз обязан
обеспечить проверяемость следующих сценариев (тестировщик на этапе `/release`):

- Кнопка «Показать на карте» открывает лист из 4 приложений + «Отмена»; выбор

- Пустые поля показываются прочерком, пустые соцсети скрыты; экран не ломается
  при недоступном API (проверка с кэшем и без).
- Консоль без ошибок; PWA installable (Lighthouse); SW обновился (новая
  `CACHE_VERSION`).

## Риски и митигации
- **Риск:** ответ `/companies?id=` приходит в форме, отличной от ожидаемой
  (массив/обёртка `data`). → **Митигация:** нормализатор в задаче 2 явно
  обрабатывает объект/`{data}`/массив-из-одного; при неудаче — кэш/прочерк.
- **Риск:** app-схема `yandexnavi://` не открывается на desktop. →
  **Митигация:** ожидаемое поведение (BA, допущения); остальные карты работают.
- **Риск:** старый кэш v1.23.0 даёт пустые новые поля до первого онлайн-старта.
  → **Митигация:** мягкая деградация (прочерк/скрытие), кэш перезаписывается при
  первом успешном ответе.
- **Риск:** забыть поднять `CACHE_VERSION` → пользователи на старом SW. →
  **Митигация:** задача 7 в чек-листе и проверке.
