# Спецификация релиза v1.30.0

- **Дата планирования:** 2026-05-31
- **Тип релиза:** minor (обратимо-совместимое улучшение UX экрана `s-slots`; публичный контракт — API, поток бронирования, структура `state` — не меняется)
- **Цели:** Сделать выбор времени удобным и осознанным на мобильном: фиксированный сворачиваемый календарь с drag-handle, явный выбор даты и слота, кнопка «Записаться» только после полного выбора, без заголовка экрана.

## Источники

- BA-требования: `ba-req/ba-requirements-035.md`
- ADR: `adrs/adr-032.md` (вытесняет часть `adrs/adr-031.md` — см. раздел «Пересмотр решений ADR-031»)

## Объём релиза

Все задачи выполняются на экране `s-slots`. Затрагиваемые файлы:
`docs/prototype.html`, `docs/modules/slots.js`, `docs/style.css`,
`docs/modules/booking.js` (при необходимости), `docs/sw.js`,
`docs/version.json`. Решения по реализации зафиксированы в ADR-032 — детали
обоснования не дублируются здесь.

### Задача 1: Фиксация карточки календаря, удаление заголовка, drag-handle (разметка)

- **Что:** Вынести `.cal-card` из общей скролл-области в фиксированную зону;
  внутри `.scroll` оставить только список слотов; удалить заголовок «Выберите
  время»; добавить видимый drag-handle под/у карточки календаря.
- **Где (файлы/модули):** `docs/prototype.html` (`#s-slots`),
  `docs/style.css` (`.page-title`, `.cal-card`, `.scroll`, новые стили
  drag-handle и скролла слотов).
- **Контракт:**
  - `.page-title` «Выберите время» удалён из `#s-slots`.
  - `.cal-card` располагается между `.nav-bar` и `.scroll`, вне `.scroll`
    (не прокручивается вместе со слотами).
  - `.scroll` содержит только `.slots-section` и имеет собственный
    `overflow-y: auto`; список слотов не перекрывает `.sticky-bottom`.
  - Добавлен элемент drag-handle (визуальная «черточка») с атрибутами
    доступности: `role="button"`, `tabindex="0"`, `aria-label`.
- **Критерии готовности:**
  - При прокрутке списка слотов карточка календаря остаётся на месте,
    скроллится только список (КП-1).
  - Заголовка «Выберите время» на экране нет (КП-10).
  - Под/у календаря видна «черточка», по которой понятно, что блок свайпается
    (КП-5).
  - Баннер переноса (`rescheduleBanner`) по-прежнему появляется в сценарии
    переноса (якорь перепривязан, см. Задача 4).
- **Связь с ADR:** ADR-032, «Решение по фиксации календаря» (Вариант А),
  «Влияние на код».

### Задача 2: Два режима календаря и переключение свайпом/кликом

- **Что:** Реализовать развёрнутый (месяц) и свёрнутый (неделя) режимы
  календаря; переключение свайпом по карточке (вверх — свернуть, вниз —
  развернуть) и кликом/тапом по drag-handle.
- **Где (файлы/модули):** `docs/modules/slots.js` (`_renderCalGrid`, новый флаг
  `_calCollapsed`, функция переключения, обработчики touch), `docs/style.css`
  (недельный вид грида, переход по высоте, состояние drag-handle).
- **Контракт:**
  - Модульный флаг `_calCollapsed` (boolean); при входе на экран — `false`
    (развёрнуто).
  - `_renderCalGrid` при `_calCollapsed === true` рисует одну строку Пн…Вс:
    неделю, содержащую `state.dateISO`; если дата не выбрана — текущую неделю
    с выделением «сегодня». availSet для недели собирается из `_calCache`
    (учесть пересечение границы месяца).
  - Функция переключения (например `toggleCalCollapsed`) экспортируется и
    привязана к `onclick` drag-handle и к Enter/Space (доступность).
  - Обработчики `touchstart`/`touchmove`/`touchend` на карточке: свайп вверх →
    `_calCollapsed = true`, свайп вниз → `false`; порог жеста не должен
    срабатывать на обычных тапах по ячейкам дат.
  - В свёрнутом виде выбранный день — тёмный кружок (класс `.sel`);
    «сегодня» при отсутствии выбора — существующий стиль `.today`.
  - Переход между состояниями плавный по высоте (паттерн `.slot-period-body`:
    `max-height` + `transition`); допустима деградация до мгновенного.
  - **ОВ-2:** стрелки «‹ ›» в свёрнутом виде остаются видимыми и меняют месяц
    как прежде (листание по неделям не вводится).
- **Критерии готовности:**
  - При входе календарь развёрнут (полный месяц) (КП-2).
  - Свайп вверх сворачивает до недели, свайп вниз разворачивает до месяца
    (КП-3).
  - В свёрнутом виде показана неделя выбранной даты с тёмным кружком; без
    выбора — текущая неделя с «сегодня» (КП-4).
  - Клик/тап по drag-handle переключает состояние (КП-6).
  - Переход плавный либо мгновенный (КП-13).
  - Свайп не конфликтует со скроллом слотов и выбором даты/слота; прокрутка и
    выбор работают (КП-11).
- **Связь с ADR:** ADR-032, «Решение по недельному виду» (Вариант В),
  «Решения по открытым вопросам BA» (ОВ-1, ОВ-2, ОВ-3).

### Задача 3: Отмена авто-выбора и условный показ кнопки «Записаться»

- **Что:** Убрать авто-выбор даты и слота; показывать `.sticky-bottom` только
  когда выбраны и `state.dateISO`, и `state.slot`.
- **Где (файлы/модули):** `docs/modules/slots.js` (`loadDates`,
  `_autoSelectFirstDate`, `loadTimes`, `_setConfirmBtnEnabled`,
  `updateStickyBottom`), `docs/style.css` (скрытое состояние `.sticky-bottom`).
- **Контракт:**
  - `_autoSelectFirstDate` и его вызовы удалены; при входе `state.dateISO` и
    `state.slot` не назначаются автоматически.
  - В `loadTimes` авто-назначение `firstTime`/`state.slot` убрано; слот
    выбирается только пользователем через `selectSlot`.
  - `_setConfirmBtnEnabled` (переключение `disabled`/`opacity`) удаляется или
    заменяется логикой видимости в `updateStickyBottom`.
  - `updateStickyBottom` показывает блок `.sticky-bottom` (снимает класс
    скрытия / `display`) только при непустых `state.dateISO` И `state.slot`;
    иначе блок скрыт (`display:none` через класс).
  - Санитизация времени до `HH:MM` перед вставкой в `onclick` сохраняется.
  - Подпись кнопки и ветка переноса (`state._rescheduleId` → «Перенести
    запись») сохраняются как в ADR-031.
- **Критерии готовности:**
  - При открытии экрана не выбраны ни дата, ни слот; выбор только по явному
    действию (КП-7).
  - Кнопка «Записаться» отсутствует, пока не выбраны и дата, и время; после
    выбора обоих — появляется (КП-8).
  - Нажатие появившейся кнопки бронирует через `startBooking` → `s-confirm`
    без промежуточного шага; для переноса — подпись и переход как в ADR-031
    (КП-9).
- **Связь с ADR:** ADR-032, «Пересмотр решений ADR-031» (пп. 1–2),
  «Решение по показу кнопки» (Вариант Д).

### Задача 4: Перепривязка якоря баннера переноса

- **Что:** Так как `.page-title` удаляется (Задача 1), баннер переноса
  (`rescheduleBanner`) нужно вставлять к другому якорю.
- **Где (файлы/модули):** `docs/modules/slots.js` (`updateSlotsScreen`),
  `docs/prototype.html` (наличие подходящего якоря в `#s-slots`).
- **Контракт:**
  - В `updateSlotsScreen` вставка `rescheduleBanner` использует существующий
    стабильный якорь (например `.cal-card` или контейнер фиксированной зоны)
    вместо удалённого `.page-title`.
  - Баннер корректно создаётся, обновляется и удаляется при смене
    `state._rescheduleId`.
- **Критерии готовности:**
  - В сценарии переноса баннер «Перенос записи» отображается над/у карточки
    календаря; в обычном сценарии баннера нет; ошибок в консоли нет.
- **Связь с ADR:** ADR-032, «Влияние на код»; ADR-031, Вариант Г.

## Изменения публичного контракта

Внешний контракт не меняется: эндпоинты YCLIENTS (`GET /book_dates/{company}`,
`GET /book_times/{company}/{staffId}/{iso}`), поток `startBooking` → `s-confirm`,
структура `state` (`slot`/`dateISO`/`dateFull`/`_rescheduleId`) сохраняются.
Внутренний контракт UI: при входе на `s-slots` дата и слот не преселектятся;
кнопка действия видима только при полном выборе. Миграция не требуется
(статический фронтенд; свежесть гарантируется bump `CACHE_VERSION`).

## Тесты

Автотестов в проекте нет; проверка ручная по критериям приёмки. Минимальный
набор проверок, который обязан быть зафиксирован в отчёте тестировщика:

- Фиксация календаря: прокрутка слотов не двигает карточку (КП-1).
- Вход: календарь развёрнут, дата/слот не выбраны, кнопки нет (КП-2, КП-7, КП-8).
- Свайп вверх/вниз сворачивает/разворачивает; клик по handle переключает
  (КП-3, КП-6).
- Свёрнутый вид: неделя выбранной даты с тёмным кружком; без выбора — текущая
  неделя с «сегодня» (КП-4).
- Появление кнопки только после выбора даты И слота; бронирование работает
  (КП-8, КП-9).
- Сценарий переноса: подпись «Перенести запись», баннер переноса виден (КП-9).
- Группировка слотов по периодам, фолбэки «нет окон»/ошибка сети — без
  регрессии (КП-12).
- Тёмная тема (токены), `hapticTap` при выборе, экранирование, время `HH:MM`,
  ноль ошибок в консоли, PWA installable, `CACHE_VERSION` поднята (КП-14).

## Риски и митигации

- **Риск:** свайп по календарю конфликтует с тапом по ячейке даты или со
  скроллом слотов. → **Митигация:** порог по смещению/времени в обработчике
  жеста; календарь и список слотов в разных контейнерах (Вариант А).
- **Риск:** недельный вид через границу месяца показывает дни без данных
  доступности. → **Митигация:** availSet собирать из обоих кэш-ключей
  (`_calCache`), для отсутствующего — недоступные ячейки.
- **Риск:** sticky-кнопка остаётся скрытой после ошибки/сброса записи в
  `booking.js`. → **Митигация:** видимость управляется только
  `updateStickyBottom` по `state.dateISO`+`state.slot`; `booking.js` не трогает
  видимость напрямую.
- **Риск:** плавная анимация высоты календаря рывками на части устройств. →
  **Митигация:** допустима деградация до мгновенного переключения (ОВ-3).
- **Риск:** забыли поднять `CACHE_VERSION` → пользователи на старом APP_SHELL.
  → **Митигация:** Задача 5 в чек-листе релиза.
