# Спецификация релиза v1.29.0

- **Дата планирования:** 2026-05-30
- **Тип релиза:** minor
- **Цели:** Привести экран «Выберите время» (`s-slots`) к привычному виду
  мобильного приложения YCLIENTS: полноценный месячный календарь с навигацией по
  месяцам и реальной подгрузкой дат, слоты сгруппированы по периодам суток со
  сворачиваемыми секциями; убрать карточку мастера и стороннее брендирование;
  кнопка действия — «Записаться».

> Обоснование версии (semver): добавление функциональности UI без изменения
> публичного контракта (эндпоинты YCLIENTS, структура `state`, поток
> `startBooking`/`s-confirm` неизменны) → **MINOR**.

## Источники

## Объём релиза

### Задача 1: Месячный календарь-карточка с навигацией и подгрузкой `book_dates`
- **Что:** заменить горизонтальную ленту дат на месячный календарь-карточку:
  заголовок месяца + стрелки «‹ ›», строка дней недели «Пн…Вс», сетка чисел
  (полный месяц, ведущие/хвостовые ячейки пустые). Стрелки переключают месяц и
  реально подгружают доступные даты соседнего месяца через `book_dates`; ответы
  кэшируются в памяти на время жизни экрана (Map по ключу `YYYY-MM`). Состояния
  дат: доступная (кликабельна), недоступная/прошедшая (приглушена,
  некликабельна), выбранная (тёмный заполненный кружок), «сегодня» (подчёркнуто).
  Границы навигации: стрелка «назад» неактивна на текущем месяце; стрелка
  «вперёд» неактивна за горизонтом доступных дат. Месяц без доступных дат —
  приглушённые числа + нейтральная подсказка об отсутствии свободных дат.
- **Где (файлы/модули):**
  - `docs/prototype.html` — разметка `#s-slots`: заменить блок `.date-picker`
    (`<div class="date-picker">…</div>`, строки ~191–196) на разметку
    календаря-карточки (заголовок месяца + стрелки + ряд дней недели + контейнер
    грида чисел, например `#calGrid`).
  - `docs/modules/slots.js` — заменить `_renderDates` / `_updateMonthLabel` /
    `_renderDatesFallback`-логику ленты на: модульное состояние текущего
    отображаемого месяца; функции рендера месячного грида, навигации
    (`prevMonth`/`nextMonth` или эквивалент через `onclick`), кэша `book_dates`.
    Сохранить экспортируемый `selectDate(el, iso)` (или адаптировать с
    сохранением совместимого вызова из разметки/онклика).
  - `docs/style.css` — стили карточки календаря: заголовок, стрелки (активная/
    неактивная), ряд дней недели, грид 7 колонок, ячейка-число с состояниями
    (`доступная` / `недоступная` / `.sel` тёмный кружок / `сегодня` подчёркнуто).
    Только токены `var(--…)`, тёмная тема корректна.
- **Контракт:**
  - Запрос дат: `GET /book_dates/{company}` с параметрами `service_ids` (и
    `staff_id`, если мастер выбран). Ответ: `r.data.booking_dates` — массив
    ISO-дат `YYYY-MM-DD`. Формат и эндпоинт не меняются.
  - Выбор даты вызывает загрузку слотов (`loadTimes(iso)`), пишет
    `state.dateISO` / `state.dateFull` (как сейчас).
- **Критерии готовности:** КП-1, КП-2, КП-3, КП-5 (в части авто-выбора первой
  доступной даты), КП-13 (фолбэки/ошибки/нет окон), КП-15 (тема/хаптик/консоль).
- **Связь с ADR:** ADR-031, Вариант А; раздел «Решение» (ОВ-2 — границы навигации).

### Задача 2: Группировка слотов по периодам суток со сворачиваемыми секциями
- **Что:** в `loadTimes` после санитизации времени разложить слоты по периодам:
  Утро `< 12:00`, День `12:00–16:59`, Вечер `≥ 17:00`. Пустые секции скрывать;
  непустые отрисовывать с заголовком и шевроном, по умолчанию раскрытыми.
  Нажатие на заголовок секции сворачивает/разворачивает её (лёгкая CSS-анимация;
  допустимо мгновенно при сложностях). Состояние раскрытия живёт только в текущем
  DOM экрана, между заходами не сохраняется. Выбранный слот — тёмная заливка.
  Авто-выбор первого доступного слота сохранить.
- **Где (файлы/модули):**
  - `docs/prototype.html` — `#s-slots`: контейнер `#slotsGrid` (строки ~199–201)
    превратить в контейнер секций (либо переименовать/обернуть; селекторы в JS
    обновить соответственно).
  - `docs/modules/slots.js` — `loadTimes`: формировать секции по периодам;
    добавить обработчик сворачивания секции; сохранить санитизацию времени до
    `HH:MM` (`replace(/[^0-9:]/g,'')`) перед вставкой в `onclick` и авто-выбор
    первого слота с `state.slot` / `updateStickyBottom` / `_setConfirmBtnEnabled`.
    Экранировать (`esc()`) любой вставляемый текст.
  - `docs/style.css` — стили секции (заголовок + шеврон), грид слотов внутри
    секции (3 колонки, как сейчас), анимация раскрытия; тёмная тема.
- **Контракт:**
  - Запрос слотов: `GET /book_times/{company}/{staffId}/{iso}` с `service_ids`.
    Ответ: `r.data` — массив `{time|datetime}`. Не меняется.
  - Граница периодов определяется по первым 5 символам `HH:MM`.
- **Критерии готовности:** КП-6, КП-7, КП-8, КП-4 (перезагрузка слотов при выборе
  даты), КП-5 (авто-выбор первого слота), КП-13 (санитизация), КП-15.
- **Связь с ADR:** ADR-031, Вариант В; раздел «Решение» (ОВ-3 — анимация).

### Задача 3: Удаление карточки мастера и брендирования; сохранение баннера переноса
- **Что:** удалить блок `#masterStrip` из `#s-slots`. Убедиться, что надписи
  «Работает на yclients» на экране нет (в текущей разметке `#s-slots` её нет —
  подтвердить отсутствие). Сохранить баннер контекста переноса
  (`#rescheduleBanner`): перепривязать его вставку к существующему якорю экрана
  (например, после `.page-title`), так как прежний якорь `#masterStrip` удаляется.
- **Где (файлы/модули):**
  - `docs/prototype.html` — `#s-slots`: удалить блок
    `<div class="master-select-strip" id="masterStrip">…</div>` (строки ~185–189).
  - `docs/modules/slots.js` — `updateSlotsScreen`: ветка рендера `masterStrip`
    становится неактуальной (обращения уже защищены `if (strip)`); вставку
    `rescheduleBanner` перепривязать к новому якорю (без зависимости от
    `masterStrip`). Логику показа/скрытия баннера по `state._rescheduleId`
    сохранить.
- **Контракт:** структура `state` и сценарий переноса не меняются; баннер
  переноса остаётся функциональным.
- **Критерии готовности:** КП-9, КП-11.
- **Связь с ADR:** ADR-031, Вариант Г.

### Задача 4: Подпись кнопки «Записаться» и сопутствующие тексты
- **Что:** изменить подпись нижней кнопки в обычном сценарии записи на
  «Записаться». Для сценария переноса (`state._rescheduleId`) подпись остаётся
  «Перенести запись» (ОВ-1). Привести сопутствующие тексты процесса/сброса
  («Записываем…» / возврат к подписи) в соответствие, не ломая ветку переноса.
  Поток `startBooking` → `s-confirm` («Запись подтверждена!») сохранить без
  промежуточного шага.
- **Где (файлы/модули):**
  - `docs/prototype.html` — `#s-slots` `.sticky-bottom`: текст кнопки по
    умолчанию «Записаться» (строка ~206).
  - `docs/modules/slots.js` — `updateStickyBottom`: вместо «Подтвердить запись»
    использовать «Записаться»; ветка `state._rescheduleId` → «Перенести запись»
    сохраняется (строка ~62).
  - `docs/modules/booking.js` — `_bookWithSession`: сброс текста кнопки в `finally`
    (строка ~88) — на «Записаться». Убедиться, что для сценария переноса
    (`_rescheduleWithSession`) подпись «Перенести запись» восстанавливается
    корректно (при необходимости — учесть `state._rescheduleId` при выборе текста
    сброса, чтобы не перезаписать кнопку переноса на «Записаться»).
- **Контракт:** поведение `startBooking()` и переход на `s-confirm` неизменны.
  Инфо-строка `#stickyInfo` (дата · слот · услуга) сохраняется над кнопкой.
- **Критерии готовности:** КП-10, КП-12, КП-14.
- **Связь с ADR:** ADR-031, раздел «Решение» (ОВ-1).

## Изменения публичного контракта
Нет. Эндпоинты YCLIENTS (`/book_dates`, `/book_times`, `/book_record`),
структура `state` (`slot` / `dateISO` / `dateFull` / `_rescheduleId`), поток
`startBooking` → `s-confirm` остаются прежними. Изменения — только во внутренней
разметке/стилях/рендере экрана `s-slots`.

## Тесты
Автотестов в проекте нет; проверка ручная (см. CLAUDE.md). Обязательная ручная
проверка по сценариям:
- Открытие `s-slots`: календарь текущего/ближайшего месяца, авто-выбор первой
  доступной даты и первого слота; ноль ошибок в консоли.
- Навигация стрелками «‹ ›»: реальная подгрузка `book_dates` соседнего месяца;
  блокировка «назад» на текущем месяце и «вперёд» за горизонтом; повторное
  листание туда-обратно не делает лишних запросов (кэш).
- Состояния дат: выбранная — тёмный кружок, «сегодня» подчёркнуто, недоступные/
  прошедшие приглушены и некликабельны.
- Слоты: группировка Утро/День/Вечер, пустые секции скрыты, шеврон сворачивает/
  разворачивает; выбранный слот — тёмная заливка; смена даты перезагружает слоты.
- Кнопка «Записаться» → бронирование через API → `s-confirm` «Запись
  подтверждена!»; сценарий переноса (`_rescheduleId`) → кнопка «Перенести запись»
  → «Запись перенесена!».
- Фолбэки: отсутствие данных API, «нет окон», ошибка сети; санитизация времени
  до `HH:MM`.
- Тёмная тема: корректные цвета через токены; тактильный отклик при выборе даты и
  слота.
- PWA: `npm run lint` без ошибок; SW активен; Lighthouse installable не сломан.

## Риски и митигации
- **Риск:** удаление `#masterStrip` ломает вставку баннера переноса
  (`insertAdjacentElement` от `masterStrip`). → **Митигация:** перепривязать
  якорь баннера к `.page-title` экрана `s-slots` (Задача 3).
- **Риск:** в `booking.js` сброс текста кнопки на «Записаться» перезапишет
  подпись «Перенести запись» в сценарии переноса. → **Митигация:** выбирать текст
  сброса с учётом `state._rescheduleId` (Задача 4).
- **Риск:** инъекция через время в `onclick`. → **Митигация:** сохранить
  санитизацию `replace(/[^0-9:]/g,'')` и `esc()` для текста (Задача 2).
- **Риск:** забыли поднять `CACHE_VERSION` → пользователи получают старый экран. →
  **Митигация:** Задача 5 в чек-листе.
- **Риск:** лишние запросы `book_dates` при листании. → **Митигация:** кэш по
  `YYYY-MM` в памяти сессии (Задача 1).
