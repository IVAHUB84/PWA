# Спецификация релиза v1.16.0

- **Дата планирования:** 2026-05-29
- **Тип релиза:** minor
- **Цели:** убрать наложенный значок-сердечко с аватарки избранного мастера в списке выбора мастера; аватар избранного выглядит так же, как у обычного. Все прочие признаки избранности (метка «Избранная», кнопка-переключатель ❤️/🤍, рамка-акцент карточки, сортировка вверх, хранение `yc_favs`) сохраняются без изменений. Точечная UI-полировка в рамках серии релизов по чистке интерфейса.

## Источники

- BA-требования: `ba-req/ba-requirements-021.md` (удалить значок `.fav-badge` с аватара мастера; ОВ-1 закрыт §2 — только значок; ОВ-2 закрыт рекомендацией BA — удалить мёртвый CSS, безопасность подтверждена Д-1).
- ADR: `adrs/adr-018.md` (Q1 — удалить и JS-формирование/вставку `fav-badge` в `docs/modules/masters.js`, и CSS-правило `.fav-badge` в `docs/style.css`; поднять `CACHE_VERSION` `v64 → v65`).

Серверная/YCLIENTS-сторона не задействуется. YCLIENTS-токен не трогается. Ветка `backup` запрещена. Без новых рантайм-зависимостей.

### Задача 1: Удалить формирование и вставку `fav-badge` в `docs/modules/masters.js`

- **Что:**
  1. В `_masterCardHtml` (`docs/modules/masters.js:10-34`) **удалить полностью** строку `const favBadge = m.fav ? '<div class="fav-badge">❤️</div>' : '';` (`masters.js:12`).
  2. В шаблоне аватара карточки (`masters.js:24`) удалить интерполяцию `${favBadge}`: блок должен стать `<div class="master-av" style="background:${m.grad};">${avatarInner}</div>` (без `${favBadge}`).
  3. Прочие строки функции (`last`, `favChip`, `availCls`, `styles`, `styleAttr`, `click`, `initStr`, `avatarInner`, `priceHtml`, шаблон карточки с классом `master-card${m.fav ? ' fav' : ''}`, вывод `favChip` рядом с именем, кнопка-переключатель `toggleFav`, `chevron`) — **не менять**.
- **Где (файлы/модули):** `docs/modules/masters.js` (`_masterCardHtml`, конкретно строки 12 и 24).
- **Контракт:**
  - В DOM карточки мастера в списке выбора мастера элемент `<div class="fav-badge">❤️</div>` **не рендерится никогда** (ни для избранных, ни для обычных).
  - Кнопка-переключатель ❤️/🤍 (`masters.js:31`) и метка `<span class="fav-chip">Избранная</span>` (`masters.js:13,26`) продолжают работать без изменений.
  - Класс карточки `master-card fav` для `m.fav === true` сохраняется (используется стилем `.master-card.fav` для рамки-акцента — `style.css:318`).
  - Поток `toggleFav` и хранение в `localStorage` ключа `yc_favs` — без изменений.
- **Критерии готовности:**
  - В списке выбора мастера у избранного мастера на аватарке **нет** значка-сердечка; аватар визуально не отличается от аватара обычного мастера. (BA-021 КП-1)
  - Метка «Избранная» рядом с именем у избранных мастеров отображается. (BA-021 КП-2)
  - Кнопка-переключатель ❤️/🤍 в карточке списка и в детальной карточке мастера работает: переключение добавляет/убирает мастера из избранного и сохраняется в `localStorage` `yc_favs` через перезагрузку. (BA-021 КП-3, КП-4)
  - Избранные мастера сортируются вверх списка. (BA-021 КП-4)
  - Вёрстка аватара корректна: нет «дыр», смещений и артефактов; аватары с фото и с инициалами отображаются правильно. (BA-021 КП-5)
  - При рендере списка мастеров и переключении избранного в консоли нет ошибок. (BA-021 КП-7)
- **Связь с ADR:** ADR-018, Вопрос Q / Q1. **BA-021: ФТ-1, ФТ-2, ФТ-3; КП-1, КП-2, КП-3, КП-4, КП-5, КП-7.**

### Задача 2: Удалить неиспользуемое CSS-правило `.fav-badge` в `docs/style.css`

- **Что:** удалить **целиком** правило `.fav-badge { position: absolute; bottom: -2px; right: -2px; width: 20px; height: 20px; background: var(--accent); border-radius: 50%; border: 2px solid #fff; display: flex; align-items: center; justify-content: center; font-size: 10px; }` (`docs/style.css:320`). Пустую строку, оставшуюся на месте правила, либо убрать, либо оставить — на усмотрение разработчика (главное — не оставить осиротевшего селектора и не задеть соседние правила).
- **Где (файлы/модули):** `docs/style.css`, строка с правилом `.fav-badge` (`style.css:320`).
- **Контракт:**
  - Селектор `.fav-badge` в `docs/style.css` отсутствует.
  - Соседние правила (`.master-av` `style.css:319`, `.master-info` `style.css:321`, `.master-name`, `.master-role`, `.master-avail`, `.avail-yes`, `.avail-no`, `.master-price`, `.chevron`, `.any-master`, `.fav-chip` `style.css:333`, `.master-card.fav` `style.css:318`) — **не изменяются**.
- **Критерии готовности:**
  - В кодовой базе **не остаётся** ссылок на класс `fav-badge` (ни в `docs/modules/*.js`, ни в `docs/style.css`, ни в `docs/prototype.html`). Грепом по проекту класс встречается только в документах процесса (`requirements/`, `ba-req/`, `adrs/`, `releases/`, `release-notes/`). (BA-021 КП-6)
  - Аватар избранного мастера в браузере без правила выглядит корректно — отсутствие правила не оставляет визуальных артефактов. (BA-021 КП-5)
- **Связь с ADR:** ADR-018, Вопрос Q / Q1. **BA-021: НФТ-2; КП-5, КП-6.**

### Задача 3: Поднять `CACHE_VERSION` в `docs/sw.js`

- **Что:** в `docs/sw.js` (`sw.js:1`) поднять `CACHE_VERSION` `'v64'` → `'v65'`. Прочие строки (`STATIC_CACHE`, `RUNTIME_CACHE`, `APP_SHELL`, `NETWORK_ONLY_HOSTS`, обработчики `install`/`activate`/`fetch`, стратегии SWR) — **не менять**.
- **Где (файлы/модули):** `docs/sw.js`, строка `const CACHE_VERSION = 'v64';`.
- **Контракт:** новые имена кэшей `studio-static-v65` и `studio-runtime-v65` создаются Service Worker'ом автоматически из обновлённого `CACHE_VERSION`. Список `APP_SHELL` не меняется (затронутые файлы `docs/modules/masters.js` и `docs/style.css` уже входят в него — `sw.js:7,21`).
- **Критерии готовности:**
  - После переустановки SW (DevTools → Application → Service Workers → Update / unregister, либо новая вкладка) пользователь получает свежие версии `docs/modules/masters.js` и `docs/style.css`; в DevTools → Application → Cache Storage появляются кэши `studio-static-v65` и `studio-runtime-v65`. (BA-021 НФТ-5, КП-8)
  - PWA остаётся устанавливаемой, оффлайн-поведение не регрессирует. (BA-021 НФТ-5, КП-8)
- **Связь с ADR:** ADR-018, раздел «Влияние на код». **BA-021: НФТ-5, КП-8.**

## Тесты

Автотестов в проекте нет (ручная приёмка). Тестировщик проходит и фиксирует сценарии:

- **С1. Список мастеров (BA-021 КП-1, КП-5):** на экране выбора мастера у избранных мастеров на аватарке **нет** значка-сердечка; аватар выглядит как у обычного мастера; нет «дыр», смещений, артефактов; проверить аватары с фото и аватары с инициалами.
- **С2. Прочие признаки избранного (BA-021 КП-2, КП-3, КП-4):** у избранных мастеров рядом с именем виден `<span class="fav-chip">Избранная</span>`; кнопка-переключатель отображает ❤️ для избранного и 🤍 для обычного; нажатие переключает состояние; избранные мастера отсортированы вверх списка; в `localStorage` обновляется `yc_favs`; после перезагрузки страницы состояние сохраняется.
- **С3. Детальная карточка мастера (BA-021 ФТ-2):** в детальной карточке кнопка-переключатель ❤️/🤍 работает; значка на аватаре нет (и до релиза не было).
- **С4. Отсутствие класса `fav-badge` в кодовой базе (BA-021 КП-6):** грепом по `docs/` класс `fav-badge` не встречается (ни в `modules/`, ни в `style.css`, ни в `prototype.html`); упоминания остаются только в документах процесса (`requirements/`, `ba-req/`, `adrs/`, `releases/`, `release-notes/`).
- **С5. Консоль и прочие экраны (BA-021 КП-7):** при рендере списка мастеров, переключении избранного и переходах в детальную карточку и обратно в консоли нет ошибок; прочие экраны (главная, услуги, слоты, история, профиль) визуально и функционально не затронуты.
- **С6. PWA (BA-021 КП-8):** после публикации DevTools → Application → Service Workers показывает active SW с обновлёнными кэшами `studio-static-v65`/`studio-runtime-v65`; установка PWA проходит; оффлайн-открытие приложения работает.

## Риски и митигации

- **Риск:** где-то ещё используется класс `fav-badge` (не учтённый потребитель). → **Митигация:** факт единственного использования подтверждён грепом по проекту (Д-1 BA-021); проверка повторяется ревьюером как С4.
- **Риск:** вёрстка аватара ломается после удаления абсолютно позиционированного потомка. → **Митигация:** `fav-badge` не участвовал в потоке (`position:absolute`); ручная проверка С1 с фото и инициалами.
- **Риск:** пользователь не различит избранного мастера. → **Митигация:** сохранены метка, рамка карточки, ❤️ в кнопке и сортировка — четыре независимых признака; С2.
- **Риск:** старый кэш SW отдаёт прежние `masters.js`/`style.css` → значок остаётся на экране. → **Митигация:** Задача 3 — поднять `CACHE_VERSION` `v64 → v65`; проверка С6.
- **Риск:** неожиданная регрессия логики избранного при правке `_masterCardHtml`. → **Митигация:** правки строго локальны строкам 12 и 24; ревьюер проверяет diff на соответствие границам ADR-018.
