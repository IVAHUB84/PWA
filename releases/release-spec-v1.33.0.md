# Спецификация релиза v1.33.0

- **Дата планирования:** 2026-05-31
- **Тип релиза:** minor (обратимо-совместимое добавление a11y-функциональности; публичные сигнатуры `ui.js` неизменны, видимое поведение для мыши/тача без изменений)
- **Цели:** сделать основной клиентский поток проходимым с клавиатуры и понятным скринридеру, корректным для пользователей с вестибулярной чувствительностью, без регрессий для мыши/тача.

## Источники
- BA-требования: `ba-req/ba-requirements-038.md` (источник `requirements/req-accessibility-pass-037.md`)
- ADR: `adrs/adr-035.md`; справочно `adrs/adr-034.md` (примитивы `ui.js`)

## Объём релиза

### Задача 1: Esc-закрытие и фокус-ловушка sheet-примитивов
- **Что:** в `ui.js` добавить приватные хелперы `_mountSheet(overlay, { onClose, returnFocusTo? })` и `_dismissSheet`, подключённые в `confirmSheet`/`errorSheet`. Поведение: Esc вызывает тот же путь закрытия, что клик по фону/«Отмена»/«Закрыть» (`confirmSheet` → resolve `false`); фокус при открытии переходит на первую кнопку `.ui-sheet`; Tab/Shift+Tab циклит внутри `.ui-sheet`; при закрытии фокус возвращается на элемент, активный до открытия; `document`-слушатель снимается при закрытии (стек активных оверлеев — Esc закрывает верхний).
- **Где (файлы/модули):** `docs/modules/ui.js`.
- **Контракт:** публичные сигнатуры `toast`/`confirmSheet`/`errorSheet`/`setOfflineBanner` **не меняются** (см. ADR-034 §«Нормативный контракт», ADR-035 §«Нормативный контракт изменений»). Добавляются только приватные хелперы.
- **Критерии готовности:** `confirmSheet` и `errorSheet` закрываются по Esc; фокус заходит в sheet при открытии и возвращается на исходный элемент при закрытии; дублирующих sheet-реализаций не создано.
- **Связь с ADR:** ADR-035, Вопрос 1 (1A).

### Задача 2: Esc и фокус для оверлея выбора категории
- **Что:** в `services.js:_openCatFilter` добавить Esc-закрытие и возврат фокуса; строки категорий `.s-row` сделать достижимыми (`role="button"`+`tabindex="0"`+доступное имя). Стиль и разметка оверлея визуально не меняются.
- **Где:** `docs/modules/services.js` (≈31-59).
- **Контракт:** оверлей закрывается по Esc и кликом по фону (как сейчас); `_pickCat` работает с клавиатуры (Enter/Space).
- **Критерии готовности:** оверлей категорий закрывается по Esc; строки выбираются с клавиатуры; вид оверлея не изменился.
- **Связь с ADR:** ADR-035, Вопрос 1 (1A), Вопрос 2 (2A).

### Задача 3: Перенос фокуса при смене экрана
- **Что:** экспортировать чистую функцию `focusTargetSelector(screenEl) → HTMLElement | null` (заголовок `.page-title`/`.section-title` внутри экрана, иначе сам контейнер; без вызова `.focus()`). В `go()` после завершения перехода перенести фокус `focusTargetSelector(toEl)?.focus({ preventScroll: true })`, **только** если активный элемент ещё не внутри `toEl` (чтобы не перебить точечный `.focus()` `auth.js`/`review.js`). Заголовкам/контейнерам при необходимости проставить `tabindex="-1"`.
- **Где:** `docs/modules/navigation.js` (`go()` и блок `setTimeout` с `onEnterScreen`); `docs/prototype.html` (`tabindex="-1"` где нужно).
- **Контракт:** `focusTargetSelector` — экспорт, чистая, без side-effects.
- **Критерии готовности:** после каждого `go()` фокус на заголовке нового экрана (или контейнере при отсутствии заголовка); `.focus()` в `auth.js`/`review.js` не сломаны.
- **Связь с ADR:** ADR-035, Вопрос 3 (3A).

### Задача 4: Конвертация критичных клиентских div[onclick]
- **Что:** критичные клиентские `div[onclick]` (tab-bar `go(...,'tab')`, карточки услуг/мастеров, строки, слоты, `.any-master`, hero-карты) сделать достижимыми: карточки/строки → `role="button"`+`tabindex="0"`+доступное имя (`aria-label` или текстовое содержимое), смысловые иконочные кнопки → `<button>` с классом-сбросом, сохраняющим вид. Зарегистрировать единый делегированный keydown-обработчик на корне (`#phone`): Enter/Space на `[role="button"]` → `el.click()` (Space с `preventDefault`). Существующие `onclick` сохраняются.
- **Где:** `docs/prototype.html` (статичные элементы, tab-bar); `docs/modules/services.js`, `docs/modules/masters.js`, `docs/modules/slots.js` (динамические шаблоны рендера карточек/строк/слотов); делегированный обработчик — `docs/app.js` или `docs/modules/navigation.js`.
- **Контракт:** активация Enter/Space выполняет то же действие, что клик мышью; стили карточек/строк/tab-bar не меняются.
- **Критерии готовности:** поток «логин → услуга → мастер → слот → бронь» полностью проходим только клавиатурой (Tab/Shift+Tab/Enter/Space); все критичные элементы получают фокус и активируются; вид элементов не изменился.
- **Связь с ADR:** ADR-035, Вопрос 2 (2A), Вопрос 6 (6A).

### Задача 5: aria-label иконочным кнопкам клиентского потока
- **Что:** иконочным кнопкам без текстовой подписи (кнопки навигации/действий в клиентском потоке: назад, фильтр категории, drag-handle календаря, колокольчик и т.п. в рамках клиентского пути) проставить непустой `aria-label`, описывающий действие.
- **Где:** `docs/prototype.html`; при динамическом рендере — соответствующие модули (`services.js`, `slots.js`, `masters.js`).
- **Контракт:** каждый иконочный интерактив клиентского потока имеет осмысленное доступное имя.
- **Критерии готовности:** в DevTools Accessibility tree у иконочных кнопок клиентского потока непустое доступное имя.
- **Связь с ADR:** ADR-035 §«Влияние на код».

### Задача 6: prefers-reduced-motion — заглушение анимаций
- **Что:** экспортировать чистый предикат `prefersReducedMotion()` в `utils.js`. Расширить блок `@media (prefers-reduced-motion: reduce)` (`style.css:1169`): `.screen { transition: none }`, scale-`@keyframes` (`pop`, `tapPulse`, `inboxBadgePulse`) → `animation: none`, карусельный трек `transition: none !important`. В `navigation.js:setTx` — при `prefersReducedMotion()` всегда мгновенный переход. В карусели `services.js` — при `prefersReducedMotion()` снап слайда без `transition`. Функциональность сохраняется, исчезает только движение.
- **Где:** `docs/modules/utils.js`, `docs/style.css` (≈1169), `docs/modules/navigation.js` (`setTx`), `docs/modules/services.js` (карусель).
- **Контракт:** `prefersReducedMotion() → boolean`, чистая обёртка над `matchMedia`.
- **Критерии готовности:** при эмуляции `prefers-reduced-motion: reduce` переходы экранов, карусель и scale-эффекты мгновенны; навигация и смена слайдов работают.
- **Связь с ADR:** ADR-035, Вопрос 4 (4A).

### Задача 7: focus-visible focus-ring в обеих темах
- **Что:** глобальные правила в `style.css`: `:focus-visible { outline: 2px solid var(--focus-ring); outline-offset: 2px; }` для `button, a, [role="button"], [tabindex], input, textarea, select`; `:focus:not(:focus-visible) { outline: none; }`. Токен `--focus-ring` уже задан в обеих темах.
- **Где:** `docs/style.css`.
- **Контракт:** ring при Tab, отсутствует при клике мышью/тачем; различим в светлой и тёмной теме.
- **Критерии готовности:** focus-ring виден при Tab, не виден после клика мышью; читаем в обеих темах.
- **Связь с ADR:** ADR-035, Вопрос 5 (5A).

### Задача 8: Подъём CACHE_VERSION
- **Что:** поднять `CACHE_VERSION` `v89`→`v90` в `docs/sw.js`. Состав `APP_SHELL` не расширяется (новых файлов нет), стратегии SW не меняются.
- **Где:** `docs/sw.js` (строка 1).
- **Контракт:** PWA остаётся installable.
- **Критерии готовности:** `CACHE_VERSION === 'v90'`; ноль ошибок в консоли; SW активен.
- **Связь с ADR:** ADR-035 §«Влияние на код»; CLAUDE.md (подъём при изменении `APP_SHELL`-файлов).

## Изменения публичного контракта
- Внешний API/CLI/схемы БД: без изменений.
- Сигнатуры `ui.js` (`toast`, `confirmSheet`, `errorSheet`, `setOfflineBanner`): **неизменны** (требование НФТ-2/КП-10).
- Новые экспорты (внутренние, для приложения и тестов): `navigation.focusTargetSelector`, `utils.prefersReducedMotion`. Миграционный путь не требуется — только добавление.

## Тесты
Минимальный набор автотестов (чистая логика, без DOM/таймеров) обязан появиться:
- `focusTargetSelector(screenEl)`: возвращает заголовок при его наличии; возвращает контейнер при отсутствии заголовка; не вызывает `.focus()`.
- `prefersReducedMotion()`: возвращает булево согласно мок-результату `matchMedia` (инжектируемому/мокаемому).
- (Если логика Esc/выбора активного оверлея sheet вынесена в чистую функцию) тест выбора цели закрытия по Esc без DOM.

Ручная проверка (тестировщик): КП-1…КП-10 из `ba-req/ba-requirements-038.md` — клавиатурная проходимость потока, Esc оверлеев/sheet, перенос фокуса, `aria-label`, `aria-live`, reduced-motion, focus-visible, отсутствие регрессий для мыши/тача, тех. целостность, отсутствие дублирования.

## Риски и митигации
- **Риск:** замена `div`→`<button>` ломает flex/grid верстку карточек. → **Митигация:** для карточек/строк использовать `role="button"`+`tabindex` (вариант 2A), `<button>` — только для смысловых кнопок с классом-сбросом; визуальная проверка КП-8.
- **Риск:** `document`-keydown sheet копится/не снимается. → **Митигация:** снятие слушателя в `_dismissSheet`; стек активных оверлеев.
- **Риск:** перенос фокуса в `go()` перебивает точечный `.focus()` `auth.js`/`review.js`. → **Митигация:** перенос только если активный элемент не внутри нового экрана; КП-3.
- **Риск:** inline-`transition` карусели/экрана игнорирует CSS reduced-motion. → **Митигация:** JS-гейт `prefersReducedMotion()` в `setTx`/карусели + `!important` для трека.
- **Риск:** focus-ring «висит» после клика. → **Митигация:** `:focus:not(:focus-visible){outline:none}`; КП-7.
