# Спецификация релиза v1.14.0

- **Дата планирования:** 2026-05-29
- **Тип релиза:** minor
- **Цели:** довести раздел услуг до качества: (1) починить листание/геометрию/зацикливание карусели фото на детальной карточке, не меняя её внешний вид; (2) дать «мягкие приподнятые» рамки карточкам каталога и слайдам карусели (свет/тёмная тема); (3) убрать «мёртвую» кнопку «Назад» на экране услуг при входе как корневой раздел.

## Источники

- BA-требования: `ba-req/ba-requirements-017.md` (фикс трёх дефектов карусели — смещение/обрезка, залипание свайпа, рывок при зацикливании), `ba-req/ba-requirements-018.md` (рамки `.svc-catalog-card` и `.svc-carousel-slide`), `ba-req/ba-requirements-019.md` (скрытие кнопки «Назад» на `s-services`).

## Объём релиза

Все задачи строго в пределах объёма BA-017/018/019 (раздел 2 каждого) и границ ADR-016. **Не трогаются:** источник галереи фото (ADR-014 G1), SWR-кэш фото (J1), визуальные параметры галереи как дизайн (H1 — высота 280px, наличие peek/точек/скруглений сохраняются), логика `go`/`back`/`navHistory` (`navigation.js`), прочие карточки (`.master-card`, `.hist-visit`, `.cs-card`, `.confirm-card`, `.hero-card`), прочие экраны и общий класс `back-btn`, поток записи (выбор мастера/слотов/бронирование/подтверждение). YCLIENTS-токен не трогается. Ветка `backup` запрещена. Без новых рантайм-зависимостей.

### Задача 1: Фикс механики и геометрии transform-карусели на детальной карточке `s-service`

- **Что:** устранить три дефекта реализации I1 (ADR-014), сохранив внешний вид v1.12.0 (peek, точки, высота 280px, скругление). Меняется только механика/геометрия:
  1. **Смещение/обрезка, неравномерный peek:** геометрия (`slideWidth`, peek, `translateForIdx`) и первичные `setPos(0,false)` + `updateDots()` считаются/вызываются **после** того, как экран `#s-service` стал видимым/`active` (не синхронно до `go('s-service')`). «Магические» константы `44` (CSS `calc(100% - 44px)`) и `8` (`SLIDE_GAP` ↔ `gap:8px`) синхронизировать с CSS единым источником истины (вычислять от фактической ширины контейнера/слайда; устранить рассинхрон JS↔CSS).
  2. **Пересчёт на resize/повороте:** добавить пересчёт позиции текущего `logIdx` (без анимации) на `resize`; слушатель снимать при ре-рендере карточки/уходе с экрана (не накапливать хендлеры при повторных открытиях).
  3. **Залипание свайпа:** подряд идущие осмысленные свайпы не теряются (допустимо завершать/фиксировать текущий переход перед применением следующего жеста; без полноценного инерционного движка). Порог и различение направления (`|dx|` vs `|dy|`) сохранить так, чтобы явные горизонтальные свайпы не отсекались.
  4. **Рывок при зацикливании:** бесшовный сброс позиции на реальный слайд — в одном месте по завершению анимации, с форсированным reflow перед возвратом `transition`; устранить гонку `transitionend` vs сторож `setTimeout(…,500)` (идемпотентный `settle()`, сторож — только страховка).
- **Где (файлы/модули):** `docs/modules/services.js` (`openServiceCard`, ветка `photos.length >= 2`, `services.js:167-308`); при необходимости `docs/style.css:822-841` (только синхронизация числовых значений геометрии, без изменения визуальных параметров); при выборе хука видимости — `docs/app.js` (добавить/использовать `registerOnEnter('s-service', …)` или эквивалент после `go('s-service')`).
- **Контракт:** ветки `photos.length === 1` (одиночное фото), `photos.length` 0 (плейсхолдер) и деградация по `onerror` — **не меняются**. Внешний вид галереи (скругление, peek, точки, высота 280px/`object-fit:cover`) визуально совпадает с v1.12.0. URL фото и `alt` — только через `esc()`, в т.ч. в клонах слайдов. Без новых рантайм-зависимостей и сторонних карусель-библиотек.
- **Критерии готовности:**
  - При 2/3/8 фото слайды выровнены, peek равномерен слева и справа **сразу при открытии карточки** (без скролла/поворота). (BA-017 КП-1, КП-5)
  - Свайп надёжно листает на одно фото в направлении жеста; серия свайпов подряд не теряется; нет застревания между слайдами. (BA-017 КП-2)
  - Переход «последний → первый» и «первый → последний» бесшовен — без рывка/перескока/мигания; активная точка соответствует фото после стыка. (BA-017 КП-3, КП-4)
  - После поворота/resize слайды остаются выровнены, peek равномерен, позиция пересчитана. (BA-017 КП-6)
  - 1 фото / 0 фото / битый URL ведут себя как прежде. (BA-017 КП-7, КП-8)
  - В консоли нет ошибок при открытии/свайпе/зацикливании/повороте. (BA-017 КП-10)
- **Связь с ADR:** ADR-016, Вопрос K / K1. **BA-017: ФТ-1…ФТ-9; КП-1…КП-13.**

### Задача 2: «Мягкие приподнятые» рамки карточек каталога и слайдов карусели

- **Что:** применить утверждённый стиль рамок (BA-018 §4, ОВ-1) к двум селекторам:
  - `.svc-catalog-card`: `border-radius: 18px` (в базовом правиле); сохранить `border: 1px solid var(--border)`; `box-shadow: 0 4px 16px rgba(0,0,0,.08)` — задать в блоке «CARD POLISH» **вместо** ослабленной `0 1px 4px rgba(0,0,0,0.05)` (строка `.svc-catalog-card`, `style.css:693`).
  - `.svc-carousel-slide`: `border-radius: 18px`; `box-shadow: inset 0 0 0 1px rgba(0,0,0,.06)` (внутренняя обводка поверх фото, **не** `border` — чтобы не сдвинуть `flex-basis`/peek).
  - `[data-theme="dark"] .svc-carousel-slide`: `box-shadow: inset 0 0 0 1px rgba(255,255,255,.08)` (светлая полупрозрачная обводка, чтобы край читался на тёмном фоне).
- **Где (файлы/модули):** `docs/style.css` — `.svc-catalog-card` базовое правило (`~275`, radius), строка `.svc-catalog-card` блока «CARD POLISH» (`693`, тень), `.svc-carousel-slide` (`834-838`, radius + inset), новый/дополненный селектор `[data-theme="dark"] .svc-carousel-slide`. Разметка `docs/modules/services.js` — **не меняется**.
- **Контракт:** строки `.master-card`, `.hist-visit`, `.cs-card`, `.confirm-card`, `.hero-card` в «CARD POLISH» (`style.css:694-698`) — **не трогать**. Не нарушать `overflow:hidden`, `aspect-ratio:16/9` обложки, `height/object-fit:cover` фото, `:active scale(0.97)` карточки и геометрию карусели BA-017 (inset поверх фото, `flex:0 0 calc(100% - 44px)` без изменений). Бордер карточки — через `var(--border)`, без захардкоженных цветов, ломающих тёмную тему. Новых hover/selected-рамок не вводить.
- **Критерии готовности:**
  - На каталоге `.svc-catalog-card` имеют радиус 18px, бордер `var(--border)` и мягкую тень `0 4px 16px rgba(0,0,0,.08)`; край аккуратный, не «плоский». (BA-018 КП-1)
  - На детальной карточке слайды `.svc-carousel-slide` имеют радиус 18px и видимую тонкую обводку. (BA-018 КП-2)
  - Рамки контрастны и аккуратны в светлой **и** тёмной теме (обводка слайда видна в тёмной). (BA-018 КП-3)
  - Обрезка/скругление фото обложки и слайдов сохранены; `aspect-ratio` и высота фото не нарушены. (BA-018 КП-4)
  - Геометрия карусели не нарушена: peek равномерен на 2/3/8 фото (сверка с Задачей 1). (BA-018 КП-5)
  - `:active scale(0.97)` карточки сохранён; прочие карточки/экраны визуально не изменились. (BA-018 КП-6, КП-7)
  - В консоли нет ошибок; скролл каталога и свайп без лагов. (BA-018 КП-8)
- **Связь с ADR:** ADR-016, Вопрос L / L1. **BA-018: ФТ-1…ФТ-7; КП-1…КП-10.**

### Задача 3: Динамическое скрытие кнопки «Назад» на `s-services`

- **Что:** кнопка «‹ Назад» в шапке `s-services` видна ⇔ `navHistory.length >= 2` (вариант «а», BA-019 ОВ-1):
  - вход как таб/корень (`go('s-services','tab')`, `navHistory.length < 2`) — кнопка **скрыта**;
  - вход push «Записать себя» (`go('s-services')`, `navHistory.length >= 2`) — кнопка **видна и работает** (ведёт на главный).
  - Заголовок «Выберите услугу» **не прыгает** при наличии/отсутствии кнопки (стабильная высота шапки — приём на усмотрение разработчика: `visibility:hidden` с сохранением высоты строки либо зарезервированная высота `nav-bar`).
- **Где (файлы/модули):** `docs/app.js` — дополнить существующий `registerOnEnter('s-services', …)` (`app.js:36-48`) переключением видимости кнопки по `navHistory.length >= 2`; импортировать `navHistory` из `docs/modules/navigation.js` (уже экспортируется, `navigation.js:4`). `docs/prototype.html:105` — дать кнопке «Назад» / её `.nav-bar` на `s-services` адресуемый селектор (id); при необходимости — стартовое скрытое состояние. При необходимости `docs/style.css` — правило стабильной высоты шапки/скрытия. `docs/modules/navigation.js` — **не менять**.
- **Контракт:** правка касается **только инстанса** кнопки на `s-services`; общий класс `back-btn` и его поведение/стиль на других экранах — без изменений. Логика `go`/`back`/`navHistory` — без изменений. Срабатывает при обоих входах (хук `onEnterScreen` вызывается и в `tab`-, и в push-, и в `back`-ветках навигации).
- **Критерии готовности:**
  - Вход через таб «Услуги» / FAB «Запись» — кнопки «Назад» нет; заголовок без смещения. (BA-019 КП-1, КП-2)
  - Вход через «Записать себя» — кнопка есть и возвращает на главный. (BA-019 КП-4, КП-5)
  - Кнопка «Назад» на других экранах (детальная карточка, мастера, слоты, бронирование, OTP, профиль, лента, история) работает и выглядит как прежде. (BA-019 КП-3)
  - При обоих входах, переключении табов и переходах — в консоли нет ошибок. (BA-019 КП-6)
- **Связь с ADR:** ADR-016, Вопрос M / M1. **BA-019: ФТ-1…ФТ-5; КП-1…КП-7.**

## Тесты

Автотестов в проекте нет (ручная приёмка). Спецификация требует, чтобы тестировщик прошёл и зафиксировал следующие сценарии:

- **Карусель (017):** на услуге с 2, 3 и 8 фото — равномерный peek сразу при открытии; надёжное листание свайпом, серия свайпов подряд; бесшовный стык в обе стороны; активная точка после стыка; поворот/resize; 1 фото / 0 фото / битый URL; ноль ошибок в консоли.
- **Рамки (018):** каталог и детальная карточка в светлой и тёмной теме — радиус 18px, тень карточки, видимая обводка слайда (в т.ч. на тёмном фоне); сохранены обрезка фото, `aspect-ratio`, `:active`; прочие карточки не изменились.
- **Кнопка «Назад» (019):** вход через таб «Услуги», через FAB «Запись», через «Записать себя» — корректная видимость; заголовок без «прыжка»; кнопка «Назад» на других экранах работает.
- **PWA:** установка/оффлайн не регрессируют; SW переустановился на новый `CACHE_VERSION` (DevTools → Application → Service Workers — статус active).

## Риски и митигации

- **Риск:** 017 — геометрия снова измеряется до видимости экрана → неравномерный peek. → **Митигация:** измерять/`setPos` после `active`/видимости `#s-service`; проверка на 2/3/8 фото при открытии.
- **Риск:** 017 — рывок из-за гонки `transitionend` vs сторож. → **Митигация:** идемпотентный `settle`, сторож как страховка, форсированный reflow перед возвратом `transition`.
- **Риск:** 017 — накопление `resize`-слушателей при повторных открытиях. → **Митигация:** снимать слушатель при ре-рендере/уходе с экрана.
- **Риск:** 018 — `border` на слайде ломает peek. → **Митигация:** только `inset box-shadow`, `flex-basis` не менять; сверка peek с Задачей 1.
- **Риск:** 018 — новая тень карточки перекрыта «CARD POLISH». → **Митигация:** задать тень в строке `.svc-catalog-card` блока «CARD POLISH».
- **Риск:** 019 — затронут общий `back-btn`/логика навигации. → **Митигация:** адресовать только инстанс `s-services`; `navigation.js` не править.
- **Риск:** старый кэш SW отдаёт прежние файлы. → **Митигация:** поднять `CACHE_VERSION` `v62 → v63`.
