# Спецификация релиза v1.25.0

- **Дата планирования:** 2026-05-30
- **Тип релиза:** minor (обратимо-совместимое добавление функциональности: drill-down-шиты в админке; новых ломающих изменений контракта нет; `_telHref` становится экспортируемым — расширение, не слом).
- **Цели:** сократить путь администратора от «вижу запись/клиента в PWA» до «позвонил клиенту» до одного-двух тапов: строки записей и клиентов становятся кликабельными и открывают bottom-sheet с деталями и кнопкой «Позвонить».

## Источники

- BA-требования: `ba-req/ba-requirements-030.md`
- ADR: `adrs/adr-027.md`

## Объём релиза

### Задача 1: Экспорт хелпера нормализации телефона `_telHref`

- **Что:** сделать существующий приватный `_telHref(phone)` экспортируемым, чтобы переиспользовать
  единую нормализацию `tel:` в `admin.js` (без дублирования регэкспа).
- **Где (файлы/модули):** `docs/modules/studio.js` — функция `_telHref` (строки 75–77).
- **Контракт:** `export function _telHref(phone)` возвращает `'tel:' + String(phone||'').replace(/[^+\d]/g,'')`.
  Поведение и сигнатура не меняются — добавляется только `export`. Существующее использование внутри
  `studio.js` (`renderStudioContacts`) продолжает работать.
- **Критерии готовности:** `_telHref` импортируется в `admin.js` без ошибок; внутренний вызов в `studio.js` не сломан.
- **Связь с ADR:** ADR-027, раздел «Решение» (нормализация телефона) и «Влияние на код».

### Задача 2: Единый каркас bottom-sheet `_openSheet` в admin.js

- **Что:** добавить локальный хелпер, создающий overlay по каноническому паттерну проекта и принимающий
  готовое (уже экранированное) тело шита.
- **Где (файлы/модули):** `docs/modules/admin.js` — новая функция `_openSheet(bodyHtml)`.
- **Контракт:** строит `div` с `style.cssText = 'position:fixed;inset:0;background:rgba(0,0,0,0.45);z-index:9999;display:flex;align-items:flex-end;'`,
  `dataset` для адресации (напр. `data-admin-sheet="1"`), внутренняя панель
  `background:var(--bg);width:100%;border-radius:24px 24px 0 0;padding:20px 20px 32px;max-width:393px;margin:0 auto;`
  с «ручкой» `36×4px` (как в `history.js:30–58`); закрытие тапом по фону через
  `overlay.addEventListener('click', e => { if (e.target === overlay) overlay.remove(); })`;
  явная кнопка «Закрыть» внутри панели; `document.body.appendChild(overlay)`. Перед открытием —
  удалить предыдущий шит, если открыт.
- **Критерии готовности:** при вызове открывается затемнённый overlay со скруглённой панелью снизу;
  тап по фону и тап по кнопке «Закрыть» убирают overlay; после закрытия `document.body` не содержит
  остаточного узла шита; прокрутка страницы и нижняя навигация работают как прежде.
- **Связь с ADR:** ADR-027, раздел «Решение» (Вариант B), «Контекст» (канонический паттерн `history.js`).

### Задача 3: Кликабельные строки записей и шит записи (Сводка)

- **Что:** сделать `.booking-row` кликабельными; по тапу открывать шит с деталями записи и кнопкой «Позвонить».
- **Где (файлы/модули):** `docs/modules/admin.js` — `renderAdminDashboard` (строки 87–98) + новая обёртка
  `_openBookingSheet(index)`; экспорт обёртки в `window`.
- **Контракт:**
  - В `renderAdminDashboard` сохранить отрисованный массив записей в модульном состоянии (по образцу
    `_adminAllClients`), чтобы `_openBookingSheet` доставал `rec` по индексу без новых запросов (Огр-3).
  - Каждая строка получает `data`-индекс и `onclick`, вызывающий `_openBookingSheet`.
  - Тело шита содержит экранированные поля: имя клиента, услуга, мастер, время, стоимость; отсутствующие
    поля — плейсхолдер «—».
  - Если `rec.client.phone` непуст — блок «Позвонить»: ссылка `<a href="${esc(_telHref(phone))}">` +
    отображаемый номер `${esc(phone)}` в элементе с классом доступности (Задача 6). Если телефон пуст/отсутствует —
    блок звонка не выводится (graceful degradation).
- **Критерии готовности:** КП-1, КП-2 из BA-030 — тап по строке открывает шит с пятью полями; при непустом
  телефоне есть рабочая кнопка «Позвонить» (`tel:` с нормализованным номером, без сетевых запросов); при пустом
  телефоне кнопки нет, остальное отображается; ошибок в консоли нет.
- **Связь с ADR:** ADR-027, разделы «Решение», «Влияние на код».

### Задача 4: Кликабельные строки клиентов и шит клиента (База)

- **Что:** сделать `.client-row` кликабельными; по тапу открывать шит с деталями клиента и кнопкой «Позвонить».
- **Где (файлы/модули):** `docs/modules/admin.js` — `_renderClientRows` (строки 299–321) + новая обёртка
  `_openClientSheet(index)`; экспорт обёртки в `window`.
- **Контракт:**
  - Каждая `.client-row` получает `data`-индекс (индекс в массиве, переданном в `_renderClientRows`) и `onclick`,
    вызывающий `_openClientSheet`. `_openClientSheet` достаёт клиента из того же массива, который был отрисован
    последним (учесть, что после `filterAdminClients` рендерится отфильтрованный список — индекс должен ссылаться
    на актуально отрисованный набор).
  - Тело шита: экранированные имя, телефон, число визитов; блок «Позвонить» (`tel:` через `_telHref`) при наличии телефона.
- **Критерии готовности:** КП-3, КП-6 из BA-030 — тап по строке клиента открывает шит с именем, телефоном и числом
  визитов и рабочей кнопкой «Позвонить»; поиск клиентов продолжает работать, кликабельность сохраняется после
  фильтрации и повторного рендера.
- **Связь с ADR:** ADR-027, разделы «Решение» (кликабельность строк, совместимость с `filterAdminClients`), «Влияние на код».

### Задача 5: Экранирование и безопасность данных в шитах

- **Что:** все пользовательские данные (имя, телефон, услуга, мастер) выводятся только через `esc()`.
- **Где (файлы/модули):** `docs/modules/admin.js` — тела `_openBookingSheet` / `_openClientSheet`.
- **Контракт:** любые подстановки из API проходят через `esc()`; `href` ссылки `tel:` формируется как
  `esc(_telHref(phone))`. Внешних ссылок в этом релизе нет; если появятся — `rel="noopener"` (ФТ-10).
- **Критерии готовности:** КП-7 — попытка инъекции через имя/телефон/услугу не приводит к исполнению разметки;
  новых рантайм-зависимостей нет.
- **Связь с ADR:** ADR-027, раздел «Контекст» (безопасность).

### Задача 6: CSS-исключение для выделяемого/копируемого телефона в шите

- **Что:** добавить класс доступности телефона, снимающий глобальный `user-select:none` точечно для номера в шите.
- **Где (файлы/модули):** `docs/style.css` — новый класс (напр. `.sheet-phone`), по образцу `.studio-contacts`
  (`style.css:1102–1106`). При необходимости — `cursor:pointer` для `.booking-row`/`.client-row` (визуальный признак кликабельности).
- **Контракт:** класс задаёт `user-select:text` (+ `-webkit-`/`-moz-`/`-ms-`) и `-webkit-touch-callout:default`;
  телефон читаем в светлой и тёмной теме (используются токены `--text`/`--text-2`/`--primary`, не хардкод цвета,
  кроме согласованного с темами).
- **Критерии готовности:** КП-4 — телефон в обоих шитах можно выделить и скопировать и одновременно нажать для звонка.
- **Связь с ADR:** ADR-027, раздел «Решение» (доступность телефона); ADR-024 (образец `.studio-contacts`).

### Задача 7: Подъём CACHE_VERSION

- **Что:** поднять версию кэша service worker, т.к. изменяются файлы из `APP_SHELL` (`admin.js`, `style.css`).
- **Где (файлы/модули):** `docs/sw.js` — `CACHE_VERSION` (`'v79'` → `'v80'`).
- **Контракт:** строковая константа в начале файла.
- **Критерии готовности:** КП-8 — после деплоя клиенты получают обновлённые `admin.js`/`style.css`;
  PWA остаётся устанавливаемой; в консоли нет ошибок.
- **Связь с ADR:** ADR-027, раздел «Влияние на код»; BA-030 НФТ-4.

## Изменения публичного контракта

- **JS-модуль `studio.js`:** `_telHref` становится экспортируемым (`export`). Обратимо-совместимое расширение —
  существующие импортёры не затрагиваются, внутреннее использование сохраняется.
- **`window`-API:** добавляются глобальные обёртки открытия шитов (`_openBookingSheet`, `_openClientSheet`)
  для `onclick`-вызовов из разметки строк — по сложившемуся в `admin.js` стилю.
- **Внешний API / CLI / схема БД / YCLIENTS:** без изменений. Новых сетевых запросов нет (Огр-3). Миграции не требуются.

## Тесты

Автотестов в проекте нет (см. CLAUDE.md). Обязательная ручная проверка (тестировщик), покрывающая критерии BA-030:

- Сводка: тап по записи → шит с клиентом/услугой/мастером/временем/стоимостью; отсутствующие поля как «—» (КП-1).
- Сводка: запись с телефоном → рабочая кнопка «Позвонить» (`tel:`, корректный номер, без сетевых запросов);
  запись без телефона → кнопки нет, шит корректен (КП-2).
- База: тап по клиенту → шит с именем/телефоном/числом визитов + рабочая «Позвонить» (КП-3).
- Оба шита: телефон выделяется и копируется и кликается для звонка (КП-4).
- Закрытие шита тапом по фону и кнопкой «Закрыть»; прокрутка и tab-bar после закрытия работают (КП-5).
- Поиск клиентов работает; кликабельность строк сохраняется после фильтрации (КП-6).
- Проверка экранирования: клиент/услуга с разметкой в имени не исполняется (КП-7).
- DevTools → Console: ноль ошибок; Application → Service Workers: active; `CACHE_VERSION` поднята (КП-8).
- Действие «Открыть в YCLIENTS» отсутствует — это соответствие решению ОВ-2, не дефект (КП-9).

## Риски и митигации

- **Риск:** клик по строке клиента конфликтует с поиском (`filterAdminClients` пере-рендерит строки) →
  **Митигация:** кликабельность задаётся в шаблоне строки в `_renderClientRows`, индекс ссылается на актуально
  отрисованный (отфильтрованный) массив; `filterAdminClients` не трогается по логике, только результат рендера.
- **Риск:** шит ломает прокрутку страницы / перекрывает tab-bar → **Митигация:** overlay — отдельный
  `position:fixed;inset:0`-слой поверх UI, удаляется при закрытии; страница под ним не модифицируется
  (без `overflow:hidden` на body — по образцу `history.js`).
- **Риск:** XSS через имя/телефон/услугу из API → **Митигация:** обязательный `esc()` на всех подстановках,
  `href` через `esc(_telHref(...))` (Задача 5).
- **Риск:** забыли поднять CACHE_VERSION → пользователи на старом `admin.js`/`style.css` → **Митигация:** Задача 7
  в чек-листе; проверка в Application → Service Workers.
- **Риск:** индекс записи устаревает при повторном рендере дашборда → **Митигация:** шит открывается из
  текущего отрисованного списка; модульное состояние перезаписывается при каждом `renderAdminDashboard`.
