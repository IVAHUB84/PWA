# Спецификация релиза v1.34.0

- **Дата планирования:** 2026-05-31
- **Тип релиза:** minor (обратимо-совместимое добавление: оформление больших экранов, hover, zoom, выделение текста, условный iOS-хром, десктоп-install; публичные контракты не ломаются)
- **Цели:** сделать приложение визуально завершённым и удобным на десктопе/планшете и при работе мышью, не ухудшив мобильный сценарий и не нарушив a11y (v1.33.0) и in-app-фидбэк (v1.32.0).

## Источники
- BA-требования: `ba-req/ba-requirements-039.md` (источник `requirements/req-cross-platform-layout-038.md`)
- ADR: `adrs/adr-036.md`
- Не конфликтовать: `adrs/adr-034.md` (контракты `ui.js`), `adrs/adr-035.md` (a11y)

## Объём релиза

Принят вариант объёма **«оформленное устройство»** (ADR-036, Вопрос 1, вариант 1A). Полный многоколоночный адаптив — вне объёма (техдолг).

### Задача 1: Снять блокировку зума
- **Что:** убрать `maximum-scale=1.0` и `user-scalable=no` из метатега viewport.
- **Где:** `docs/prototype.html` (стр. 5).
- **Контракт:** `<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">`.
- **Критерии готовности:** Ctrl+= / Ctrl+− меняют масштаб в десктоп-браузере; pinch-zoom работает на тач; iOS не делает auto-zoom при фокусе на инпут (обеспечено существующим `font-size: max(16px,1em)`, `style.css:8`); ноль ошибок в консоли.
- **Связь с ADR:** ADR-036, Вопрос 6 (6A).

### Задача 2: Чистая функция определения iOS-standalone + класс на корень
- **Что:** добавить чистую функцию `isIosStandalone({ standalone, iosPlatform }) → boolean` (= `standalone && iosPlatform`, без обращения к DOM/`navigator` внутри) и хелпер сбора флагов (читает `isStandalone()` + платформенные признаки без основного UA-снифинга, UA — только фолбэк); поставить класс `ios-standalone` на корневой элемент при инициализации, если хелпер вернул `true`.
- **Где:** `docs/modules/install.js` (экспорт функций), точка инициализации (`docs/app.js` или существующий вызов `initInstall`) — установка класса на корень.
- **Контракт:**
  - `export function isIosStandalone({ standalone, iosPlatform })` — чистая, возвращает булево.
  - Хелпер сбора флагов (например `detectIosStandalone()`) использует существующий `isStandalone()` и платформенные признаки (`navigator.platform`/`navigator.maxTouchPoints`, `isIOS(ua)` как фолбэк); при неоднозначности — `false` (в пользу скрытия хрома).
  - Существующие экспорты `isStandalone`, `isIOS`, `shouldOffer`, `triggerInstall` и поведение install-флоу **не меняются**.
- **Критерии готовности:** функция чистая (детерминирована от аргументов); класс `ios-standalone` присутствует на корне только в iOS-standalone, отсутствует на десктопе/в браузере; UA не является основным механизмом.
- **Связь с ADR:** ADR-036, Вопрос 2 (2A), раздел «Нормативный контракт».

### Задача 3: Скрыть фейковый iOS-хром вне iOS
- **Что:** схлопнуть высоту `.status-bar` (убрать пустую полосу 59px) вне `ios-standalone`; узел и дети (`.time`, `.dynamic-island`) остаются в DOM. При наличии класса `ios-standalone` — текущее поведение, включая мобильное safe-area-переопределение.
- **Где:** `docs/style.css` (правила `.status-bar`, разделы у стр. 158-165 и мобильный медиа стр. 763). Разметку `prototype.html:38-39` **не трогать**.
- **Контракт:** вне `.ios-standalone` — `.status-bar { height:0; min-height:0; padding:0 }`; внутри — без изменений.
- **Критерии готовности:** на десктопе/в браузере над шапкой нет пустой полосы и нет «9:41»/островка; на iOS-standalone статус-бар с safe-area отображается как прежде; смена ориентации корректна.
- **Связь с ADR:** ADR-036, Вопрос 3 (3A).

### Задача 4: Оформление «устройства» на больших экранах
- **Что:** на ширинах > 500px оформить окружение `.phone` брендово — фон-градиент (вместо плоского `#E4DDD8`), осознанное центрирование, рамка/тень «устройства»; контент остаётся читаемым и центрированным. Многоколоночные сетки не делать.
- **Где:** `docs/style.css` (`body` стр. 78-86, `.phone`/`.phone-wrap` стр. 127-138; новые правила вне мобильного медиа).
- **Контракт:** только токены тем (`var(--bg/--surface/--border/--accent/--accent-light/--shadow/...)`); новые токены — в обе темы (`:root` и `[data-theme="dark"]`). Мобильный медиа (`style.css:755+`) не трогать.
- **Критерии готовности:** на десктопе/планшете приложение не выглядит «мелким телефоном-прототипом»; пустое пространство оформлено брендово; обе темы корректны; мобильный вид (≤500px) визуально не изменился.
- **Связь с ADR:** ADR-036, Вопрос 1 (1A), ОВ-4.

### Задача 5: Hover-состояния для мыши
- **Что:** добавить hover-реакцию для интерактивных элементов только при точном указателе.
- **Где:** `docs/style.css` — новый блок `@media (hover:hover) and (pointer:fine)`.
- **Контракт:** селекторы `.svc-catalog-card`, `.master-card`, `.s-row`, `.slot`, `.tab`, `.hero-card`, `.any-master`, смысловые `button`; hover-эффект через токены тем (фон/тень/`brightness`); НЕ перекрывать `:focus-visible` (`style.css:88-102`) и `:active` press-эффекты; layout не сдвигать.
- **Критерии готовности:** при наведении мыши на десктопе видна реакция; на тач-устройстве hover не «залипает» после тапа; обе темы корректны.
- **Связь с ADR:** ADR-036, Вопрос 4 (4A).

### Задача 6: Ревизия выделения текста
- **Что:** сделать контентный текст выделяемым по умолчанию; `user-select:none` оставить только на «кнопочных»/служебных элементах.
- **Где:** `docs/style.css` (`body` стр. 11-17; точечные правила; сохранить уже выделяемые блоки стр. 18-24, 1218-1221, 1340-1343; `.tab` стр. 185, `.pin-key` стр. 946 и т.п.).
- **Контракт:** база — выделяемый текст (цены, описания, адрес, телефон, имена мастеров, заголовки); `user-select:none` — точечно на: `.tab`/`.tab-bar`, `.pin-key`, кнопках (`button`, `[role="button"]` действий), иконках, FAB, drag-handle, служебных индикаторах. Inputs/textarea/contenteditable не трогать (уже `text`). Финальный перечень «кнопочных» классов — в пределах этой политики (ОВ-3).
- **Критерии готовности:** на десктопе выделяются и копируются цена, адрес, телефон, имя мастера, описание услуги; «кнопочные»/служебные элементы не выделяются.
- **Связь с ADR:** ADR-036, Вопрос 5 (5A), ОВ-3.

### Задача 7: Десктоп-install affordance
- **Что:** на десктопе (Chrome/Edge, есть `deferredPrompt`) показывать install-affordance с нейтральной формулировкой; iOS-инструкции Safari («Поделиться» → «На экран Домой») — только на iOS.
- **Где:** `docs/modules/install.js` (условие/формулировка), при необходимости тексты `docs/prototype.html` (баннер стр. 70-72).
- **Контракт:** формулировка не использует мобильно-ориентированный «Добавить на экран»/iOS-overlay на десктопе (например «Установить приложение»); `installOverlay` (`prototype.html:907`, заголовок «Установите приложение») остаётся нейтральным; iOS-overlay (`s-install`, шаги Safari) — только при iOS. Публичные функции install не меняются (только условие показа/тексты).
- **Критерии готовности:** в Chrome/Edge на десктопе affordance/текст адекватны платформе; на iOS — прежние инструкции Safari.
- **Связь с ADR:** ADR-036, Вопрос 7 (7A).

### Задача 8: Поднять CACHE_VERSION
- **Что:** изменены файлы из `APP_SHELL` (`prototype.html`, `style.css`, `install.js`, возможно `app.js`) — поднять версию кэша. Состав `APP_SHELL` не расширяется.
- **Где:** `docs/sw.js` (стр. 1).
- **Контракт:** `CACHE_VERSION = 'v91'` (было `v90`). Стратегии SW не меняются.
- **Критерии готовности:** `CACHE_VERSION` = `v91`; `APP_SHELL` без новых записей.
- **Связь с ADR:** ADR-036, раздел «Влияние на код».

## Изменения публичного контракта
- **Viewport:** снят зум-лок (поведенческое изменение для пользователя, не API).
- **`install.js`:** добавлены экспорты `isIosStandalone` и хелпер сбора флагов; существующие экспорты и поведение неизменны (аддитивно).
- **CSS-классы:** новый корневой класс `ios-standalone` (внутренний, не публичный API).
- Миграция не требуется: localStorage, API YCLIENTS, схемы данных не затрагиваются.

## Риски и митигации
- **Риск:** ревизия `user-select` пропустит контентный текст или случайно сделает выделяемой кнопку → **Митигация:** база `text` + явный список «кнопочных» классов; ручная проверка КП-6 по всем перечисленным типам контента.
- **Риск:** hover-правила задевают мобильный/тач вид → **Митигация:** строго внутри `@media (hover:hover) and (pointer:fine)`; КП-4 на тач-устройстве.
- **Риск:** эвристика iOS-определения ложно скрывает/показывает хром на iPadOS → **Митигация:** политика «в пользу скрытия» при неоднозначности; КП-2 на iOS-standalone и десктопе.
- **Риск:** изменение оформления `.phone` ломает мобильный фуллскрин-кейс → **Митигация:** мобильный медиа (`style.css:755+`) не трогается; новые правила вне него; КП-8.
- **Риск:** снятие зум-лока вернёт iOS auto-zoom на инпутах → **Митигация:** уже предотвращён `font-size: max(16px,1em)`; проверка на iOS/эмуляции.
- **Риск:** не поднят `CACHE_VERSION` → у пользователей старый shell → **Митигация:** Задача 8 + КП-10 в ревью.
