# Спецификация релиза v1.23.0

- **Дата планирования:** 2026-05-30
- **Тип релиза:** minor (обратимо-совместимое добавление функциональности — новый
  контактный блок; публичный контракт приложения не ломается)
- **Цели:** показать клиенту контакты студии (адрес, маршрут, телефон, часы)
  сразу после брони на `s-confirm` и постоянно в профиле `s-profile`. Источник —
  гибрид YCLIENTS `/company` (приоритет непустых полей) + плейсхолдеры `STUDIO`.

## Источники
- BA-требования: `ba-req/ba-requirements-028.md`
- ADR: `adrs/adr-025.md`

## Объём релиза

### Задача 1: объект дефолтов `STUDIO`
- **Что:** добавить экспортируемый объект `STUDIO` с плейсхолдерами контактов.
- **Где:** `docs/modules/constants.js` — после `REVIEW_URLS` (после строки 33).
- **Контракт:** объект `STUDIO` с полями `title`, `address`, `lat`, `lon`,
  `phone`, `schedule`, `mapYandex`, `map2gis`. Значения:
  `title='Реснички, сестрички'`, `address='Троицкий бульвар, 4, Троицк, Москва'`,
  `lat=55.466536`, `lon=37.289422`, `phone='+7 495 123-45-67'`,
  `schedule='Ежедневно 10:00–21:00'`, `mapYandex=REVIEW_URLS.yandex`,
  `map2gis=REVIEW_URLS['2gis']`.
- **Критерии готовности:** телефон сопровождается комментарием-меткой
  плейсхолдера (напр. `// ПЛЕЙСХОЛДЕР — заменить на реальный телефон`); объект
  ссылается на `REVIEW_URLS` (объявлен после него в том же файле).
- **Связь с ADR:** ADR-025, ОВ-1.

### Задача 2: модуль `studio.js` (нормализация, кэш, рендер)
- **Что:** новый модуль: нормализация ответа `/company`, кэш в `localStorage`,
  слияние с `STUDIO`, единая функция-рендер контактного блока, построение ссылок
  карт.
- **Где:** `docs/modules/studio.js` (новый файл).
- **Контракт (публичные экспорты):**
  - `setCompanyData(raw)` — принимает сырой ответ `/company` (или `{}`),
    нормализует контактные поля; если есть хотя бы одно непустое контактное поле —
    сохраняет нормализованный объект в `localStorage` под ключом `yc_company` и в
    модульное состояние.
  - `getCompanyContacts()` — возвращает объект с финальными значениями для
    отображения, по приоритету **свежий API → `localStorage` (`yc_company`) →
    `STUDIO`**: `{ title, address, phone, schedule, lat, lon, mapYandex, map2gis }`.
  - `renderStudioContacts(opts)` — возвращает HTML-строку блока;
    `opts.variant ∈ {'confirm','profile'}` переключает обёртку (компактная
    карточка vs секция «О студии» в стиле `settings-group`). Внутренние строки
    контактов общие для обоих вариантов.
- **Слияние полей (приоритет API над плейсхолдером):** «непустое» строковое поле =
  после `trim()` не пустое; координаты берутся из API только если `lat` и `lon`
  оба численно ≠ 0; телефон — первый непустой из `phones[]`, иначе `phone`.
- **Карты:** «Маршрут» → Я.Карты: при координатах из API
  `https://yandex.ru/maps/?pt=<lon>,<lat>&z=17&l=map`, при плейсхолдере —
  `mapYandex`. Опциональная ссылка 2ГИС → `map2gis`. Обе:
  `target="_blank" rel="noopener"`.
- **Экранирование:** все динамические строки (название, адрес, телефон, часы)
  через `esc` из `modules/utils.js`. Телефон в `href="tel:"` — нормализовать к
  допустимым символам (`+`, цифры) перед подстановкой в href.
- **Критерии готовности:** функции экспортированы; нет циклических импортов
  (`studio.js` импортирует только из `constants.js` и `utils.js`); при пустом/
  отсутствующем `/company` `getCompanyContacts()` возвращает плейсхолдеры; при
  непустых полях — данные API.
- **Связь с ADR:** ADR-025, ОВ-3, ОВ-4, ОВ-5, ОВ-6.

### Задача 3: загрузка `/company` на старте
- **Что:** добавить `GET /company/{YC.company}` в общий `Promise.all` старта и
  передать результат в `setCompanyData`.
- **Где:** `docs/app.js`, `initApp()` — `Promise.all` на строке ~166 (рядом с
  `book_services`); импорт `setCompanyData` из `./modules/studio.js`.
- **Контракт:** запрос с `.catch(() => ({}))`; результат передаётся в
  `setCompanyData(res)`. Отрисовка контактного блока не блокируется и не зависит
  от успеха запроса.
- **Критерии готовности:** при недоступном `/company` старт не падает, консоль
  без необработанных ошибок; данные доступны через `getCompanyContacts()` к
  моменту отрисовки экранов.
- **Связь с ADR:** ADR-025, ОВ-2, ОВ-6.

### Задача 4: блок на экране подтверждения `s-confirm`
- **Что:** под карточкой записи (`#confirmRows`) показать компактный контактный
  блок, не трогая кнопки «На главную» и «Добавить услугу к записи».
- **Где:** контейнер в `docs/prototype.html` (после `#confirmRows`, ~226, перед
  кнопкой «На главную» ~227); наполнение в `docs/modules/slots.js` →
  `updateConfirmScreen()` (после формирования `rows.innerHTML`, ~94) вызовом
  `renderStudioContacts({variant:'confirm'})`.
- **Контракт:** блок содержит название, адрес, кнопку «Маршрут», кликабельный
  телефон (`tel:`), часы. Существующие кнопки и поток брони/кросс-селла не
  меняются.
- **Критерии готовности:** КП-1, КП-3, КП-4. Блок виден под карточкой; кнопки
  работают как прежде.
- **Связь с ADR:** ADR-025, ОВ-3.

### Задача 5: секция «О студии» в профиле `s-profile`
- **Что:** добавить секцию «О студии» в стиле существующих секций профиля
  (`label` + `settings-group`).
- **Где:** `docs/prototype.html` `s-profile` — внутри `.scroll`, перед блоком
  «Выйти из аккаунта» (~364) либо после «Аккаунт»/настроек (контейнер ~324–344);
  наполнение в `docs/modules/profile.js` → `renderProfileScreen()` вызовом
  `renderStudioContacts({variant:'profile'})`.
- **Контракт:** тот же набор контактов, что на `s-confirm`; оформление
  консистентно с `settings-group`/`label`.
- **Критерии готовности:** КП-2. Секция присутствует и стилистически совпадает с
  соседними секциями.
- **Связь с ADR:** ADR-025, ОВ-3.

### Задача 6: локальное исключение из user-select
- **Что:** разрешить выделение/копирование текста внутри контактного блока
  поверх глобального `user-select:none`.
- **Где:** `docs/style.css` — новый класс `.studio-contacts` (или равнозначный
  селектор блока) с `user-select:text` + вендорные префиксы +
  `-webkit-touch-callout:default`, по образцу `input, textarea`
  (`docs/style.css:18-23`). Класс назначается обёртке блока в `studio.js`.
- **Контракт:** глобальное правило `body { user-select:none }` (строки 11-17) не
  изменяется; стили блока используют CSS-токены, без хардкода цвета.
- **Критерии готовности:** КП-5 — телефон и адрес выделяются и копируются.
- **Связь с ADR:** ADR-025, раздел «Локальное исключение».

### Задача 7: регистрация модуля в Service Worker и версии
- **Что:** добавить новый модуль в кэш приложения и поднять версии.
- **Где:** `docs/sw.js` — добавить `'./modules/studio.js'` в массив `APP_SHELL`
  (после строки 35, рядом с другими модулями); поднять `CACHE_VERSION`
  (`'v76'` → `'v77'`, строка 1). `docs/version.json` — инкремент `build` и `date`
  (обновляет мейнтейнер на этапе релиза).
- **Контракт:** все APP_SHELL-файлы перечислены; `CACHE_VERSION` поднята при
  изменении `prototype.html`/`style.css`/`app.js`/модулей.
- **Критерии готовности:** КП-10 (часть про `CACHE_VERSION`); новый модуль
  доступен оффлайн.
- **Связь с ADR:** ADR-025, «Влияние на код»; CLAUDE.md (правило `CACHE_VERSION`).

## Тесты
Автотестов в проекте нет — проверка ручная (CLAUDE.md). Обязательный набор
проверок этого релиза:
- `s-confirm`: после брони виден контактный блок (название, адрес, «Маршрут»,
  телефон, часы); кнопки «На главную» и «Добавить услугу к записи» работают.
- `s-profile`: секция «О студии» присутствует и оформлена как соседние секции.
- Тап по телефону → системный звонок (`tel:`); тап «Маршрут» → открывается
  Я.Карты в новой вкладке; внешние ссылки имеют `rel="noopener"`.
- Выделение и копирование телефона/адреса работает (несмотря на глобальный
  `user-select:none`).
- При непустых полях `/company` отображаются данные API; при пустом ответе —
  плейсхолдеры; при оффлайн-старте с ранее закэшированным `yc_company` —
  закэшированные данные.
- DevTools → Console: ноль ошибок; Application → Service Workers: статус active,
  `CACHE_VERSION` поднята; PWA installable (Lighthouse).
- `npm run lint` — без новых ошибок.

## Риски и митигации
- **Риск:** запрос `/company` замедлит старт. → **Митигация:** запрос в общем
  `Promise.all` с `.catch`, отрисовка блока от него не зависит (плейсхолдеры).
- **Риск:** инъекция через серверные строки (название/адрес/часы). →
  **Митигация:** `esc` для всех динамических значений; телефон в `tel:` —
  очистка до `+` и цифр.
- **Риск:** локальное `user-select:text` непреднамеренно включит выделение шире
  блока. → **Митигация:** селектор ограничен классом обёртки блока.
- **Риск:** забыть поднять `CACHE_VERSION` → пользователи получат старый кэш. →
  **Митигация:** явный пункт в Задаче 7 и чек-листе.
- **Риск:** ссылка на карту по координатам из API имеет неверный формат. →
  **Митигация:** при плейсхолдере использовать готовый `REVIEW_URLS.yandex`;
  формат для API-координат зафиксирован в ADR-025 (ОВ-4).
