# Спецификация релиза v1.22.0

- **Дата планирования:** 2026-05-30
- **Тип релиза:** minor
- **Цели:** Сделать тёмную тему читаемой и доступной (WCAG AA) — устранить
  «светлое на светлом», адаптировать акцент, синхронизировать theme-color,
  поднять контраст вторичного текста, вернуть видимый фокус с клавиатуры.

> Тип релиза — **minor**: публичный контракт (URL, схемы данных, поведение
> светлой темы) не ломается; добавляется/исправляется поведение тёмной темы
> и фокуса. Это не patch, т. к. меняется наблюдаемое поведение theme-color и
> добавляются новые CSS-токены.

## Источники
- BA-требования: `ba-req/ba-requirements-027.md`
- ADR: `adrs/adr-024.md`
- Исходное требование: `requirements/req-dark-theme-uiux-027.md`

## Утверждённые значения (из ADR-024)
Использовать как есть. При реализации проверять итоговый контраст каждой пары
«токен/фон» в DevTools или WebAIM на порог AA (текст ≥ 4.5:1, UI ≥ 3:1); при
недоборе — двигать значение в ту же сторону, не меняя семантику токена.

| Токен | Light | Dark |
|---|---|---|
| `--accent` | `#8B3558` (без изменений) | `#E58AAB` |
| `--accent-light` | без изменений | `#4A2433` |
| `--on-accent-light` (новый) | `#8B3558` | `#F2A8C2` |
| `--on-accent` (новый) | `#FFFFFF` | `#3A0E1E` |
| `--text-2` | `#8E8E93` (без изменений) | `#AEAEB6` |
| `--on-primary` (новый) | `#FFFFFF` | `#1C1C1E` |
| `--focus-ring` (новый) | `#8B3558` | `#E58AAB` |

theme-color: light `#FAF7F3`, dark `#1C1C1E`. `color-scheme: light dark`.

## Объём релиза

### Задача 1: Семантические токены вместо хардкода `color:#fff`
- **Что:** Ввести токены `--on-primary` и `--focus-ring` в `:root` (light) и
  `[data-theme="dark"]`. Заменить жёсткий `color:#fff` на `var(--on-primary)`
  в состояниях «выбрано/активно».
- **Где:** `docs/style.css` — блок `:root`; блок `[data-theme="dark"]`
  (строки ~56-66); селекторы `.chip.active` (226), `.slot.sel` (389),
  `.date-item.sel .date-num` (378), `.hist-upcoming` (423),
  `.admin-header` (781) и связанный блок (676).
- **Контракт:** `--on-primary` = `#FFFFFF` (light) / `#1C1C1E` (dark).
  В компонентах — только `color: var(--on-primary)`, без литерала `#fff`.
- **Критерии готовности:** в `docs/style.css` нет `color:#fff` (или `#ffffff`)
  в перечисленных селекторах; в тёмной теме текст в этих состояниях тёмный и
  читаем на светлом фоне `--primary`; в светлой теме вид не изменился.
- **Связь с ADR:** ADR-024, Решение / ОВ-2 (`--on-primary`).

### Задача 2: Адаптация акцента в тёмной теме
- **Что:** Переопределить `--accent` и `--accent-light` в `[data-theme="dark"]`;
  ввести `--on-accent-light` (текст на фоне `--accent-light`) и `--on-accent`
  (текст/иконки на фоне `--accent`); заменить соответствующий хардкод цвета на
  эти токены.
- **Где:** `docs/style.css`:
  - блок `[data-theme="dark"]` (~56-66) — переопределения и новые токены;
  - текст акцентом, лежащий на `var(--accent-light)` → `var(--on-accent-light)`:
    `.chip.accent` (227), `.fav-chip` (357), `.offer-save` (611),
    `.offer-urgency` (640), `.cs-add` (617) и аналогичные;
  - элементы с фоном `var(--accent)` и хардкодом `color:#fff` →
    `var(--on-accent)`: `.btn-primary` (~208), `.tab-center-btn` (176/182),
    `.offer-tag` (604), `.cs-add.sel` (618), `.feed-cat-chip.active` (755),
    `.feed-book` (763), `.tip-btn.sel` (774), `.consent-check` (747),
    `.aud-check.on` (832), `.scenario-btn.playing` (94), `.sheet-btn.playing`
    (708), `#sw-update-banner button` (854) и аналогичные;
  - акцентный текст/иконки на `--bg`/`--surface` (цены мастеров (351),
    `.btn-ghost` (214), подписи/иконки табов — `currentColor` по ADR-007 (165/166),
    шевроны, ссылки, имя мастера в баннере) — переопределение `--accent`
    устраняет дефект без правки селекторов.
- **Не трогать:** `color:#fff`, не связанный с фоном `--accent`/`--primary`:
  `.btn-danger` (`--red`), `.btn-yandex` (`#FC3F1D`), `.btn-spinner`, аватары
  (`.booking-av`, `.client-av`) с собственным фоном, баннер `.hero-card` (247)
  и поле `#homeSearchBar input` (285) если их фон не `--accent`/`--primary`
  (проверить визуально; если фон тёмный в обеих темах — оставить).
- **Контракт:** dark `--accent #E58AAB`, `--accent-light #4A2433`,
  `--on-accent-light #F2A8C2`, `--on-accent #3A0E1E`; light `--on-accent-light
  #8B3558`, `--on-accent #FFFFFF`.
- **Критерии готовности:** акцентный текст/иконки на `--bg`, `--surface`
  достигают ≥ 4.5:1 (текст) / ≥ 3:1 (иконки); текст на `--accent-light`
  использует `--on-accent-light` (≥ 4.5:1); текст/иконки на фоне `--accent`
  используют `--on-accent` (≥ 4.5:1); ни один акцентный элемент не нечитаем
  ни в одной теме; светлая тема визуально не изменилась.
- **Связь с ADR:** ADR-024, ОВ-1, ОВ-2 (`--on-accent`, `--on-accent-light`).

### Задача 3: Динамический theme-color и color-scheme
- **Что:** Заменить статичный `theme-color` на два мета-тега с `media`-условиями;
  добавить `<meta name="color-scheme" content="light dark">`; в `_setTheme`
  обновлять `content` активного `theme-color` мета-тега при явном выборе темы.
- **Где:** `docs/index.html` (~8 — мета-теги в `<head>`); `docs/prototype.html`
  (функция `_setTheme`).
- **Контракт:**
  - `<meta name="theme-color" content="#FAF7F3" media="(prefers-color-scheme: light)">`
  - `<meta name="theme-color" content="#1C1C1E" media="(prefers-color-scheme: dark)">`
  - `<meta name="color-scheme" content="light dark">`
  - `_setTheme`: при выборе light/dark/system выставляет `content` (light
    `#FAF7F3`, dark `#1C1C1E`; в режиме «системная» — по текущей системной теме).
- **Критерии готовности:** при тёмной теме плашка/хром PWA тёмные; при
  переключении на светлую — светлеют без перезагрузки; в системном режиме
  цвет соответствует системной теме; декларация `color-scheme` присутствует.
- **Связь с ADR:** ADR-024, ОВ-5 (механика C2), ФТ-4, ФТ-5.

### Задача 4: Контраст вторичного текста в тёмной теме
- **Что:** Переопределить `--text-2` в `[data-theme="dark"]`.
- **Где:** `docs/style.css` — блок `[data-theme="dark"]` (~56-66).
- **Контракт:** dark `--text-2 #AEAEB6` (light без изменений).
- **Критерии готовности:** `.svc-meta`, `.confirm-row-lbl`, подписи профиля в
  тёмной теме достигают ≥ 4.5:1 на `--surface` и `--bg`; светлая тема неизменна.
- **Связь с ADR:** ADR-024, ОВ-1, ОВ-2; ФТ-6.

### Задача 5: Индикатор фокуса :focus-visible
- **Что:** Добавить токен `--focus-ring` (обе темы) и единое правило
  `:focus-visible` для полей, с которых снят системный outline.
- **Где:** `docs/style.css` — `.search-bar input` (~283), `.review-textarea`
  (~598), `.admin-textarea` (~823); добавить общее правило `:focus-visible`.
- **Контракт:** `:focus-visible { outline: 2px solid var(--focus-ring);
  outline-offset: 2px; }` для затронутых элементов; `--focus-ring` =
  `#8B3558` (light) / `#E58AAB` (dark). `outline:none` сохраняется только для
  состояния без `:focus-visible` (обычный клик мышью).
- **Критерии готовности:** при навигации Tab по затронутым полям виден
  фокус-кольцо в обеих темах; при клике мышью кольцо не «прилипает»; контраст
  кольца к фону поля ≥ 3:1.
- **Связь с ADR:** ADR-024, ОВ-3; ФТ-7.

### Задача 6: Повышение CACHE_VERSION
- **Что:** Поднять `CACHE_VERSION` в Service Worker (затронуты файлы APP_SHELL:
  `style.css`, `index.html`, `prototype.html`).
- **Где:** `docs/sw.js`.
- **Критерии готовности:** `CACHE_VERSION` увеличена относительно текущего
  значения.
- **Связь с ADR:** ADR-024, раздел «Влияние на код»; НФТ-5.

## Изменения публичного контракта
- **Наблюдаемое поведение:** `theme-color` системной плашки PWA теперь зависит
  от активной темы и меняется без перезагрузки. Миграция не требуется.
- **CSS-токены:** добавлены `--on-primary`, `--on-accent-light`, `--focus-ring`;
  переопределены в dark `--accent`, `--accent-light`, `--text-2`. Светлая тема
  визуально неизменна. Внешнего API/CLI/схем БД релиз не затрагивает.

## Тесты
Автотестов в проекте нет; обязательны ручные проверки (выполняет тестировщик),
которые в этом релизе должны быть зафиксированы как сценарии приёмки:
- Проход тёмной темы: логин → фильтр (чипы) → услуга → мастер → дата → слот →
  история (карточка ближайшей записи) → бронь — все состояния «выбрано/активно»
  читаемы.
- Сравнение светлой темы до/после — без визуальных отличий.
- Переключение темы внутри приложения: цвет системной плашки меняется без
  перезагрузки; системный режим следует системной теме.
- Навигация Tab по `.search-bar input`, `.review-textarea`, `.admin-textarea`
  в обеих темах — виден фокус; клик мышью не оставляет кольцо.
- Замер контраста утверждённых пар токен/фон в DevTools — порог AA выполнен.
- `npm run lint` — без новых ошибок; консоль браузера — без ошибок.

## Риски и митигации
- **Риск:** новые токены без значения в light сломают светлую тему. →
  **Митигация:** объявить `--on-primary`/`--on-accent-light`/`--focus-ring` в
  `:root` со значениями, эквивалентными текущему виду (`#FFFFFF`/`#8B3558`).
- **Риск:** вспышка светлой плашки до выполнения `_setTheme`. → **Митигация:**
  `media`-мета-теги дают корректный стартовый цвет (механика C2).
- **Риск:** подобранные hex не дотянут до AA. → **Митигация:** проверка
  контраста при реализации, корректировка в ту же сторону без смены семантики.
- **Риск:** забыть поднять CACHE_VERSION → старые стили из кэша. →
  **Митигация:** Задача 6 + чек-лист.
