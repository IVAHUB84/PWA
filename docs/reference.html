2Функциональные блоки

Блок Что делает Ключевые функции Источник данных
Авторизация Вход по телефону → OTP на email → сессия в localStorage. Регистрация нового клиента. Email-fallback если email не задан в CRM. sendCodeByPhone, verifyOtp, registerClient, retryEmail, logout YCLIENTS EmailJS localStorage
Бронирование Услуга → Мастер → Дата → Время → Подтверждение. Защита от двойного нажатия. Запись сохраняется локально сразу после ответа API. startBooking, _bookWithSession, quickBook, bookWithMaster, rebook YCLIENTS localStorage
Перенос записи Выбор новой даты/времени → PUT /record/. Перед PUT подтягивает актуальный staff_id и svc_id из API. rescheduleRecord, _rescheduleWithSession YCLIENTS
Отмена записи DELETE /record/ с hash. Обновляет статус в localStorage. AbortController 15 с. cancelRecord, confirmCancel YCLIENTS
История визитов Синхронизирует записи с сервером при каждом открытии главной. Объединяет серверные и локальные, учитывает отменённые. renderHistoryScreen, _fetchAndMergeServerRecords YCLIENTS localStorage
Оценка визита 1–5 звёзд + текст → сохранение в localStorage (до 100 записей). Ссылки на 2GIS и Яндекс.Карты для публичного отзыва. setStar, submitReview, openReview, openRateVisit localStorage
Профиль Фото профиля (кроп 200×200, base64). Избранный мастер. Уровень важности клиента из CRM. Последние 2 визита. renderProfileScreen, renderLoyaltyBlock, _onProfilePhotoPicked YCLIENTS localStorage
Лента постов Посты студии читаются из GitHub (posts.json). Кэш в localStorage. Лайки, фильтр по категории. renderClientFeed, likeFeed, _toggleFeedFilter GitHub API localStorage
Push-уведомления Web Push через Service Worker (Notification API). Показ симулированного баннера в интерфейсе. _sendNotification, showPushNotification, handlePushTap Notification API / SW
PWA / Установка Service Worker с network-first для JS-модулей, cache-first для статики. Web App Manifest. Баннер установки на iOS/Android. sw.js, manifest.json Service Worker
Поиск мастера Поиск по имени/телефону клиента. Запись «за другого» с подтверждением. _homeSearch, showBookForOtherModal, _confirmBookOther localStorage
Согласие / GDPR Тогл согласия на ПД. Запрос удаления аккаунта. Хранение флага в localStorage. acceptConsent, consentToggle localStorage
Админ — Дашборд Записи на сегодня, уникальные клиенты, выручка. Данные из YCLIENTS API в реальном времени. renderAdminDashboard YCLIENTS
Админ — Лента Создание поста (текст + фото), черновики. Синхронизация posts.json на GitHub через Personal Access Token. publishPost, deletePost, renderAdminFeed, _ghSyncPosts GitHub API localStorage
Админ — Клиенты Полный список клиентов с поиском. Подгружает до 200 записей из YCLIENTS. renderAdminClients, filterAdminClients YCLIENTS
Админ — Рассылки Шаблоны и кастомные push. Аудитория: все / активные / с записями. История в localStorage. renderAdminPush, sendNewPush, deletePushCampaign localStorage Notification API
Demo-сценарии Автоматическое воспроизведение пользовательских сценариев для презентации. 14 сценариев. runScenario, stopScenario, sheetRun Внутренний таймер

5YCLIENTS API — запросы

Base URL: https://api.yclients.com/api/v1 · Company ID: 1940850 · Таймаут: 15 с (AbortController)

Метод Endpoint Когда вызывается Ключевые параметры
GET /service_categories/{company} Инициализация приложения (app.js initApp) —
GET /services/{company} Инициализация приложения —
GET /book_staff/{company} Инициализация + экран мастеров service_ids
GET /clients/{company} Вход по телефону / email phone или email
GET /clients/{company}/{client_id} Загрузка уровня лояльности (экран профиля) —
GET /book_dates/{company} Экран выбора даты service_ids, staff_id
GET /book_times/{company}/{staff_id}/{date} Экран выбора времени (при смене даты) service_ids
GET /records/{company} Синхронизация истории записей / Дашборд админа client_id, count или start_date, end_date
GET /record/{company}/{id} Перенос записи — получить актуальные staff_id и svc_id —
POST /book_record/{company} Создание новой записи phone, fullname, email, appointments[]
POST /clients/{company} Регистрация нового клиента name, phone, email
PUT /record/{company}/{id} Перенос записи на новую дату/время datetime, staff_id, services[], client
DELETE /record/{company}/{id} Отмена записи —
GET /clients/{company}?count=200 Список всех клиентов (экран Admin Clients) count

7Service Worker (sw.js)

Аспект Поведение
Версия кэшаv8 — поднимается при каждом изменении APP_SHELL
Статика (APP_SHELL)Cache-first: prototype.html, style.css, manifest.json, иконки, все модули *.js — кэшируются при установке
JS-модулиNetwork-first с fallback на кэш: /modules/ и /app.js — всегда запрашиваются из сети, при ошибке отдаётся кэшированная версия
YCLIENTS APINetwork-only (не кэшируется) — данные должны быть актуальными
ОффлайнПриложение открывается из кэша, показывает последние данные из localStorage
Активацияclients.claim() — немедленное управление всеми вкладками без перезагрузки
