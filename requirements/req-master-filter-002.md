# Требование: Фильтрация мастеров по выбранной услуге

## Что нужно

Когда клиент выбирает услугу и переходит к списку мастеров — показывать только тех мастеров,
которые реально оказывают эту услугу. Сейчас показываются все мастера студии подряд.

## Зачем

Клиент видит мастеров, которые не могут его принять на выбранную услугу — это вводит в заблуждение
и приводит к ошибкам при бронировании. Например, мастер по бровям не делает ресницы.

## Как должно работать

- Клиент выбирает услугу (например, «Коррекция бровей»)
- На экране мастеров отображаются только те, кто оказывает эту услугу
- Мастера которые не оказывают услугу — не показываются (не серые/заблокированные, а просто отсутствуют)
- Если по каким-то причинам ни один мастер не найден — показать понятное сообщение

## Источник данных

YCLIENTS API уже возвращает по каждому мастеру список его услуг (поле services или cats).
Нужно использовать эти данные для фильтрации. Не изобретать свою привязку вручную.

## Что не менять

- Режим «Любой мастер» (кнопка в начале списка) — оставить
- Режим когда мастер уже предвыбран (пришли с карточки мастера) — оставить без изменений
- Внешний вид карточек мастеров — не трогать
- Логика бронирования после выбора мастера — не трогать
