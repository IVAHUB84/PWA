# Требование 033: Экран «Выберите время» — редизайн под мобильное приложение YCLIENTS

## Описание
Переделать экран выбора времени (`s-slots`) так, чтобы он визуально и по структуре повторял экран выбора даты/времени в мобильном приложении YCLIENTS (см. референс-скриншот `IMG_20260530_230100.png`), с двумя отличиями от оригинала:
1. внизу — **наша кнопка «Записаться»** (в оригинале «Продолжить»);
2. **без брендинга «Работает на yclients»**.

### Что на референсе (целевой вид)
- **Шапка:** кнопка «‹ Назад» (как сейчас).
- **Календарь-карточка** (белая скруглённая карточка с тенью) вместо текущей горизонтальной ленты дат:
  - заголовок месяца «Май 2026» слева;
  - стрелки навигации по месяцам «‹ ›» справа;
  - строка дней недели «Пн Вт Ср Чт Пт Сб Вс»;
  - строка(и) с числами месяца; **выбранная дата** — тёмный заполненный кружок (на скрине «31»); **сегодня** — подчёркнуто (на скрине «30»); недоступные/прошедшие даты — приглушённые;
  - под календарём — серая «ручка»-индикатор (drag handle), намекающая на раскрытие на полный месяц.
- **Слоты времени, сгруппированные по периодам дня** вместо плоской сетки:
  - секции «Утро», «День», «Вечер» (и при необходимости «Ночь») с заголовком и **сворачивающим шевроном** «^»;
  - внутри секции — сетка кнопок-слотов (на скрине 3 колонки), отступы как сейчас;
  - **выбранный слот** — тёмная заполненная кнопка (на скрине «14:00»).
- **Низ экрана:** широкая тёмная кнопка действия. В нашем варианте — **«Записаться»**.

## Текущее состояние (подтверждено кодом)
- **Экран:** `prototype.html:180-215` (`#s-slots`): шапка «‹ Назад», заголовок «Выберите время», `#masterStrip` (карточка выбранного мастера), `.scroll` с `.date-picker` и `.slots-section`, `.sticky-bottom` с инфо-строкой и кнопкой, таб-бар.
- **Дата — горизонтальная лента, НЕ календарь.** `.date-picker` (`prototype.html:191-196`): текстовый ярлык месяца `.date-picker-lbl` + `.dates-row#datesRow` (flex-ряд). Рендер — `_renderDates()` (`slots.js:134-152`): проходит 21 день вперёд, оставляет только доступные (из `book_dates`), каждый = `.date-item` (день недели + число). Стрелок навигации по месяцам, сетки чисел и шапки дней недели **нет**. Ярлык месяца ставит `_updateMonthLabel()` (`slots.js:127-132`) по первой доступной дате. Фолбэк без API — `_renderDatesFallback()` (`slots.js:154-167`), 7 дней.
- **Слоты — плоская сетка из 3 колонок, БЕЗ группировки.** `.slots-grid#slotsGrid` (`prototype.html:199-201`), CSS `grid-template-columns: repeat(3,1fr)` (`style.css:390`). Рендер — `loadTimes()` (`slots.js:170-206`): `GET /book_times/{company}/{staffId}/{iso}` → `r.data` (массив `{time|datetime}`), мапится в `<button class="slot" onclick="selectSlot(...)">HH:MM</button>` одним списком. Группировки по утро/день/вечер и сворачивания **нет**. Время санитизируется до `HH:MM` (защита от инъекции в onclick) — сохранить.
- **Выбор:** `selectDate()` (`slots.js:208-213`) — снимает/ставит `.sel`, грузит время; `selectSlot()` (`slots.js:215-221`) — `hapticTap('select')`, `.sel` на слот, пишет `state.slot`, `updateStickyBottom()`. Первая дата и первый слот авто-выбираются.
- **Кнопка низа:** `.sticky-bottom` (`prototype.html:204-207`): `#stickyInfo` (дата · слот · услуга) + `<button class="btn-primary" onclick="startBooking()">`. Текст ставится в `updateStickyBottom()` (`slots.js:57-63`): «Подтвердить запись», либо «Перенести запись» при `state._rescheduleId`. После этого экрана идёт отдельный экран подтверждения `#s-confirm` (`updateConfirmScreen`, `slots.js:70-99`).
- **Стили:** `style.css:375-398` — `.date-picker`, `.dates-row`, `.date-item(.sel)`, `.date-day`, `.date-num`, `.slots-section`, `.slots-lbl`, `.slots-grid`, `.slot(.sel/.busy)`. Тёмная тема и тактильный отклик `.slot:active` уже есть (`style.css:1085-1107`).
- **Тема:** селектор использует `var(--primary)`/`var(--on-primary)` для выбранного состояния — «тёмная заливка» на референсе ложится на это естественно.
- **Текущая версия:** `docs/version.json` (проверить актуальный build перед релизом).

## Решения пользователя (зафиксированы, не пересматривать)
1. Вид — как в мобильном YCLIENTS (скриншот-референс).
2. Кнопка внизу — текст **«Записаться»**.
3. Без надписи «Работает на yclients».
4. **Календарь — полноценный раскрывающийся месяц** (месячный грид с раскрытием/сворачиванием, не просто недельная лента).
5. **Реальная подгрузка соседних месяцев** стрелками «‹ ›» (доп. запросы `book_dates`), а не навигация в пределах одного окна.
6. **После нажатия «Записаться» → переход на экран «Запись подтверждена»** (это существующий `s-confirm`, `prototype.html:218-223`, заголовок «Запись подтверждена!»). Текущая кнопка `startBooking()` уже бронирует через API и делает `go('s-confirm')` — **поведение сохраняем, меняем только подпись кнопки** (и текст «Записываем…»/сброс в `booking.js:47,88`). Отдельного промежуточного шага подтверждения не добавлять.
7. **Карточку мастера `#masterStrip` убрать** с экрана выбора времени (на референсе её нет).

## Решения архитектуры/UX (определены при оформлении требования; BA может уточнить)
- **Границы периодов суток:** Утро `< 12:00`, День `12:00–16:59`, Вечер `≥ 17:00`. Пустые секции **скрывать**. Все непустые секции по умолчанию **раскрыты**. Состояние раскрытия — только в рамках сессии экрана (не запоминать между заходами).
- **Числа месяца:** рисовать полный месячный грид; **недоступные/прошедшие даты — приглушённые и некликабельные**, доступные (из `book_dates`) — активные; «сегодня» подчёркнуто; выбранная — тёмный кружок.
- **Авто-выбор** первой доступной даты и первого слота — сохранить текущее поведение.
- **Инфо-строка `#stickyInfo`** (дата · слот · услуга) над кнопкой — оставить (полезный контекст перед бронированием); решение не критично, на усмотрение реализации.

## Открытые вопросы (для BA/архитектора)
- **Перенос (`_rescheduleId`).** Сейчас при переносе кнопка = «Перенести запись», успех → `s-confirm` с заголовком «Запись перенесена!» (`booking.js:133-137`). Меняем ли подпись кнопки переноса тоже на «Записаться» или оставляем «Перенести запись»? (Предложение: для переноса оставить «Перенести запись», т.к. семантика иная.)
- **Месяцы без доступных дат при навигации стрелками.** Поведение, когда `book_dates` за выбранный месяц вернул пусто или ушли за горизонт записи (≈ месяц вперёд): блокировать стрелку «вперёд», показывать пустой месяц с подсказкой, или ограничить диапазон. Прошлые месяцы — стрелка «назад» неактивна до текущего.
- **Анимация раскрытия секций/календаря** — нужна ли (предложение: лёгкая, по аналогии с инбоксом/существующими паттернами) или мгновенное переключение.

## Ограничения
- Vanilla JS + HTML + CSS, **без рантайм-зависимостей**; код в `docs/prototype.html`, `docs/modules/slots.js`, `docs/style.css`.
- **Минимальные изменения строго по объёму требования.** Не трогать поток бронирования (`startBooking`, `s-confirm`, `state.slot/dateISO/dateFull`, перенос `_rescheduleId`) сверх необходимого.
- Сохранить **санитизацию времени** до `HH:MM` перед вставкой в `onclick` (`slots.js:200`).
- Сохранить интеграцию с API: `GET /book_dates/{company}` и `GET /book_times/{company}/{staffId}/{iso}`, фолбэки и состояния «нет окон / ошибка сети».
- Корректная работа в **тёмной теме** (использовать токены `var(--…)`, не хардкодить цвета) и **тактильный отклик** (`hapticTap`) при выборе.
- При изменении `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION` (если затронуто).
- Экранирование (`esc()`) для любого вставляемого текста.

## Критерий готовности
- Экран `s-slots` визуально соответствует референсу YCLIENTS: календарь-карточка (месяц + стрелки + дни недели + числа, выбранная дата тёмным кружком, сегодня подчёркнуто) и слоты, сгруппированные по периодам дня со сворачиваемыми секциями.
- Календарь — полноценный месячный грид; стрелки «‹ ›» реально подгружают соседние месяцы (`book_dates`); недоступные/прошедшие даты приглушены и некликабельны; «сегодня» подчёркнуто; выбранная дата — тёмный кружок.
- Слоты разбиты по периодам (Утро `<12:00` / День `12:00–16:59` / Вечер `≥17:00`), пустые секции скрыты, непустые раскрыты по умолчанию, шеврон сворачивает/разворачивает секцию.
- Выбранный слот отображается тёмной заливкой; выбор даты перезагружает слоты; авто-выбор первой даты/слота сохранён.
- **Карточка мастера `#masterStrip` на экране отсутствует.**
- Кнопка внизу подписана **«Записаться»**; нажатие бронирует через API и ведёт на экран `s-confirm` «Запись подтверждена!» (для переноса — по согласованному тексту).
- Нет надписи «Работает на yclients».
- Загрузка дат/времени, фолбэки, состояния «нет окон»/ошибки сети работают; санитизация времени сохранена.
- Корректно в светлой и тёмной теме; тактильный отклик при выборе; ноль ошибок в консоли; PWA installable не сломан; при изменении `sw.js` поднята `CACHE_VERSION`.
