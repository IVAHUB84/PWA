# Требование 018: Красивые рамки плашек в каталоге услуг и карусели

## Описание
В разделе услуг — карточках каталога услуг (`s-services`) и слайдах карусели в детальной карточке (`s-service`) — сделать **красивые рамки у плашек**. Цель: визуально аккуратные, выразительные границы карточек/слайдов, а не плоские/невыразительные края.

## Текущее состояние (подтверждено кодом)
- **Карточка каталога** `.svc-catalog-card` (`docs/style.css:273-282`): `background: var(--surface)`, `border-radius: 16px`, `box-shadow: var(--shadow)`, `border: 1px solid var(--border)`, `overflow: hidden`. На `:active` — `transform: scale(0.97)`. В одном из переопределений тень ослаблена: `.svc-catalog-card { box-shadow: 0 1px 4px rgba(0,0,0,0.05) }` (`style.css:693`) — т.е. **итоговая рамка = тонкий бордер `--border` + лёгкая тень**.
- Внутри карточки каталога: обложка `.svc-cover` (`aspect-ratio:16/9`, `overflow:hidden`, `style.css:283-289`), `.svc-cover-img`/`.svc-cover-placeholder`, блок `.svc-info` (`padding:14px 16px 16px`).
- **Слайд карусели** `.svc-carousel-slide` (`docs/style.css:834-838`): `border-radius: 16px`, `overflow: hidden`, **без бордера** — только скругление, рамки как таковой нет. Картинка `.svc-carousel-slide img` — `height:280px; object-fit:cover`.
- **Одиночное фото / плейсхолдер детальной карточки** (`style.css:812-820`): `border-radius:16px`, без бордера.
- Переменные темы: `--border`, `--surface`, `--shadow`, `--accent`, `--radius` — используются по проекту; есть тёмная тема (переопределения теней в районе `style.css:690-696`).
- Разметка карточек каталога — `services.js:91-96` (`.svc-catalog-card` → `.svc-cover` → `.svc-info`), слайдов — `services.js:168-169`.

## Открытые вопросы (для BA/архитектора)
- **Что значит «красивая рамка» (ключевой вопрос).** Это субъективное требование — нужен конкретный визуальный ориентир. Варианты: (а) более выраженный, но аккуратный бордер (цвет/толщина — например, `1.5px` `--border` или акцентный оттенок); (б) мягкая «карточная» тень вместо/вместе с бордером; (в) скругление крупнее (`--radius`/20px); (г) тонкая внутренняя обводка (`box-shadow: inset 0 0 0 1px ...`) для чёткого края на фото; (д) комбинация. Запросить у пользователя референс/ожидаемый стиль.
- **Где именно «плашки».** Подтвердить охват: только карточки каталога `.svc-catalog-card` и слайды карусели `.svc-carousel-slide`, или ещё одиночное фото/плейсхолдер и блок тела детальной карточки. По формулировке — «в услугах и в карусели».
- **Единый стиль рамки** для каталога и карусели или они оформляются по-разному (у слайда фото — обводка поверх изображения, у карточки каталога — рамка вокруг всей плашки).
- **Тёмная тема.** Рамка должна корректно смотреться в обеих темах (есть переопределения теней для тёмной темы). Какой цвет/контраст границы в тёмной теме.
- **Состояния.** Сохранить ли `:active scale(0.97)`; нужна ли отдельная рамка для выбранного/наведённого состояния.

## Ограничения
- Только CSS-изменения (`docs/style.css`), без правок логики. Если без разметки не обойтись — минимально в `docs/modules/services.js`, сохраняя `esc()`.
- Не ломать `overflow: hidden` (скругление обрезает фото), `aspect-ratio` обложки, геометрию карусели из требования 017 (рамка слайда не должна нарушать расчёт ширины/peek).
- Изменения строго в пределах раздела услуг (каталог + детальная карточка/карусель). Не трогать карточки мастеров, истории, ленты и т.п.
- Корректность в светлой и тёмной теме; без ошибок в консоли.
- При изменении версий модулей в `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION`.
- Минимальные изменения строго по объёму.

## Критерий готовности
Карточки каталога услуг и слайды карусели имеют аккуратные, визуально выразительные рамки в согласованном с пользователем стиле; оформление единообразно и корректно отображается в светлой и тёмной теме; скругление/обрезка фото и геометрия карусели (выравнивание, peek) не нарушены; прочие разделы приложения не затронуты; ошибок в консоли нет.
