# Требование 008: Прямой переход с выбора мастера на выбор времени

## Описание
В сценарии онлайн-записи клиент выбирает услугу (например, «Стрижка»), на следующем экране выбирает мастера — и **сразу** попадает на экран «Выберите время», без промежуточного экрана.

## Целевой поток
1. Клиент выбирает услугу → экран списка мастеров.
2. Клиент тапает по мастеру в списке → **сразу** экран «Выберите время» (`s-slots`).
3. Промежуточная карточка-профиль мастера (`s-master`) в этом потоке **не показывается**.

## Текущее состояние (подтверждено кодом)
- Поток сейчас: услуга → `s-masters` (список) → тап по мастеру вызывает `openMasterCard` (`docs/modules/masters.js:16`, `onclick`) → открывается экран-карточка мастера `s-master` → кнопка «Записаться к этому мастеру» → `s-slots` («Выберите время»).
- Уже существует функция `selectMaster(id)` (`masters.js:112`), которая ставит `state.masterId` и сразу делает `go('s-slots')` — то есть нужный переход мастер → время технически уже есть, но в списке он не используется (вместо него открывается карточка).
- Вариант «Любой мастер» (`selectAnyMaster`, `masters.js:236`) уже ведёт сразу на `s-slots` — то есть желаемое поведение для него уже работает.

## Нюансы для проработки (BA/архитектор)
- **Перенос данных мастера.** `selectMaster` сейчас ставит только `state.masterId`. Путь через карточку (`bookServiceFromMaster`) дополнительно заполняет `state.masterName`, `masterAvatar`, `masterGrad`. Нужно убедиться, что экран «Выберите время» и экран подтверждения корректно показывают мастера (имя/аватар) и при прямом переходе.
- **Режим «Все мастера».** Карточка `_masterCardHtml` (`masters.js`) используется и в потоке записи (услуга выбрана), и в режиме просмотра «Все мастера студии» (`renderMasters` с `browseAll`, вход из главного экрана/избранного). Прямой переход на время логичен только когда **услуга уже выбрана**. При простом просмотре мастеров (без выбранной услуги) тап, вероятно, должен по-прежнему открывать профиль мастера — уточнить.
- **Судьба экрана `s-master`.** Карточка-профиль убирается только из потока записи или удаляется как экран совсем? Если она нужна для просмотра мастера (без записи) — оставить доступной из «Все мастера»; если нет — удалить экран и связанный код (`openMasterCard`, `bookFromMaster` и т.п.). Уточнить.

## Ограничения
- Vanilla JS, без рантайм-зависимостей; код в `docs/`.
- Изменения только в сценарии записи; не ломать YCLIENTS-интеграцию (получение мастеров по услуге, слотов).
- При изменении `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION`.
- Минимальные изменения строго по объёму.

## Критерий готовности
В сценарии записи после выбора услуги тап по мастеру в списке открывает экран «Выберите время» напрямую, без промежуточной карточки мастера; имя/аватар выбранного мастера корректно отображаются дальше по потоку; вариант «Любой мастер» и (если сохраняется) просмотр мастеров вне записи продолжают работать.
