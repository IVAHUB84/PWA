# Требование: Визуальный каталог услуг

## Что нужно

Переделать экран выбора услуг из простого списка в красивый визуальный каталог с фотографиями.
Сейчас услуги показаны скучным списком — название, цена, длительность. Хочу чтобы клиент
"поедал глазами" услуги и сразу хотел записаться.

## Зачем

Студия продаёт эстетику — реснички, брови, макияж. Текстовый список не передаёт это.
Красивые фото процедур и результатов повысят конверсию в запись.

## Как должно выглядеть

- Карточки услуг с большой фотографией (как в интернет-магазине или Airbnb)
- На карточке: фото, название услуги, цена, длительность
- Категории услуг — горизонтальный скролл чипов сверху (как сейчас), но карточки внизу крупные
- При нажатии на карточку — переход к выбору мастера (как сейчас)
- Должно выглядеть дорого и красиво — студия премиум-сегмент

## Фотографии

Фотографии услуг нужно хранить где-то и привязывать к услугам.
Возможные варианты:
- Загружать фото в GitHub репозиторий (в папку docs/images/)
- Или просто задать URL на внешние фото
- Привязка по названию услуги или категории (не по ID из YCLIENTS — они могут меняться)

Для старта можно использовать красивые стоковые фото или фото самой студии.

## Что не менять

- Логика выбора услуги и переход к мастеру — не трогать
- Поиск по услугам — сохранить
- Фильтр по категориям — сохранить
- Если фото для услуги нет — показывать красивый градиентный плейсхолдер с эмодзи категории

## Дополнительный контекст

- Студия называется "Реснички, сестрички" — направление: брови, реснички, макияж, ПМ
- Целевая аудитория: женщины 20-40 лет, Троицк (Москва)
- Конкуренты которые делают красиво: любое beauty-приложение типа Fresha, Booksy
- Текущий код услуг: docs/modules/services.js, экран s-services в prototype.html
