# Требование 034: Экран «Выберите время» — фиксированный сворачиваемый календарь

## Описание
Доработать экран выбора времени (`s-slots`, выпущен в v1.29.0) под удобную работу на мобильном:
1. **Зафиксировать календарь** — при прокрутке списка слотов карточка календаря остаётся на месте (не уезжает вверх). Скроллится только область слотов.
2. **Сворачиваемый календарь** — вид календаря можно уменьшать до **одной строки** (как на референс-скриншоте) и разворачивать обратно до полного месяца:
   - свайп **вверх** — свернуть (до одной недели/строки);
   - свайп **вниз** — развернуть (полный месяц).
3. **Drag handle («черточка»)** — серый индикатор-ручка, по которому понятно, что блок можно свайпать.
4. **Кнопка «Записаться» появляется только когда выбраны И дата, И время.** Пока выбор не сделан — кнопки нет (а не просто disabled).
5. **Убрать заголовок «Выберите время»** (`.page-title`) — занимает много вертикального места.

## Текущее состояние (подтверждено кодом)
- **Экран `s-slots`** (`prototype.html:180-215`): `.status-bar`, `.nav-bar` («‹ Назад»), `.page-title` «Выберите время», затем `.scroll`, внутри которого **и календарь, и слоты**:
  - `.cal-card` (`prototype.html:186-196`) — `.cal-header` (стрелки `calPrev`/`calNext` + `.cal-month-lbl`), `.cal-weekdays` (Пн…Вс), `.cal-grid#calGrid` (месячный грид);
  - `.slots-section` (`prototype.html:197-202`) — `.slots-lbl#slotsDateLbl` + `#slotsGrid` (секции по периодам суток).
- **Календарь сейчас внутри `.scroll`** → скроллится вместе со слотами. `.scroll { flex:1; overflow-y:auto }` (`style.css:152`). `.cal-card` (`style.css:368-371`) — обычная карточка в потоке. Чтобы зафиксировать — вынести из скролла (отдельный нескроллируемый блок над `.scroll`) либо `position: sticky`.
- **Месячный грид** рисует `_renderCalGrid()` (`slots.js:53-132`): строит полные недели (Пн-базис), `totalCells` кратно 7; выбранная дата — `.cal-cell.sel`, сегодня — `.today`, недоступные/прошлые — `.unavail`. Свёрнутого (недельного) режима **нет**.
- **Навигация месяцев** — `calNavMonth()` (`slots.js:199-231`), стрелки `‹ ›`; подгрузка соседних месяцев (`book_dates`) + дизейбл стрелок на границах.
- **Кнопка низа** — `.sticky-bottom` (`prototype.html:204-207`): `#stickyInfo` + `<button class="btn-primary" onclick="startBooking()">Записаться</button>`. Кнопка **всегда в DOM**; включение/выключение — `_setConfirmBtnEnabled()` (`slots.js:7-12`): меняет `disabled` и `opacity` (0.45), **не скрывает**. Текст ставит `updateStickyBottom()` (`slots.js:271-277`): «Записаться» / «Перенести запись» (при `_rescheduleId`).
- **Авто-выбор.** `_autoSelectFirstDate()` (`slots.js:191-197`) авто-выбирает первую доступную дату; `loadTimes()` (`slots.js:353-378`) авто-выбирает **первый слот** (`firstTime`, `.sel`) и включает кнопку. То есть сейчас при заходе и дата, и время уже выбраны, и кнопка активна сразу — **конфликтует с п.4** (см. открытые вопросы).
- **Слоты** сгруппированы по периодам суток со сворачиваемыми секциями (`.slot-period`, `slots.js:355-385`, `style.css:415-432`) — это поведение **не трогаем**.
- **`.page-title`** (`style.css:207`) — `font-size:28px; font-weight:800; padding:0 20px 4px`.

## Пожелания/решения пользователя (со слов)
1. Календарь зафиксирован, скроллятся только слоты.
2. Календарь сворачивается до одной строки и разворачивается до месяца жестом свайпа (вверх — свернуть, вниз — развернуть).
3. Нужна «черточка» (drag handle) как визуальный намёк на свайп.
4. Кнопка «Записаться» появляется только при выбранных дате и времени.
5. Заголовок «Выберите время» убрать.
6. Референс свёрнутого вида — «как на скриншоте» (одна строка дней).

## Решения пользователя (зафиксированы, не пересматривать)
1. **Авто-выбор убрать.** Дата и слот не выбираются автоматически: пользователь сам тапает дату → тапает слот → только тогда появляется кнопка «Записаться». Затрагивает `_autoSelectFirstDate()` (`slots.js:191-197`) и авто-выбор первого слота в `loadTimes()` (`slots.js:353,366-367,378`).
2. **Свёрнутый вид = неделя выбранной даты.** Одна строка Пн-Вс, содержащая выбранную дату; выбранный день — тёмный кружок. Та же сетка, что и месяц, но одна строка. Если дата ещё не выбрана — показывать текущую неделю (с «сегодня»).
3. **По умолчанию календарь развёрнут (полный месяц).** Сворачивание — свайпом вверх.

## Открытые вопросы (для BA/архитектора)
- **Десктоп/без тач-события.** Свайп — тач-жест. Нужен ли альтернативный триггер (тап/клик по «черточке» сворачивает-разворачивает) для мыши и доступности? **Предложение: да — клик по handle тоже переключает.**
- **Стрелки навигации месяцев `‹ ›` в свёрнутом виде** — оставлять видимыми/рабочими или прятать (в недельном виде логичнее листать недели)? **Предложение: в свёрнутом виде оставить стрелки месяца как есть (минимизировать изменения), либо скрыть — на усмотрение реализации.**
- **Анимация сворачивания** — плавная (по высоте, как у `.slot-period-body`) или мгновенная? **Предложение: плавная.**

## Ограничения
- Vanilla JS + HTML + CSS, **без рантайм-зависимостей**; код в `docs/prototype.html`, `docs/modules/slots.js`, `docs/style.css`.
- **Минимальные изменения строго по объёму требования.** Не трогать: группировку слотов по периодам, поток бронирования (`startBooking`, `s-confirm`), подгрузку дат/времени (`book_dates`/`book_times`), перенос (`_rescheduleId`), санитизацию времени до `HH:MM`.
- Календарный грид и навигация месяцев уже работают — переиспользовать, не переписывать.
- Корректная работа в **светлой и тёмной теме** (токены `var(--…)`, без хардкода цветов) и **тактильный отклик** (`hapticTap`) при выборе даты/слота.
- Жесты не должны конфликтовать со скроллом списка слотов и с горизонтальной навигацией.
- При изменении `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION`.
- Экранирование (`esc()`) для любого вставляемого текста.

## Критерий готовности
- Карточка календаря **зафиксирована**: при прокрутке слотов остаётся на месте, скроллится только список времени.
- При входе календарь **развёрнут (полный месяц)**; **сворачивается до одной строки** (свайп вверх) и **разворачивается обратно** (свайп вниз); в свёрнутом виде показана **неделя выбранной даты** (выбранный день — тёмный кружок; если дата не выбрана — текущая неделя).
- **Авто-выбора нет**: при входе ни дата, ни слот не выбраны.
- Под календарём — видимая **«черточка» (drag handle)**, по которой понятно, что блок свайпается; (опц.) клик по ней также переключает состояние.
- Кнопка **«Записаться» отсутствует**, пока не выбраны и дата, и время; после выбора обоих — появляется и бронирует через существующий поток (`startBooking` → `s-confirm`). Для переноса — текст по согласованию (см. req-033).
- Заголовок **«Выберите время» удалён**.
- Группировка слотов по периодам суток, подгрузка дат/времени, фолбэки и состояния «нет окон»/ошибка сети — **работают как прежде**.
- Корректно в светлой и тёмной теме; тактильный отклик при выборе; ноль ошибок в консоли; PWA installable не сломан; при изменении `sw.js` поднята `CACHE_VERSION`.
