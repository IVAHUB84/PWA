# Требование 011: Открытие конкретного поста из превью ленты на главной

## Описание
В блоке «Лента студии» на главном экране при тапе по **конкретной плашке** должен открываться **именно этот пост**, а не общий список всех постов (как сейчас — будто нажата ссылка «Все →»).

## Целевое поведение
- Тап по конкретной плашке превью ленты на главной открывает **именно выбранный пост**.
- Ссылка «Все →» сохраняет текущее поведение — открывает полный список ленты (`s-feed`).
- Из открытого поста клиент может вернуться назад (к главной/ленте) обычной навигацией.

## Текущее состояние (подтверждено кодом)
- Превью ленты на главной рендерит `_renderHomeFeedPreview` (`docs/modules/profile.js:132-160`). У каждой плашки `onclick="go('s-feed')"` (`profile.js:154`) — **тот же переход, что и у «Все →»** (`prototype.html:72`). Поэтому тап по любой плашке = тап по «Все →».
- Экран ленты `s-feed` рендерится `_paintFeed` (`docs/modules/feed.js:53-80`) — показывает **все** опубликованные посты с баром фильтра по категориям. Представления одного поста сейчас нет.
- У постов есть стабильный `id` (создаётся в `admin.js:111` как `Date.now()`), доступный в `yc_feed_posts` (`localStorage`). По нему можно адресовать конкретный пост.

## Открытые вопросы (для BA/архитектора)
- **Как именно «открыть пост»?** Отдельное представление одного поста (например, экран `s-post` или модалка) **или** открыть `s-feed` с прокруткой к выбранному посту и его подсветкой? Формулировка «именно этот пост, а не все» скорее предполагает чистое представление одного поста.
- **Передача идентификатора.** Использовать `p.id` для адресации (надёжнее индекса, т.к. список может меняться).
- **Навигация назад.** Подтвердить ожидаемое поведение «Назад» (на главную или в полный список ленты).
- **Бар фильтра категорий** в представлении одного поста, вероятно, не нужен — подтвердить.
- **Несуществующий/удалённый пост.** Что показывать, если поста с переданным id уже нет (например, удалён через админку).

## Ограничения
- Vanilla JS, без рантайм-зависимостей; код в `docs/`.
- Источник постов — `yc_feed_posts` в `localStorage` (+ обновление из GitHub через `_ghReadFn`, `feed.js:38`), как в существующем коде. Только неудалённые/неопубликованные-черновики фильтруются (`!p.draft`).
- При изменении `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION`. Минимальные изменения по объёму.
- Не ломать «Все →» (полный список ленты), фильтр по категориям и кнопку «Записаться →» внутри карточки поста.

## Критерий готовности
Тап по конкретной плашке превью ленты на главной открывает именно выбранный пост (а не общий список); «Все →» по-прежнему открывает полный список ленты; из поста доступен возврат назад; при отсутствии поста по id поведение не приводит к ошибке/пустому экрану без объяснения.
