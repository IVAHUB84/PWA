# Требование 012: Раздел «Услуги» — карточки в один столбец и заметная строка поиска

## Описание
Переоформить экран выбора услуги (`s-services`). Сейчас услуги показаны маленькими карточками в **два столбца** — нужно сделать их в **один столбец**, по визуальному стилю похоже на «Ленту» (крупные карточки с обложкой и текстовым блоком). Дополнительно строка поиска по услугам сейчас **сливается с фоном** — её нужно сделать более выделенной.

## Целевое поведение
- Каталог услуг отображается **в один столбец**: одна карточка на строку, во всю ширину контентной области.
- Карточки крупнее текущих, визуально в стиле карточек Ленты (обложка + блок с названием и метаданными цена · длительность).
- Строка поиска по услугам визуально **выделена** на фоне экрана (заметная рамка/фон/тень), не сливается с подложкой; плейсхолтер и иконка читаемы.
- Поведение не меняется: тап по карточке услуги выбирает услугу и ведёт дальше (`selectService` → `s-masters` или `s-slots`), поиск и фильтр по категории работают как прежде, баннер предвыбранного мастера сохраняется.

## Текущее состояние (подтверждено кодом)
- Каталог рендерит `_renderList` в `docs/modules/services.js:77-100`: формирует `<div class="svc-catalog">` со списком карточек `.svc-catalog-card` (обложка `.svc-cover` + `.svc-info` с `.svc-name` / `.svc-meta`).
- Сетка в два столбца задана в CSS: `.svc-catalog { grid-template-columns: 1fr 1fr; gap: 12px; padding: 0 16px 24px; }` (`docs/style.css:261-265`). Размеры карточки/обложки/текста — `docs/style.css:267-304` (обложка `padding-top:75%`, имя 13px, мета 12px).
- Для сравнения, карточки Ленты — один столбец: `.feed-card { margin:0 16px 16px; border-radius:16px; }`, фото `aspect-ratio:16/9`, тело `.feed-body` (`docs/style.css:706-712`).
- Строка поиска: HTML `prototype.html:110-113` (`<div class="search-bar">` с иконкой 🔍 и `#serviceSearch`). CSS `.search-bar { background: var(--surface); height:44px; ... }` (`docs/style.css:252-257`) — фон `--surface` близок к фону экрана, поэтому строка плохо различима.

## Открытые вопросы (для BA/архитектора)
- **Формат карточки в один столбец.** Обложка сверху во всю ширину (как Лента, `aspect-ratio:16/9`) и текст под ней, **или** горизонтальная компоновка (миниатюра слева + текст справа)? Уточнить желаемый вид.
- **Размер обложки.** Сохранить пропорцию 75% / перейти на 16/9 / задать фиксированную высоту — что предпочтительнее визуально.
- **Выделение строки поиска.** Каким приёмом: более тёмный/контрастный фон, заметная рамка, тень, либо комбинация? Нужно ли менять также строки поиска на других экранах, использующие `.search-bar`, или только на экране услуг.
- **Размеры текста.** Увеличивать ли название/мету относительно текущих 13/12px при переходе на крупные карточки.

## Ограничения
- Vanilla JS + CSS, без рантайм-зависимостей; код в `docs/`.
- Класс `.search-bar` может использоваться на других экранах — изменения стиля не должны ломать их вид (проверить переиспользование перед глобальной правкой).
- Минимальные изменения по объёму, без рефакторинга соседнего кода.
- При изменении `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION`.

## Критерий готовности
На экране `s-services` услуги отображаются в один столбец крупными карточками (визуально в духе Ленты); строка поиска заметно выделяется на фоне и не сливается с подложкой; выбор услуги, поиск, фильтр по категории и баннер предвыбранного мастера продолжают работать без ошибок в консоли.
