# Требование 029: Drill-down записей и клиентов в админке (детали + быстрые действия)

## Описание
Экраны «Сводка» (`s-admin`) и «Клиенты» (`s-admin-clients`) в админ-панели — только для просмотра. Строки записей и клиентов некликабельны: администратор не может открыть детали, позвонить клиенту, написать или перейти к записи. Чтобы выполнить любое действие (позвонить, отменить, перенести), приходится уходить в веб-кабинет YCLIENTS. Для основной задачи администратора — оперативно управлять записями и клиентами — это тупиковые экраны.

## Целевое поведение
Сделать строки записей (Сводка) и клиентов (База) кликабельными → открывать bottom-sheet с деталями и быстрыми действиями.

- **Запись (Сводка):** тап по строке → шит с деталями (клиент, услуга, мастер, время, стоимость) и действиями:
  - Позвонить клиенту (`tel:`) — если есть телефон.
  - Открыть запись в YCLIENTS (внешняя ссылка) — если возможно сформировать URL.
- **Клиент (База):** тап по строке → шит с деталями (имя, телефон, число визитов) и действиями:
  - Позвонить (`tel:`).
  - (опц.) написать / открыть карточку клиента в YCLIENTS.

Минимально достаточный объём — `tel:`-звонок по телефону клиента из обоих экранов; остальное — по решению архитектора и доступности данных.

## Открытые вопросы (для BA/архитектора)
- **ОВ-1. Есть ли телефон в `/records`.** Отдаёт ли YCLIENTS `client.phone` в ответе `/records/{company}` (Сводка), или нужен доп. запрос `/clients`? База: использовать то, что есть в ответе; не делать лишних запросов на каждую запись.
- **ОВ-2. Глубокие ссылки в YCLIENTS.** Можно ли сформировать стабильный URL записи/клиента в веб-кабинете YCLIENTS из `record.id` / `client.id`? Если нет — ограничиться `tel:` и просмотром деталей.
- **ОВ-3. Компонент шита.** Использовать существующий bottom-sheet проекта или сделать лёгкий локальный для админки. База: переиспользовать имеющийся, чтобы не плодить дубли.
- **ОВ-4. Действия отмены/переноса.** Включать ли отмену/перенос записи через API в этот релиз, или только звонок + просмотр? База: в этом релизе — просмотр + `tel:`; отмена/перенос — отдельным требованием (риск деструктивных операций по API).

## Ограничения
- Vanilla HTML/CSS/JS, без новых рантайм-зависимостей; код в `docs/`.
- `tel:` и внешние ссылки — экранирование (`esc`), `rel="noopener"` для внешних.
- Не ломать существующий поиск клиентов и рендер строк.
- Никаких деструктивных операций (отмена/удаление записи) без отдельного согласования — этот релиз только просмотр + связь.
- При изменении `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION`.
- Минимальные изменения: кликабельность строк + шит с деталями и действиями.

## Критерий готовности
- Строка записи в Сводке кликабельна → открывает шит с деталями записи и кнопкой звонка клиенту (`tel:`), если есть телефон.
- Строка клиента в Базе кликабельна → открывает шит с деталями и кнопкой `tel:`.
- Телефон в шите можно нажать (звонок) и выделить/скопировать.
- Шит закрывается тапом по фону/кнопке; не ломает прокрутку и tab-bar.
- В консоли нет ошибок; PWA остаётся устанавливаемой; при изменении `APP_SHELL`/`sw.js` поднята `CACHE_VERSION`.
