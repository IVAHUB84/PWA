# Требование 031: Плашка «О студии» — данные только из YCLIENTS, выбор карты, новое оформление

## Описание
Контактный блок студии (введён в v1.23.0, req-027/BA-028) на профиле `s-profile`
и экране подтверждения `s-confirm` сейчас показывает **плейсхолдеры и битые
ссылки**. Боевая проверка `/company/1940850` и `/companies?id=1940850`
(2026-05-30) выявила три дефекта:

1. **«Маршрут» ведёт не туда — на форму отзыва о ЧУЖОЙ студии.** Координат из
   `/company/{id}` нет (`0,0`), поэтому срабатывает фолбэк
   `STUDIO.mapYandex = REVIEW_URLS.yandex` — а это ссылка «оставить отзыв» про
   другую организацию («БьютиФлайБар», Троицк), не маршрут и не та студия.
2. **Телефон — фейк `+7 495 123-45-67`.** Partner-токен телефон компании не
   отдаёт (`phone:""`, `phones:[]`), показывается плейсхолдер из `STUDIO` как
   кликабельный `tel:` → звонок уходит в никуда.
3. **Город теряется.** Показывается `address` («улица Генерала Кузнецова, 22»)
   без `city` («Люберцы») — это разные поля API.

## Целевое поведение (согласовано с заказчиком)

### Принцип данных
- Данные плашки берутся **ИСКЛЮЧИТЕЛЬНО из YCLIENTS** (API + кэш `localStorage`
  `yc_company`). **Никаких хардкод-плейсхолдеров.**
- Если значение в API пустое — **строка остаётся, метка остаётся, значение =
  прочерк `—`** (напр. `Телефон: —`). Это касается и заголовка (пустой → `—`).
- Полностью **убрать объект `STUDIO`** как источник данных и **перестать
  использовать `REVIEW_URLS`** в плашке (REVIEW_URLS остаётся только для модуля
  отзывов `review.js`).

### Источник — другой endpoint (важно!)
- Переключить загрузку с `GET /company/{id}` на **`GET /companies?id={id}`**.
  Проверено: `/companies?id=` отдаёт **реальные координаты**
  (`coordinate_lat=55.686006, coordinate_lon=37.85368`), тогда как
  `/company/{id}` на ту же компанию возвращает `0,0`. Также `/companies?id=`
  содержит `title`, `public_title`, `address`, `city`, `schedule`, `site`,
  `social` — всё, что нужно плашке, одним запросом.

### Состав плашки
- **Заголовок:** `public_title` (имя для онлайн-записи) с фолбэком на `title`.
- **Адрес:** склейка `city + ', ' + address` («Люберцы, улица Генерала
  Кузнецова, 22»).
- **Телефон:** значение из API; пусто → `—` (без `tel:`-ссылки).
- **Часы:** `schedule` как есть; пусто → `—`.
- **Сайт:** `site` — ссылка, текст без `https://` (напр. `albertivanov.com`).
- **Telegram:** `social.telegram` — ссылка `https://t.me/<handle без @>`,
  текст `@handle`. Пустые соцсети не показывать.
- **Метро НЕ выводим** (в API YCLIENTS поля метро нет вообще — проверено на
  `/company`, `/companies`, `include[]=metro`; в адресе тоже отсутствует).

### Кнопка карты + выбор приложения
- Вместо двух ссылок «Маршрут» + «2ГИС» — **одна кнопка «Показать на карте»**.
- По тапу открывается **лист выбора приложения** (bottom-sheet) с вариантами:
  **Apple Maps, Google Maps, Yandex Navigator, Yandex Maps** + «Отмена».
  Показывать все 4 всегда (на desktop откроются веб-версии, где применимо).
- Ссылки строятся **по координатам** (`coordinate_lat/lon` из `/companies?id=`):
  - Apple Maps: `https://maps.apple.com/?ll=<lat>,<lon>&q=<title>`
  - Google Maps: `https://www.google.com/maps/search/?api=1&query=<lat>,<lon>`
  - Yandex Maps: `https://yandex.ru/maps/?pt=<lon>,<lat>&z=17&l=map`
  - Yandex Navigator: `yandexnavi://build_route_on_map?lat_to=<lat>&lon_to=<lon>`
    (построение маршрута — работает только при наличии координат).
- Если координат нет (API вернул `0`), а адрес есть — фолбэк на поиск по тексту
  адреса (Yandex `?text=`, Navigator `map_search?text=`, и т.п.); если нет ни
  координат, ни адреса — кнопку скрыть.

### Оформление
- Плашка должна быть **красиво оформлена**: иконки у строк (адрес/телефон/часы/
  сайт/telegram), аккуратная карточка, стилизованный bottom-sheet выбора карты.
  Конкретику визуала определяет архитектор/дизайн на этапе ADR.

## Открытые вопросы (для BA/архитектора)
- **ОВ-1. Реализация bottom-sheet выбора карты.** Новый компонент или
  переиспользовать существующий паттерн модалок/листов проекта. Все ссылки —
  `target="_blank" rel="noopener"`; app-схемы (`yandexnavi://`) — обычные `href`.
  Решает архитектор.
- **ОВ-2. Кэш `yc_company`.** Сохранять расширенный набор (coords, site, social,
  city). Поведение при оффлайне/пустом ответе — показ кэша, иначе прочерки;
  экран не ломать.
- **ОВ-3. Поведение Navigator без координат.** База: фолбэк на поиск по тексту
  адреса (`yandexnavi://map_search?text=`); когда координаты есть — построение
  маршрута.
- **ОВ-4. Какой `viber` делать.** В API `social.viber=@Alb0203` тоже заполнен,
  но заказчик просил только Сайт + Telegram. База: показывать только Сайт и
  Telegram, остальные соцсети — вне scope.

## Ограничения
- Vanilla HTML/CSS/JS, без новых рантайм-зависимостей; код в `docs/`.
- Данные ТОЛЬКО из YCLIENTS (+ кэш); никаких хардкод-контактов/плейсхолдеров.
- Не вводить новые секреты; токен YCLIENTS уже в проекте — не усугублять.
- Контактный блок — выделяемый/кликабельный текст (исключение из глобального
  `user-select:none` уже есть).
- Внешние ссылки — `rel="noopener"`; экранирование серверных строк (`esc`);
  `tel:`/handle нормализовать перед подстановкой в href.
- Не ломать существующие кнопки/поток на `s-confirm` и секции профиля.
- При изменении файлов из `APP_SHELL` / `sw.js` — поднять `CACHE_VERSION`.
- Минимальные изменения по объёму — только переработка плашки и источника
  данных.

## Критерий готовности
- Плашка показывает данные **только из YCLIENTS** (`/companies?id=` + кэш):
  заголовок (`public_title`), адрес с городом, телефон (или `—`), часы (или `—`),
  сайт, Telegram. Пустые значения → `—` с сохранением метки; пустые соцсети не
  отображаются; метро отсутствует.
- Объект `STUDIO` больше не используется как источник; `REVIEW_URLS` в плашке не
  используется.
- Кнопка «Показать на карте» открывает лист выбора: Apple Maps / Google Maps /
  Yandex Navigator / Yandex Maps; ссылки строятся по реальным координатам
  (`55.686006, 37.85368`); Navigator прокладывает маршрут.
- Плашка аккуратно и красиво оформлена (иконки строк, карточка, bottom-sheet);
  работает и на `s-profile`, и на `s-confirm`.
- При недоступности API — показ кэша/прочерков, экран не ломается.
- В консоли нет ошибок; PWA остаётся устанавливаемой; `CACHE_VERSION` поднята.
