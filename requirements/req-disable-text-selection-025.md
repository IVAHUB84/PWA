# Требование 025: Запрет выделения текста и изображений в интерфейсе (анти-selection / анти-callout)

## Описание
При использовании PWA на мобильном телефоне долгое удержание пальца на экране (long-press) приводит к нежелательному поведению нативного браузера:
- выделяется текст (подсветка, маркеры выделения);
- по изображениям (фото услуг, аватары, иконки) всплывает контекстное меню браузера («Сохранить изображение», «Копировать», «Поделиться»);
- картинку можно «утащить» (drag) на десктопе.

Это ломает ощущение нативного приложения: пользователь, удерживая палец на карточке/кнопке/фото, видит системное выделение вместо реакции элемента управления. Нужно запретить выделение и контекстное меню по всему интерфейсу.

## Текущее состояние
- В `docs/style.css` глобального `user-select: none` нет — выделение работает по умолчанию во всём приложении.
- `-webkit-touch-callout` нигде не задан — на iOS Safari long-press по `<img>` (фото услуг, карусель, аватары мастеров) открывает системное меню сохранения.
- `-webkit-user-drag` / draggable по изображениям не ограничен.
- Это усугубляется недавно добавленным press feedback (req-023, v1.19.x): пользователь жмёт/удерживает элемент → вместо «нажатия клавиши» получает выделение текста.

## Целевое поведение (согласовано с заказчиком)

### Область применения
- **Запретить выделение во всём UI, КРОМЕ полей ввода.** Глобально `user-select: none`, но `input`, `textarea` и любые редактируемые элементы (`[contenteditable]`) остаются с `user-select: text` — иначе в них нельзя редактировать/выделять текст (PIN-код, формы, поля админа).
- **Отключить контекстное меню и перетаскивание по изображениям.** Long-press по фото/иконкам/аватарам не должен открывать нативное меню сохранения; картинку нельзя выделить и нельзя перетащить.

### Технические средства (ориентир для архитектора)
- `user-select: none` + вендорные префиксы (`-webkit-user-select`, `-moz-user-select`, `-ms-user-select`) на корневом контейнере приложения (или `body`).
- Возврат `user-select: text` (с префиксами) для `input, textarea, [contenteditable]`.
- `-webkit-touch-callout: none` на корне (и/или явно на `img`) — убирает iOS callout-меню.
- `-webkit-user-drag: none` / `user-drag: none` и атрибут/стиль для запрета drag на `img`.
- Реализация — чистый CSS, без JS, без новых зависимостей. (Опционально, если CSS не покрывает кейс на каком-то движке — точечный обработчик, но это на усмотрение архитектора; базово — CSS.)

## Открытые вопросы (для BA/архитектора)
- **ОВ-1. Куда вешать правило-корень.** На `body`, на `#app`-обёртку или на конкретный корневой контейнер приложения — архитектор определяет реальный корневой селектор по `prototype.html`.
- **ОВ-2. Полный список исключений для выделения.** Точно `input, textarea, [contenteditable]`. Нужно ли разрешить выделение/копирование где-то ещё — например, в длинных описаниях услуг или текстах отзывов, чтобы пользователь мог скопировать (адрес, телефон, промокод)? Заказчик выбрал «весь UI, кроме полей ввода» — по умолчанию НЕ делаем дополнительных исключений; архитектор проверяет, нет ли мест, где копирование критично (телефон студии, адрес), и при необходимости поднимает это как осознанное исключение.
- **ОВ-3. Поля ввода PIN-кода.** Убедиться, что экран ввода PIN (`.pin-key` — это кнопки, а собственно поле/инпут) и любые формы остаются работоспособными: каретка, выделение, вставка кода из SMS.
- **ОВ-4. Десктоп.** На десктопе глобальный запрет выделения может мешать (например, скопировать текст мышью). Приложение в первую очередь мобильное — заказчик хочет запрет; уточнить, оставлять ли запрет и на десктопе (база — да, оставлять, поведение единое) или ослабить через media/pointer. База: единое поведение, без усложнения.

## Ограничения
- Vanilla CSS/HTML, без новых рантайм-npm-зависимостей.
- Изменения преимущественно в `docs/style.css`; при необходимости точечно `docs/prototype.html` (атрибуты на img) — минимально.
- Не сломать редактирование/выделение в полях ввода (PIN, формы записи, формы админа push).
- Не сломать функциональные жесты: свайп карусели фото услуг (req-015/017), скролл, тапы, press feedback (req-023). `user-select: none` не должен мешать pointer-событиям карусели.
- Уважать существующий функционал копирования, если он есть (проверить — есть ли кнопки «скопировать» где-либо).
- При изменении файлов из `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION` в `sw.js` (если `style.css` входит в APP_SHELL).

## Критерий готовности
- На мобильном (эмуляция touch в DevTools + реальное устройство) long-press по кнопкам, карточкам, чипам, иконкам, тексту меток НЕ вызывает выделения текста.
- Long-press по изображениям (фото услуг, карусель, аватары мастеров, иконки) НЕ открывает нативное контекстное меню сохранения (iOS Safari / Android Chrome) и не выделяет/не перетаскивает картинку.
- Поля ввода (PIN-код, все формы) сохраняют возможность ввода, выделения и вставки текста.
- Свайпы карусели, скролл, тапы и press feedback работают без регрессий.
- В консоли нет ошибок; PWA остаётся устанавливаемой; при изменении `APP_SHELL`/`sw.js` поднята `CACHE_VERSION`.
