# Требование 037: Доступность (a11y) и клавиатурная навигация

## Описание
Сделать приложение доступным с клавиатуры и для скринридеров, корректным для
пользователей с вестибулярной чувствительностью. Релиз v1.33.0. Источник —
UX-ревью от 2026-05-31. Видимое поведение для обычного пользователя не меняется —
добавляется доступность.

Объём:
1. **Клавиатурная доступность** — интерактивные `div[onclick]` критичных действий
   → `<button>` (или `role="button"` + `tabindex` + обработка Enter/Space).
2. **Закрытие по Esc** — overlay/bottom-sheet/модалки закрываются клавишей Esc
   (использовать sheet-примитив из v1.32.0).
3. **Перенос фокуса при смене экрана** — фокус переходит на заголовок/первый
   элемент нового экрана (скринридер не «теряется»).
4. **ARIA** — `aria-label` иконочным кнопкам; `aria-live` для тостов/ошибок;
   роли где нужно.
5. **`prefers-reduced-motion`** — глушить анимации переходов/карусели/scale.
6. **Видимый focus-ring** для клавиатурной навигации.

## Текущее состояние (подтверждено кодом)
- **76 `<div onclick>`** против 89 `<button>` (`prototype.html`) — треть
  интерактива недоступна с Tab/Enter и для скринридеров.
- **Всего 4 `aria-label`, 1 `role=`** на всё приложение.
- **Перенос фокуса почти отсутствует** — `.focus()` всего в 2 местах
  (`auth.js`, `review.js`); смена экрана (`go()` в `navigation.js`) фокус не двигает.
- **Esc обрабатывается лишь в 2 местах** (`admin.js`, `slots.js` — 2); overlay
  выбора категории (`services.js:31-54`), просмотр фото, прочие модалки по Esc
  не закрываются.
- **`prefers-reduced-motion`** — 1 правило в `style.css:1169` (минимально);
  слайды экранов (`.screen` transform, `style.css:125-139`), карусель фото
  (`services.js`), scale-анимации не заглушены.
- **Переходы экранов** — `transform: translateX` (`style.css:131-138`).

## Открытые вопросы (для BA/архитектора)
- **Объём конвертации `div[onclick]`.** Все 76 или только клиентские критичные
  (бронь/выбор услуги/мастера/слота/навигация)? *Предложение: критичные
  клиентские в приоритете; админские — по возможности, не блокируют релиз.*
- **`<button>` vs `role+tabindex`.** Полная замена тега может задеть вёрстку/стили.
  *Предложение: где элемент визуально не кнопка (строки списка, карточки) —
  `role="button"` + `tabindex="0"` + Enter/Space, сохраняя стили; где это уже
  кнопка по смыслу — `<button>`.*
- **Перенос фокуса** — на контейнер экрана (`tabindex="-1"` + focus) или на
  заголовок? *Предложение: на заголовок экрана, фолбэк — на контейнер.*

## Ограничения
- Vanilla JS + HTML + CSS, без зависимостей; `docs/prototype.html`,
  `docs/modules/*.js` (особенно `navigation.js`), `docs/style.css`.
- **Не менять видимое поведение** для обычного пользователя (мышь/тач) — только
  добавлять доступность; focus-ring не должен «висеть» при клике мышью
  (`:focus-visible`).
- Переиспользовать sheet/Esc из v1.32.0, не дублировать.
- Корректно в светлой/тёмной теме.
- При изменении `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION`.
- Минимальные изменения строго по объёму.

## Критерий готовности
- Основной клиентский поток (логин → услуга → мастер → слот → бронь) полностью
  проходим с клавиатуры (Tab/Enter/Space/Esc).
- Overlay/bottom-sheet/модалки закрываются по Esc.
- При смене экрана фокус переносится на новый экран (проверка скринридером/Tab).
- Иконочные кнопки имеют `aria-label`; тосты/ошибки озвучиваются (`aria-live`).
- При `prefers-reduced-motion: reduce` анимации переходов и карусели заглушены.
- Виден focus-ring при навигации с клавиатуры, отсутствует при кликах мышью.
- Ноль ошибок в консоли; PWA installable; при изменении `sw.js` поднята
  `CACHE_VERSION`.
