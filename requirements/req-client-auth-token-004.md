# Требование 004: Получение user_token клиента при авторизации

## Суть
При логине клиента через телефон (SMS-код) приложение должно получать и сохранять personal user_token клиента из YCLIENTS API (через POST /user/auth или аналогичный endpoint). Этот токен подставляется в заголовок Authorization при запросах, требующих прав клиента — в частности POST /comments для отправки отзыва.

## Контекст
Сейчас в api.js захардкожен статический admin userToken. POST /comments возвращает 401, потому что сервер ожидает личный токен клиента, а не admin-токен. При авторизации клиента сессия сохраняет user_token: '' (пустой).

## Детали
- При логине клиента вызвать YCLIENTS auth endpoint и получить его личный user_token.
- Сохранить user_token в сессии.
- YC._h() использовать user_token из сессии, если он есть; иначе fallback на статический.
- Результат: POST /comments начинает проходить с кодом 2xx и отзыв реально попадает в YCLIENTS.
