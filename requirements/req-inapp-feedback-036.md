# Требование 036: In-app обратная связь вместо нативных диалогов

## Описание
Заменить браузерные `alert()/confirm()/prompt()` на собственные in-app компоненты,
согласованные со стилем приложения, и дать пользователю выход из ошибок.
Релиз v1.32.0. Источник — UX-ревью от 2026-05-31.

Объём:
1. **Toast/снэкбар** — неблокирующая всплывающая подсказка (успех/ошибка/инфо),
   авто-скрытие, не ломает анимации и «нативное» ощущение.
2. **Bottom-sheet диалог** — для подтверждений (заменяет `confirm()`), в стиле уже
   существующих оверлеев (как `_openCatFilter`).
3. **Восстановление после ошибки** — в диалоге сетевой ошибки кнопки «Повторить»
   и «Позвонить в студию» (`tel:` из данных филиала).
4. **Индикатор офлайна / устаревших данных** — тонкий баннер при отсутствии сети
   или провале фоновой дозагрузки (после v1.31.0 каталог может быть из кэша).

## Открытые вопросы (для BA/архитектора)
- **`prompt()` для GitHub-токена** (`github.js:61,76`) — это админский ввод токена.
  Заменять на in-app форму или оставить (узкий админ-кейс, не клиентский UX)?
  *Предложение: оставить вне объёма этого релиза (админ-функция), сфокусироваться
  на клиентских alert/confirm.*
- **Тосты успеха** — добавлять везде или только там, где сейчас нет визуального
  подтверждения? *Предложение: только где нет (создание брони уже даёт экран
  s-confirm — там тост не нужен; для отмены/мелких действий — добавить).*
- **Офлайн-баннер: триггер.** Только `navigator.onLine`/offline-событие, или ещё и
  провал revalidate каталога (показ устаревших данных)? *Предложение: оба —
  «Нет соединения» при offline, «Данные могли устареть» при провале дозагрузки.*
- **Звонок в студию** — номер из данных филиала или хардкод? *Предложение: из
  данных филиала (уже загружены), фолбэк — без кнопки звонка если номера нет.*

## Ограничения
- Vanilla JS + HTML + CSS, без рантайм-зависимостей; код в `docs/modules/*.js`,
  `docs/prototype.html`, `docs/style.css`. Возможен новый модуль (напр. `toast.js`).
- Toast/sheet — переиспользуемые примитивы (будут нужны в v1.33/v1.34).
- Корректная работа в светлой и тёмной теме (токены `var(--…)`).
- Не ломать защиту от двойной отправки и существующие переходы на экраны успеха.
- `esc()` для любого вставляемого текста (сообщения ошибок от API).
- При изменении `APP_SHELL`/новый модуль в `sw.js` — поднять `CACHE_VERSION`.
- Минимальные изменения строго по объёму.

## Критерий готовности
- Клиентские `alert()` (бронь/перенос/отмена/отзыв) заменены на toast или
  bottom-sheet; нативных браузерных модалок в клиентских сценариях нет.
- При сетевой ошибке брони/отмены доступны действия «Повторить» и «Позвонить»
  (`tel:`), если номер филиала известен.
- При offline виден ненавязчивый баннер; при провале фоновой дозагрузки каталога
  — индикатор устаревших данных.
- Toast/sheet работают в обеих темах, не блокируют UI, авто-скрываются (toast).
- Защита от двойной отправки и экраны успеха не сломаны; ноль ошибок в консоли;
  PWA installable; при изменении `sw.js` поднята `CACHE_VERSION`.
