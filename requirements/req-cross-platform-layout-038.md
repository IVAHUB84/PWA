# Требование 038: Кросс-платформенный лейаут (десктоп / планшет / мышь)

## Описание
Приложение сейчас жёстко свёрстано как телефон и на десктопе/планшете выглядит как
прототип. Сделать его полноценным на больших экранах и для мыши. Релиз v1.34.0.
Источник — UX-ревью от 2026-05-31.

Объём:
1. **Адаптивный лейаут на больших экранах** — на `≥768px` контент не «телефон по
   центру», а полноценная раскладка (шире, сетка услуг/мастеров в 2-3 колонки)
   ИЛИ осознанно оформленное «устройство» с брендовым фоном (решение BA/архитектора).
2. **Скрыть фейковый iOS-хром вне iOS** — статус-бар «9:41» и dynamic island не
   показывать на десктопе/не-iOS.
3. **Hover-состояния** для мыши — карточки, строки списков, кнопки (`@media (hover:hover)`).
4. **Снять блокировку зума** (`user-scalable=no, maximum-scale=1.0`) — доступность
   масштабирования на всех платформах.
5. **Ревизия запрета выделения текста** — цены/адрес/телефон/имена выделяемы,
   запрет только на «кнопочных» элементах.
6. **Десктоп-установка PWA** — формулировки/affordance для Chrome/Edge.

## Текущее состояние (подтверждено кодом)
- **`.phone` фикс. 393×852px** со скруглением 54px и тенью (`style.css:113-122`) —
  телефон-макет. На десктопе — маленький «телефон» по центру.
- **Единственный брейкпоинт** — `@media (max-width:500px), (max-height:500px) and
  (pointer:coarse)` (`style.css:739`) уходит в фуллскрин только на малых экранах;
  десктоп/планшет не адаптированы.
- **Фейковый хром**: `.status-bar` «9:41», `.dynamic-island` (`prototype.html:39`)
  рисуются на всех платформах.
- **Только 4 `:hover`** на 60 KB CSS — мышь почти без обратной связи.
- **Зум заблокирован**: `prototype.html:5` — `maximum-scale=1.0, user-scalable=no`.
- **21 правило `user-select` (в т.ч. none)** — наследие req-025; на десктопе
  мешает копировать цену/адрес/телефон.
- **`beforeinstallprompt` обрабатывается** (`install.js:92`) — десктоп-установка
  технически работает, но копирайт «Добавить на экран»/iOS-overlay
  ориентированы на мобильный.

## Открытые вопросы (для BA/архитектора)
- **Главная развилка: настоящий адаптив vs «оформленное устройство».** Сделать на
  ≥768px широкую раскладку (большой рефактор CSS/сеток) или оставить «телефон»,
  но красиво оформить фон и убрать фейковый хром (минимальные изменения)?
  *Предложение: начать с малого — оформить «устройство» + hover + зум + select +
  скрыть iOS-хром; полный адаптив (многоколоночные сетки) вынести отдельным
  требованием, если потребуется. Решение за архитектором по объёму.*
- **Определение «iOS» для скрытия хрома** — по user-agent или по
  `display-mode: standalone` + платформе? *Предложение: показывать фейковый
  статус-бар только в standalone на iOS; на десктопе/в браузере — скрывать.*
- **`user-select`** — какие именно элементы оставить выделяемыми? *Предложение:
  весь контентный текст (цены, описания, адрес, телефон, имена); запрет — только
  на кнопках/иконках/служебных элементах.*
- **Зум** — снять полностью или ограничить (`maximum-scale=5`)? *Предложение:
  снять полностью (убрать `maximum-scale` и `user-scalable=no`).*

## Ограничения
- Vanilla JS + HTML + CSS, без зависимостей; `docs/prototype.html`,
  `docs/style.css`, при необходимости `docs/modules/install.js`.
- Мобильный вид (≤500px) **не ухудшать** — это основной кейс; изменения для
  десктопа не должны ломать телефон.
- Корректно в светлой/тёмной теме и при смене ориентации.
- При изменении `APP_SHELL`/`sw.js` — поднять `CACHE_VERSION`.
- Не трогать ветку `backup`.
- Минимальные изменения строго по согласованному с архитектором объёму.

## Критерий готовности
- На десктопе/планшете приложение не выглядит «телефоном-прототипом»: либо
  адаптивная раскладка, либо осознанно оформленное устройство (по решению ADR);
  фейковый статус-бар «9:41»/dynamic island вне iOS не показываются.
- Интерактивные элементы дают hover-обратную связь при наведении мышью.
- Зум (pinch / Ctrl+=) работает; масштабирование не заблокировано.
- Контентный текст (цены, адрес, телефон, имена, описания) выделяется и копируется
  на десктопе; «кнопочные» элементы — нет.
- Десктоп-установка PWA имеет адекватную формулировку.
- Мобильный вид (≤500px) не ухудшился; обе темы корректны; ноль ошибок в консоли;
  при изменении `sw.js` поднята `CACHE_VERSION`.
