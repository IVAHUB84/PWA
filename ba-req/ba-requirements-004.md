# BA-требования 004: Отправка отзыва через Cloudflare Worker

- **Источник:** `requirements/req-client-auth-token-004.md`
- **Дата:** 2026-05-28
- **Статус:** готово

## 1. Контекст и бизнес-цель

Studio PWA (студия «Реснички, сестрички») работает без собственного бэкенда поверх YCLIENTS API. В релизе v1.3.0 реализована отправка отзыва клиента через `POST /comments/{company}` прямо из браузера. Однако этот запрос стабильно возвращает **401 Unauthorized**.

**Причина 401 — архитектурная:**
- YCLIENTS `user_token` получается только двумя способами: (1) через сайт yclients.com по логину+паролю для сотрудников/владельцев, (2) через мобильное приложение YPLACES по SMS для клиентов.
- Ни тот ни другой способ недоступен из браузерного JS через партнёрское API.
- Статический `user_token` в `api.js` (`YC.userToken`) принадлежит admin-аккаунту и протух либо не имеет нужных прав на `POST /comments` от лица клиента.
- `POST /book_code/{company}` отправляет SMS-код для **подтверждения брони**, а не для авторизации пользователя — код от него не принимается в `POST /user/auth`.

**Бизнес-цель:** сделать так, чтобы отзыв клиента реально попадал в YCLIENTS и становился виден на карточке мастера (цель ADR-003 и BA-003). Для этого запрос `POST /comments` должен выполняться **на серверной стороне** с валидными admin-токенами, а не из браузера клиента.

**Решение:** расширить существующий Cloudflare Worker (`cf-worker/src/index.js`) новым endpoint-ом, который принимает данные отзыва от PWA и проксирует их в YCLIENTS `POST /comments` с серверными токенами.

**Стейкхолдеры:**
- Заказчик / владелец студии — отзывы должны попадать в YCLIENTS.
- Клиенты студии — кнопка «Отправить отзыв» должна работать.
- Архитектор / разработчик — реализация.

## 2. Область (scope)

### В рамках задачи
- Добавить в Cloudflare Worker новый endpoint (например `POST /review`) для проксирования отзыва в YCLIENTS.
- Worker хранит YCLIENTS partner-токен и user-токен в переменных среды (Cloudflare Secrets) — не в коде.
- PWA отправляет данные отзыва на Worker, Worker делает `POST /comments/{company}` к YCLIENTS.
- Обновить `docs/modules/review.js`: `postComment` теперь POST-ит на Worker URL, а не напрямую в YCLIENTS.
- При успехе Worker вернул ОК → поведение в PWA без изменений (подтверждение, пометка визита оценённым).
- Fail-safe сохраняется: при ошибке Worker → клиент видит сообщение, поля не очищаются.

### Вне рамок
- Авторизация клиентов через YCLIENTS — невозможна технически (см. контекст).
- Изменение внешнего вида экрана `s-review` — не затрагивается.
- Другие запросы к YCLIENTS (кроме `POST /comments`) — не переносятся на Worker.
- Хранение отзывов в Worker или промежуточном хранилище — не нужно (Worker только проксирует).

## 3. Действующие лица и сценарии использования

**Клиент студии** — оставляет отзыв из истории визитов.

Основной сценарий:
1. Клиент выбирает звёзды и текст на `s-review`, нажимает «Отправить отзыв».
2. `postComment` в `review.js` POST-ит данные на Worker URL (`https://studio-push.ivahub84.workers.dev/review` или аналогичный endpoint).
3. Worker принимает данные, делает `POST /comments/1940850` к YCLIENTS с серверными токенами.
4. YCLIENTS сохраняет отзыв — он появляется на карточке мастера (`GET /comments`).
5. Worker возвращает успех → клиент видит подтверждение, визит помечается оценённым.

Граничный сценарий (ошибка):
- Worker недоступен или YCLIENTS вернул ошибку → Worker возвращает ошибку → fail-safe в `review.js` (существующее поведение, без изменений).

## 4. Функциональные требования

- **ФТ-1.** Worker принимает `POST /review` с данными отзыва (`rating`, `text`, `staff_id`, `record_id`) и проксирует их в YCLIENTS `POST /comments/{company}`.
- **ФТ-2.** YCLIENTS-токены хранятся в Cloudflare Secrets (переменные среды Worker), не в коде PWA.
- **ФТ-3.** `postComment` в `review.js` отправляет запрос на Worker URL, а не напрямую в YCLIENTS.
- **ФТ-4.** Worker возвращает различимые исходы: успех (2xx) / ошибка — для корректной обработки fail-safe в PWA.
- **ФТ-5.** Отзыв, отправленный через Worker, появляется в `GET /comments/{company}?staff_id=N` на карточке мастера.

## 5. Нефункциональные требования

- **НФТ-1 (Безопасность токенов).** YCLIENTS-токены не передаются в браузер и не хранятся в коде PWA.
- **НФТ-2 (Отзывчивость).** Проксирование не добавляет заметной задержки сверх прямого запроса к YCLIENTS.
- **НФТ-3 (Совместимость).** Worker уже задеплоен (`studio-push.ivahub84.workers.dev`); новый endpoint добавляется к существующему коду.
- **НФТ-4 (CORS).** Worker должен разрешать запросы с домена GitHub Pages (`ivahub84.github.io`).
- **НФТ-5 (Деградация).** При недоступности Worker fail-safe в PWA работает как сейчас — клиент видит сообщение об ошибке.

## 6. Ограничения и допущения

- Cloudflare Worker уже существует: `cf-worker/src/index.js`, URL: `https://studio-push.ivahub84.workers.dev`.
- Деплой Worker: `cd cf-worker && npx wrangler deploy` (требует `CLOUDFLARE_API_TOKEN`).
- YCLIENTS-токены добавляются через `wrangler secret put` или Cloudflare Dashboard.
- Worker работает на бесплатном плане Cloudflare Workers — лимиты достаточны для объёма трафика студии.

## 7. Критерии приёмки

- **КП-1.** После нажатия «Отправить отзыв» в DevTools → Network виден запрос на Worker URL (не на `api.yclients.com/comments`).
- **КП-2.** Отзыв появляется на карточке мастера (`s-master`, `GET /comments`) после успешной отправки.
- **КП-3.** YCLIENTS-токены отсутствуют в коде PWA (`docs/`) — они в Cloudflare Secrets.
- **КП-4.** При ошибке Worker fail-safe в `review.js` работает: сообщение клиенту, поля не очищаются.
- **КП-5.** Worker успешно деплоится командой `cd cf-worker && npx wrangler deploy`.

## 8. Открытые вопросы

Все вопросы закрыты. Выбранное решение (Cloudflare Worker) подтверждено после исследования:
- Вариант A (YCLIENTS auth для клиентов) — невозможен: YCLIENTS не предоставляет API для получения client user_token из браузера.
- Вариант C (обновить статический userToken) — не решает проблему: даже свежий admin userToken может не иметь прав `client_comments_add_access`.
- Вариант B (Cloudflare Worker) — единственный работающий подход.
