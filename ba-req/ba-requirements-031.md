# BA-требования 031: Навигация админки и честные метрики сводки

- **Источник:** `requirements/req-admin-nav-metrics-030.md`
- **Дата:** 2026-05-30
- **Статус:** черновик

> Примечание о нумерации: номер исходного требования — `030`, номер BA-документа — `031` (следующий свободный в `ba-req/`, где уже есть `ba-requirements-030.md` по другому требованию). Несовпадение номеров — норма для проекта.

## 1. Контекст и бизнес-цель

В админ-панели PWA («Режим студии») есть две проблемы, снижающие удобство и доверие администратора студии:

1. **Деструктивное действие в зоне частых тапов.** Действие «Выйти» вынесено пятой вкладкой в нижний tab-bar на всех четырёх админских экранах (Сводка, Лента, Клиенты, Рассылка). Выход — не раздел навигации, а одноразовое деструктивное действие; его соседство с рабочими вкладками провоцирует случайные нажатия и непреднамеренный выход из режима студии.
2. **Фейковая метрика подрывает доверие к сводке.** На экране «Сводка» карточка «4.8 ★ / Средняя оценка / за 30 дней» содержит захардкоженное число — оно не связано ни с каким источником данных и не меняется. Рядом стоят реальные цифры (записи, выручка, клиенты, обновляемые из YCLIENTS API), поэтому статичный рейтинг выглядит как живая метрика и обесценивает доверие ко всей сводке. Дополнительно элементы «динамики» (`stat-diff`) присутствуют в разметке, но пусты.

**Заказчик / стейкхолдеры:** владелец студии и администратор, работающие в режиме студии; владелец продукта (доверие к данным).

## 2. Область (scope)

### В рамках задачи
- Убрать действие «Выйти» из нижнего tab-bar на всех четырёх админских экранах.
- Перенести выход в шапку админ-панели (`admin-header`) как явное действие с подтверждением.
- При подтверждённом выходе — переход на экран логина (`go('s-login')`) без сброса клиентской сессии (решение по ОВ-1, см. раздел 8).
- Заменить на экране «Сводка» захардкоженную карточку рейтинга «4.8 ★ / Средняя оценка» честной метрикой «Средний чек сегодня», вычисляемой из уже загруженных в `renderAdminDashboard()` данных (решение по ОВ-2, см. раздел 8). Сетка остаётся из 4 карточек — карточка не удаляется, а перепрофилируется.
- Убрать пустые элементы «динамики» (`stat-diff`) из интерфейса сводки.

### Вне рамок (explicitly out of scope)
- Сброс клиентской сессии (полный `logout()` / `clearSession()`) при выходе из режима студии — по решению ОВ-1 не выполняется.
- Удаление карточки рейтинга — по решению ОВ-2 карточка не удаляется, а перепрофилируется в «Средний чек сегодня».
- Интеграция реального источника средней оценки/рейтинга из отзывов (агрегата отзывов в PWA сейчас нет — см. раздел 6).
- Расчёт динамики «за период» с сравнением с предыдущим периодом (требует загрузки данных за прошлый период — отдельная задача).
- Изменение механизма входа в админку (PIN-код), управления GitHub-токеном (кнопка «🔑 Токен»), Worker-настроек.
- Любые изменения клиентской (не-админской) части приложения.
- Техническая реализация (структура DOM, выбор компонента подтверждения) — задача архитектора.

## 3. Действующие лица и сценарии использования

**Администратор студии** — входит в режим студии по PIN-коду, работает с экранами Сводка / Лента / Клиенты / Рассылка преимущественно с мобильного устройства, тапает по нижнему tab-bar.

- **Сценарий А (текущая боль):** администратор переключается между разделами, случайно задевает пятую вкладку «Выйти» и неожиданно покидает режим студии.
- **Сценарий Б (целевой):** администратор завершает работу — осознанно нажимает кнопку выхода в шапке, подтверждает действие в диалоге и выходит из режима студии.
- **Сценарий В (доверие к данным):** администратор смотрит сводку за день и видит только реальные показатели; ни одна карточка не показывает выдуманное число под видом живой метрики.

## 4. Функциональные требования

- **ФТ-1. Tab-bar без выхода.** На всех четырёх админских экранах (`s-admin`, `s-admin-feed`, `s-admin-clients`, `s-admin-push`) нижняя панель навигации содержит ровно 4 раздела: Сводка, Лента, Клиенты, Рассылка. Действие «Выйти» из tab-bar удалено. Переключение между оставшимися разделами и индикация активной вкладки работают как прежде.
- **ФТ-2. Выход из шапки.** На админских экранах в шапке (`admin-header`) доступно явное действие выхода из режима студии (кнопка/иконка с понятной семантикой выхода). Оно не должно конфликтовать с уже существующими элементами шапки (на «Сводке» — кнопка «🔑 Токен»; на «Ленте»/«Рассылке» — кнопки «+ Новый»/«+ Новая»).
- **ФТ-3. Подтверждение выхода.** Перед фактическим выходом администратору показывается запрос подтверждения с текстом вида «Выйти из режима студии?». При отмене администратор остаётся на текущем экране в режиме студии; при подтверждении выполняется выход.
- **ФТ-4. Поведение выхода (решено по ОВ-1).** При подтверждённом выходе выполняется переход на экран логина (`go('s-login')`). Клиентская сессия при этом **не сбрасывается** — полный `logout()` / `clearSession()` не вызывается. Таким образом выход из режима студии возвращает на экран входа без потери клиентской сессии.
- **ФТ-5. Никаких фейковых метрик в сводке.** На экране «Сводка» отсутствует захардкоженное значение метрики, выдаваемое за живой показатель. Захардкоженного «4.8 ★ / Средняя оценка / за 30 дней» в HTML не остаётся: карточка перепрофилируется в «Средний чек сегодня» (см. ФТ-6).
- **ФТ-6. Средний чек сегодня (решено по ОВ-2).** Карточка рейтинга заменяется метрикой «Средний чек сегодня». Значение вычисляется из уже загруженных в `renderAdminDashboard()` данных как `rev / recs.length` (выручка за сегодня ÷ число записей за сегодня), без дополнительных запросов к API. При нуле записей (`recs.length === 0`) показывается прочерк «—», без деления на ноль. Подпись карточки и её значение задаются/обновляются в `renderAdminDashboard()`; в HTML захардкоженного значения и подписи «Средняя оценка / 4.8 ★ / за 30 дней» нет. Сетка сводки остаётся из 4 карточек.
- **ФТ-7. Без пустых элементов динамики.** Пустые элементы `stat-diff` удаляются из интерфейса сводки. Наполнение их динамикой за период в этом релизе не выполняется (вне scope, см. раздел 2).

## 5. Нефункциональные требования

- **НФТ-1. Консистентность.** Tab-bar админки повторяется в разметке четыре раза; изменение должно быть внесено одинаково во всех четырёх экранах.
- **НФТ-2. Только реальные данные.** В интерфейсе сводки не отображаются выдуманные «живые» показатели. Допустимы только значения из реальных источников или явно нейтральные заглушки (прочерк).
- **НФТ-3. Без новых зависимостей.** Реализация в рамках стека проекта (vanilla HTML/CSS/JS, код в `docs/`), без новых рантайм-зависимостей.
- **НФТ-4. Отсутствие ошибок и устанавливаемость.** После изменений в консоли браузера нет ошибок; PWA остаётся устанавливаемой (Lighthouse).
- **НФТ-5. Кэш Service Worker.** Если изменения затрагивают `APP_SHELL` / `sw.js`, версия `CACHE_VERSION` повышается (требование проекта).

## 6. Ограничения и допущения

**Ограничения (установлены по фактическому коду):**
- Текущая пятая вкладка вызывает `go('s-login','tab')` — это **только навигация на экран входа, без сброса сессии**. Реальный выход (`clearSession()`) реализован в отдельной функции `logout()` (`docs/modules/auth.js`), которая из админки сейчас не вызывается. То есть нынешний «Выйти» в админке не выполняет настоящего logout.
- Вход в режим студии (`enterAdmin()` в `auth.js`) — проверка PIN-кода; **отдельная админ-сессия в хранилище не создаётся**. Признака «администратор залогинен» в localStorage нет — режим студии определяется только тем, что пользователь находится на админских экранах.
- Реальные цифры сводки уже вычисляются в `renderAdminDashboard()` (`docs/modules/admin.js`): количество записей, выручка (`rev`), уникальные клиенты. Средний чек выводится из этих же данных без новых запросов.
- **Реального источника средней оценки/рейтинга в PWA нет.** Отзывы (`docs/modules/review.js`) — это клиентская форма, отправляющая оценку наружу (`postComment` / внешние URL 2GIS, Яндекс); агрегированного рейтинга, доступного для чтения в приложении, нет. Следовательно, заменить «4.8 ★» реальной средней оценкой из имеющихся данных невозможно — остаются варианты: средний чек либо удаление карточки.
- Шапки админских экранов уже заняты справа: «Сводка» — «🔑 Токен» (`_adminResetToken()`, управление GitHub-токеном через `prompt`), «Лента» — «+ Новый», «Рассылка» — «+ Новая»; «Клиенты» — без правой кнопки. Кнопка выхода должна разместиться, не вытесняя и не дублируя эти элементы.

**Допущения:**
- Администратор работает преимущественно на мобильном устройстве; зона нижнего tab-bar — зона частых тапов.
- Подтверждение выхода через `confirm()` или существующий компонент модалки приемлемо (выбор — за архитектором).
- Удаление пустых `stat-diff` визуально приемлемо без замены на иной контент.
- Выход из режима студии без сброса клиентской сессии (решение ОВ-1) приемлем: режим студии определяется только нахождением на админских экранах, отдельной админ-сессии в хранилище нет (раздел 6), поэтому переход на `s-login` достаточен для покидания режима.

## 7. Критерии приёмки

- **КП-1.** На всех четырёх админских экранах нижний tab-bar содержит ровно 4 вкладки (Сводка / Лента / Клиенты / Рассылка); вкладки «Выйти» нет ни на одном экране. (ФТ-1, НФТ-1)
- **КП-2.** Переключение между разделами и подсветка активной вкладки работают корректно на всех четырёх экранах. (ФТ-1)
- **КП-3.** В шапке админских экранов есть действие выхода из режима студии, не перекрывающее «🔑 Токен», «+ Новый», «+ Новая». (ФТ-2)
- **КП-4.** При нажатии выхода появляется запрос подтверждения; отмена оставляет администратора на текущем экране, подтверждение выполняет выход. (ФТ-3)
- **КП-5.** Подтверждённый выход выполняет переход на экран логина (`go('s-login')`); клиентская сессия не сбрасывается (`logout()`/`clearSession()` не вызываются). (ФТ-4)
- **КП-6.** На экране «Сводка» нет захардкоженного «4.8 ★ / Средняя оценка / за 30 дней» в HTML; вместо него карточка «Средний чек сегодня». Сетка сводки содержит 4 карточки. (ФТ-5, ФТ-6)
- **КП-7.** Карточка «Средний чек сегодня» показывает `rev / recs.length` из уже загруженных данных, без дополнительных запросов к API; при нуле записей показывает «—» без ошибки/деления на ноль; подпись и значение формируются в `renderAdminDashboard()`. (ФТ-6)
- **КП-8.** В интерфейсе сводки нет пустых элементов `stat-diff` (они удалены). (ФТ-7)
- **КП-9.** В консоли браузера нет ошибок; PWA остаётся устанавливаемой; при изменении `APP_SHELL`/`sw.js` повышена `CACHE_VERSION`. (НФТ-4, НФТ-5)

## 8. Открытые вопросы

Блокирующих открытых вопросов нет: ОВ-1 и ОВ-2 решены заказчиком 2026-05-30.

- **ОВ-1 (поведение сессии при выходе) — РЕШЕНО (2026-05-30).** Принят вариант (а): при подтверждённом выходе выполняется переход на экран логина (`go('s-login')`), клиентская сессия **не сбрасывается** — полный `logout()` / `clearSession()` не вызывается. Перед выходом показывается диалог подтверждения «Выйти из режима студии?». Отражено в ФТ-3, ФТ-4, КП-4, КП-5.
- **ОВ-2 (чем заменить рейтинг) — РЕШЕНО (2026-05-30).** Принят вариант (а): фейковая карточка «4.8 ★ / Средняя оценка» заменяется на «Средний чек сегодня» = `rev / recs.length` из уже загруженных в `renderAdminDashboard()` данных, без новых запросов; при нуле записей — «—». Карточку **не удаляем**, сетка остаётся из 4 карточек. Отражено в ФТ-5, ФТ-6, КП-6, КП-7.
- **ОВ-3 (размещение кнопки выхода — для архитектора/дизайна, не блокирует).** На «Сводке» шапка уже содержит «🔑 Токен». Уточнить: выход — отдельная кнопка/иконка рядом, или объединить «Токен» + «Выход» в меню. На «Ленте»/«Рассылке» шапка занята кнопкой «+». Решение по единому паттерну размещения — за архитектором; на оформление и приёмку не влияет.

Противоречий с существующими ADR не выявлено.
